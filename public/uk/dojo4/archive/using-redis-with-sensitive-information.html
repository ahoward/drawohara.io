Я провів години, досліджуючи та плануючи, як запобігти redis записувати будь-які значення бази даних на диск, оскільки екземпляр redis буде обробляти конфіденційну інформацію. (У нашому випадку ми зберігаємо cvv на 60 хвилин і через вимоги PCI до використання кредитних карток, абсолютно не можемо записувати ці значення на диск - ніколи.)

Прочитав документацію, файл конфігурації, загуглив, протестував і навіть спробував сказати йому писати в /dev/null.

Нарешті відповідь прийшла від ‘TheRealBill_here’ на ірк каналі #redis:

dbfilename ""

у файлі конфігурації.

Ось посилання на нашу цілу розмову, якщо вас це цікавить.

Я також надіслав pull-запит, щоб додати примітку у файлі конфігурації для redis.

Ця відповідь прийшла до мене після того, як я вже прийняв кілька інших рішень щодо налаштування redis для зберігання інформації про кредитні картки:

  • loglevel має значення за замовчуванням info, яке не друкує кожне читання/запис, тому з цим все гаразд. Хоча для нашого випадку ми, ймовірно, відчували би себе комфортно, просто відключивши реєстрацію.
  • save директива встановлена на save "", щоб забезпечити відключення знімків.
  • replication не використовується. Під час використання реплікації майстер ініціалізує раб через запис свого власного вмісту бази даних на диск, надсилає рабу цей файл, а раб завантажує дамп бази даних у свою базу даних. Очевидно, ми не можемо цього зробити. (Також зверніть увагу, що якщо ми додамо раб у якийсь момент, він не зможе синхронізуватися з нашою поточною конфігурацією dbfilename "". Дивіться лог помилок, коли ви викликаєте bgsave з redis-cli самостійно.)

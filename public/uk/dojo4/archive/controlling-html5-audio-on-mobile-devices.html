Теги HTML5 <audio> досить прості. Дано:

<audio id="player"></audio>

Цей фрагмент JavaScript:

player = document.getElementById('player');
player.src = 'some-audio-file-url';
player.play()

почне відтворювати те, на що вказує src. (Порада: якщо ви використовуєте jQuery, вам потрібно отримати фактичний HTML-елемент за допомогою player = $('#player')[0] або player = $('#player').get(0))

Однак, якщо користувач знаходиться на мобільному пристрої під управлінням iOS або Android, вищевказане не відтворить нічого. Обидві ці операційні системи не дозволяють веб-аудіо (або відео) починати трансляцію без взаємодії користувача. Логіка полягає в тому, що трансляція даних через мобільну мережу може коштувати грошей і не повинна відбуватися без того, щоб користувач фактично натиснув кнопку.

Все добре, але що, якщо ви хочете програти звук у відповідь на подію, наприклад, видавати звуковий сигнал браузера кожну хвилину:

player = document.getElementById('player');
player.src = 'beep-url';
beeper = setInterval(function() { player.play(); } ,1000 * 60);

Це буде працювати добре в настільних браузерах, але на мобільних пристроях виклик play() нічого не зробить.

На щастя, існує три речі, які роблять обхід простим:

  1. Користувачу потрібно лише один раз натиснути кнопку відтворення для кожного аудіоелемента.
  2. «Натискання кнопки відтворення» також може означати запуск play() через подію кліку.
  3. Відтворення аудіоелемента без джерела є допустимою операцією, що нічого не робить.

Коротше кажучи, все, що нам потрібно, це щоб користувач натиснув кнопку, яка відтворює порожній аудіоелемент, щоб отримати контроль над ним:

<audio id="player"></audio>
<button id="ok">OK щоб сигналити</button>
// Припускається jQuery
$("#ok").click(function() {
 player = document.getElementById('player');
 $( this ).slideUp();
 player.play(); // Відтворити порожній елемент, щоб отримати контроль
 player.src = 'beep-url'; // Встановити справжнє аудіоджерело
 beeper = setInterval(function() { player.play(); } ,1000 * 60);
});

Клік може бути будь-яким, більш тонка версія полягала б у прив'язці її до кнопки закриття інструктивного модального вікна на початку гри.

Ця техніка, можливо, трохи розмиває межу між духом і буквою закону, але вона виконує свою роботу і дозволяє створити послідовний досвід для користувачів як настільних, так і мобільних пристроїв.

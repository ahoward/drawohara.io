З цією багатовіконною конфігурацією, коли мені потрібно змінити проект, над яким я працюю, мені потрібно або відкрити нове вікно Terminal з його власними вкладками для цього проекту та мати справу з більш ніж необхідною кількістю відкритих вікон (не варіант для мінімаліста, як я) або закрити всі вкладки, які я маю відкритими, щоб лише відкрити їх знову, коли я перемикаюся назад на цей проект.

Ну, як і в більшості днів, я дізнався геніальне рішення цієї проблеми від мудрого та всемогутнього лідера dojo4, Ари. Він використовує програму GNU Screen. У рамках однієї вкладки Terminal Screen дозволяє відкривати кілька "вікон" (також відомих як вкладки) у тій самій вкладці Terminal. Ось як ним користуватися:

# створення та повторне підключення до сеансу
screen -S [name] = створити іменований екран
screen -D -R [name] = повторно підключитися до іменованого екрану та відключити інших користувачів, якщо потрібно
screen -list = перелік екранів

# (вихід зі всіх вікон у сеансі екрану вийде з цього сеансу. Але сеанс екрану є запущеним процесом, тому ви можете вручну завершити його.)

# у сеансі екрану:
ctrl-A + ? = допомога
ctrl-A + " = перелік вікон
ctrl-A + c = створити нове вікно
ctrl-A + n = наступне вікно
ctrl-A + p = попереднє вікно
ctrl-A + A = перейменувати вікно

# псевдоніми, які ми використовуємо
alias sl='screen -list '
alias sdr='screen -d -r '
alias s='screen -D -R '
alias ss='screen -S'

sdr [name] = повторно підключитися до іменованого екрану та відключити інших користувачів, якщо потрібно
ss [name] = створити іменований екран
sl = перелік екранів

Ще одна дуже крута функція Screen полягає в тому, що коли ви відключаєтеся від сеансу Screen, всі ваші вікна залишаються запущеними, і коли ви повторно підключаєтеся до сеансу Screen, у вас знову є всі ваші вікна.

Коли ми обговорювали це в нашому офісі, Раян Кук створив гарний маленький допоміжник Ruby для створення сеансів Screen, які мають назву вашої поточної директорії.

Таким чином, мій робочий процес змінився від використання кількох вкладок у Terminal до використання однієї вкладки у Terminal, де я повторно підключаюся до сеансу Screen для кожного проекту, над яким працюю. Це дозволяє мені дуже швидко перемикатися між проектами та припинити втрачати час на повторне налаштування та закриття всіх вкладок Terminal, які я використовую для проекту.

Тепер мені просто потрібно перевести більше речей на використання терміналу (наприклад, мого редактора), щоб дійсно максимально скористатися Screen...

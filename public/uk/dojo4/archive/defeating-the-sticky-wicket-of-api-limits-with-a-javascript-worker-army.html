TL;DR

коли у вас занадто багато роботи на сервері з обмеженнями API

перекладіть деяку роботу своїм клієнтам у браузери для виконання - і відправте результати назад у вашу базу даних

проблема

Мій тиждень минув у боротьбі з (g-provider) обмеженнями API і занадто великою кількістю запитів щодо геолокації, що надходили з серверів. Я трохи попрацював над цим, виконуючи завдання на задньому плані з періодичними повторними спробами - ви знаєте: стандартні речі...

API, про який йдеться, обмежений за IP, і в документації рекомендується скористатися цим, виконуючи більшість запитів API на стороні клієнта - в JS.

У даному випадку мені абсолютно потрібні результати запитів API на стороні сервера, тому я вирішив це компромісне (в більш ніж одному сенсі) рішення

рішення

припустімо, що деякі клієнти готові витратити свій CPU на заповнення даними, які їм цікаво обробляти. Простіше кажучи, змусіть деяких своїх клієнтів виконувати роботи з черги JavaScript для того, щоб запустити помпу, якою їм цікаво користуватися.

У моєму випадку деякі вибрані інтерфейси (ті, які цікавляться результатами цих завдань) запускають невеликий iframe

<!-- вставте виконавця черги завдань на певні сторінки/перегляди/макети -->
<iframe height="0" width="0" style="display:none;" src="/javascript_jobs/runner"></iframe>
<!-- виконавець завдань -->
<%= javascript_include_tag :jquery %>
<%= javascript_include_tag :jobs %>
<script>
  jQuery(function(){
    jobs.complete = function(job){
      document.write('завершено завдання ' + job.id);
      document.write('<br>');
    };
    setTimeout(function(){ jobs.start(); }, 1000);
  });
</script>

JS, необхідний для інтерфейсу з чергою завдань, короткий та простий

Все, що він робить, це отримує завдання, виконує його та відправляє результати назад на сервер - дотримуючись як пропускної здатності, так і максимальної кількості виконаних завдань.

if(!window.jobs){
//
  window.jobs = {};
  window.jobs.count = 0;
  window.jobs.max = 256;
  window.jobs.throttle = 1000;
  window.jobs.complete = function(){};
//
  jobs.get_next_job = function(){
    var success = function(response){
      job = response['data']['job'];

        jobs.count++;
        jobs.run(job, function(job){
          if(jobs.count < jobs.max){
            setTimeout(jobs.get_next_job, jobs.throttle);
          }
        });
      }
    };
    jQuery.ajax({
      'url'     : '/api/jobs/next

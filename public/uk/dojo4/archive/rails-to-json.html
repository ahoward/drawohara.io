rails, включно з rails3, має цей чудовий шматок коду, який стосується надання методу “to_json” вбудованим об'єктам:

# Хак для завантаження гему json першим, щоб ми могли перевизначити його to_json.
    begin
      require 'json'
    rescue LoadError
    end
    # Гем JSON додає кілька модулів до базових класів Ruby, які містять визначення :to_json, перевизначаючи
    # їх поведінку за замовчуванням. Тобто, нам потрібно визначити базовий метод to_json для всіх з них,
    # інакше вони завжди будуть використовувати реалізацію гему to_json, що є несумісним у
    # декількох випадках (наприклад, реалізація JSON для Hash не працює) з успадкуванням
    # і, наслідком, класи, такі як ActiveSupport::OrderedHash, не можуть бути серіалізовані до json.
    [Object, Array, FalseClass, Float, Hash, Integer, NilClass, String, TrueClass].each do |klass|
      klass.class_eval <<-RUBY, __FILE__, __LINE__
# Скидає об'єкт у JSON (JavaScript Object Notation). Див. www.json.org для більшої інформації.
def to_json(options = nil)
ActiveSupport::JSON.encode(self, options)
end
RUBY
    end

те, що це ефективно робить, - затирає метод “to_json”, який надає гем json, своїм, який, на думку active_support, кращий. Коментар про те, що json зламаний, застарів, і це був хак з самого початку. У моєму випадку мені дійсно подобається такий підхід у моєму /api контролері:

  def to_json(object)
    if Rails.env.production?
      JSON.generate(object)
    else
      JSON.pretty_generate(object)
    end
  end

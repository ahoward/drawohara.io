Récemment, j'ai dû me plonger dans AngularJS sur un site Rails que nous construisions pour un client. En plus du contenu de la page, le client a certaines pages qui affichent des titres qui lient à des sources externes mais sont intégrés dans notre pile Rails via une tâche rake exécutée périodiquement. Le site doit être fortement mis en cache via un CDN, comme CloudFront, mais pour garder les titres à jour, nous avons utilisé AngularJS (car le client était familier avec, donc plus facile à maintenir pour eux) pour interroger directement la pile rails pour les derniers titres.

Cela nécessitait de faire les choses de manière inter-domaines puisque le domaine du site serait pointé vers CloudFront. Je n'avais jamais écrit d'API Rails qui retournait du JSONP ou un service AngularJS auparavant, donc j'ai dû explorer et c'est ce qui a fonctionné pour moi. (Vous pouvez également lire le gist.)

Note : Si vous n'êtes pas familier avec JSONP, cela n'aura pas beaucoup de sens. Lisez ceci.

Service AngularJS

AngularJS a le concept de services, qui sont simplement des singletons pouvant être utilisés où vous en avez besoin. Dans notre cas, vous pouvez considérer notre service de titres comme le client API de titres qui encapsule nos appels JSONP dans une syntaxe plus simple et centralisée. AngularJS fournit certains services par défaut, mais vous pouvez écrire les vôtres. En fait, nous aurions pu utiliser le service intégré $resource service au lieu d'écrire le nôtre, cependant j'ai eu des difficultés à faire fonctionner JSONP et je voulais mieux comprendre les composants AngularJS de bas niveau, alors j'ai écrit le mien :

angular.module('app')
  .factory('HeadlineService', ['$http',
    function($http) {
      'use strict';

      var BASE_URL = "<%= [App.settings.api_base_url, '/services/headlines'].join %>",
          CALLBACK_STRING = "?callback=JSON_CALLBACK";

      return {
        getHeadlines: function(){
          return $http.jsonp(BASE_URL + CALLBACK_STRING)
        },
        getHeadlineForId: function(id){
          return $http.jsonp(BASE_URL + "/" + id + ".json" + CALLBACK_STRING)
        }
      }

    }
  ]);

Le service ci-dessus utilise le service intégré $http service, qui fournit simplement un wrapper AJAX, pour faire des requêtes à notre backend rails pour obtenir des titres. Quelques points à noter :

  • Il retourne un objet avec 2 fonctions : getHeadlines et getHeadlineForId. Nous pouvons ensuite utiliser ces fonctions comme raccourcis pour parler à l'API Rails dans nos contrôleurs AngularJS.
  • Il utilise JSONP. Remarquez que nous utilisons $http.jsonp et que nous passons une chaîne de requête ?callback=JSON_CALLBACK. AngularJS remplace la chaîne JSON_CALLBACK par le nom d'une fonction de rappel qu'il crée pour vous.
  • Il utilise un domaine différent. Eh bien, c'est attendu puisque nous faisons du JSONP, mais nous y parvenons en ajoutant une extension .erb au fichier et en utilisant ce que nous avons défini pour App.settings.api_base_url dans l'application Rails comme domaine avec lequel AngularJS doit communiquer. Rappelez-vous que cela permet de contourner la couche de mise en cache du CDN afin que nos titres soient toujours les plus récents.

Contrôleur AngularJS

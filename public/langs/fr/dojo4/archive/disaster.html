J'ai lancé une machine aujourd'hui.

J'ai travaillé jusqu'à près de 2h du matin, la machine ronronnant chaudement, et je me suis levé pour trouver une boîte lente et non réactive.

Le disque s'est brisé en morceaux.

http://dojo4.com est sous pression sur plusieurs projets ce mois-ci, donc, bien sûr, il est logique que certaines machines décident de quitter le fantôme aujourd'hui…. #feck.

Chaque fois que je récupère une machine en panne, cela devient plus facile, et aujourd'hui n'a pas fait exception :

Depuis la dernière panne de boîtes, j'ai travaillé dur pour mettre la plupart de mes données dans le cloud.

J'étais donc heureux de n'avoir presque rien à m'inquiéter après avoir réalisé que l'Apple Store allait prendre 72 heures complètes pour commander un disque pour mon 17" en fin de vie. Plutôt, j'ai juste dépensé 4k pour un nouveau 15" MBP rapide comme l'éclair et je me suis lancé dans la récupération…

Voici un bref aperçu de ma stratégie de sauvegarde actuelle, mélangée à quelques réflexions sur la façon dont la gestion des données change rapidement :

  • les applications gèrent vos données plus que jamais. utilisez cela à votre avantage.

  • les sauvegardes complètes sont stupides. vous devez être sélectif pour restaurer rapidement. cela demande de la réflexion. qu'est-ce qui est plus rapide à restaurer après un crash vs réinstaller ? qu'est-ce qui est plus simple à réinstaller vs reconfigurer ? indice : les données, pas le code et la configuration.

  • les systèmes d'exploitation évoluent plus vite que votre logiciel de sauvegarde : vous ne devriez sauvegarder que les données et non la configuration ou, pire, les informations binaires. les fichiers point sont bien sûr une exception !

  • google est le roi pour les e-mails, les contacts et le calendrier. abandonnez toutes les autres solutions.

  • utilisez git pour votre code. ne le sauvegardez pas. passez à autre chose.

  • utilisez things.app ou quelque chose basé sur le cloud pour votre productivité. les listes de tâches non synchronisées sont mortes.

  • posséder de la musique est une idée démodée. utilisez rdio, sspotify, ou en dernier recours iCloud. mais assurez-vous définitivement de ne pas avoir de raison de sauvegarder votre musique.

  • les photos sont un casse-tête. ceci est un appel d'offres pour des suggestions.

  • les films. les films ne devraient pas être sur la machine d'un développeur. voilà, je l'ai dit.

  • dropbox. utilisez ceci pour tout le reste et évitez les mécanismes intégrés stupides de votre système d'exploitation.

Résumé :

1) utilisez des applications basées sur le cloud

2) posséder de la musique et des films est un concept stupide. laissez tomber.

3) mettez tout le reste dans dropbox

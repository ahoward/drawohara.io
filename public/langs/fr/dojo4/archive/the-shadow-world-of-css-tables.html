L'avenir des mises en page web est prometteur, en particulier pour les développeurs web chevronnés dont les tableaux profondément imbriqués hantent toujours internet. Mais jusqu'à ce que flexbox soit prêt pour le grand public, les flottements CSS—le mécanisme de mise en page de base pour les frameworks CSS comme Twitter Bootstrap et Zurb Foundation—devront suffire.

Ou le feront-ils ?

Certains des brillants jeunes développeurs web d'aujourd'hui ne sont peut-être pas au courant qu'il y a eu un vide créé lorsque les mises en page basées sur les TABLEAUX ont été détrônées. CSS Zen Garden, qui a mis en avant la promesse des mises en page CSS et a porté un coup fatal aux mises en page basées sur les TABLEAUX, n'a pas nécessairement couronné un successeur.

Il y avait la technique du Saint Graal, le Positionnement Absolu Factice, et bien d'autres tranches de PIE. C'était une période d'expérimentation, dont l'esprit s'est quelque peu estompé à la lumière de tous les beaux frameworks disponibles (bien que certaines personnes expérimentent de temps en temps).

Une technique qui n'a pas réussi à s'imposer était les mises en page de tableaux CSS. Plusieurs choses ont conspiré contre elle :

  1. le titre du livre qui en faisait la promotion, Tout ce que vous savez sur CSS est faux, était, admettons-le, un peu alienant
  2. le nom même de la technique évoquait des démons du passé (mises en page TABLEAUX !)
  3. elle était en avance sur son temps—avant que deux navigateurs importants de l'époque ne la supportent (IE6/IE7)

Mais récemment, plusieurs personnes ont réexaminé la technique car, en une ou deux lignes de code, elle permet un positionnement que d'autres techniques ne peuvent pas réaliser sans astuces ou JS. De plus, elle offre des tours de passe-passe de l'ordre de la source qui sont utiles dans les contextes réactifs.

Centrage vertical

Les développeurs frontend sont familiers avec le défi de l'alignement vertical des éléments dans une ligne lorsque les hauteurs sont inconnues. En effet, c'est généralement le premier problème à résoudre dans le HEADER de tout site Web (logo et navigation).

Les TABLEAUX facilitaient le centrage vertical avec l'attribut VALIGN. Dans les mises en page basées sur les flottements, les choses sont beaucoup plus floues, nécessitant beaucoup de ruses (positionnement, rembourrage ou hauteurs de ligne généralement).

Entrent les tableaux CSS, qui rendent le centrage vertical trivial avec la propriété vertical-align: middle. Pour illustrer, prenez le balisage suivant : 

          <div class="row-table">
            <div class="cell">
              foo
            </div>
            <div class="cell">
              bar
            </div>
            <div class="cell">
              baz
            </div>
            <div class="cell">
              foo<br>bar<br>baz<br>qux
            </div>
          </div>

Combinez-le avec le CSS suivant : 

          .row-table {
            display: table;
            table-layout: fixed;
            width: 100%;
          }

          .cell {
            display: table-cell;
            width: 25%;
            vertical-align: middle;
          }

Et vous obtenez la mise en page centrée verticalement suivante, malgré le contenu plus grand de la dernière colonne :

foo
bar
baz
foo
bar
baz
qux

Un avantage supplémentaire de cette méthode est que la plupart des développeurs backend comprennent les TABLEAUX (Je te regarde, Ara). Donc, si vous écrivez des styles pour que d'autres les utilisent, vous pouvez créer un système de grille OOCSS avec des classes et sous-classes intuitives (".col", ".col-valign-middle", ".col-valign-top", etc) pour faciliter leur vie.

Colonnes de hauteur égale

Avec le même balisage que ci-dessus, il est assez facile de créer des colonnes de hauteur égale ; nous allons simplement ajouter quelques couleurs de fond pour démontrer :

foo
bar
baz
foo
bar
baz
qux

Bien sûr, il existe des bibliothèques JS qui font en sorte que les colonnes flottantes fassent cela, mais il y a tout un tas de choses amusantes de redimensionnement de fenêtre à faire dans un contexte réactif pour empêcher les hauteurs (calculées par JS) des colonnes de s'écouler vers les vues plus petites.

Au lieu de cela, avec une simple requête média, je peux faire en sorte que les tableaux CSS se transforment en blocs (display: block) ou en groupes de tableaux (display: table-header-group) en fonction de ce qui est approprié pour le contexte donné.

Astuces de tri de source

Les propriétés des tableaux CSS peuvent être utiles lorsque vous souhaitez remanier la main qui vous a été distribuée par l'ordre de la source HTML. Dans les mises en page basées sur les flottements, vous pouvez pousser et tirer des éléments dans des vues plus grandes, mais les choses s'empileront de la même manière que l'ordre de la source HTML dans les plus petites.

Les tableaux CSS vous permettent de réorganiser considérablement les choses, en particulier sur les vues plus petites. Prenons notre exemple précédent, par exemple :

0
1
2
3

En changeant le paramètre d'affichage pour l'une des propriétés de groupe de tableaux (display: table-caption, display: table-header-group, display: table-row-group, ou display: table-footer-group), nous pouvons empiler les éléments de manière très différente de celle qu'ils apparaissent dans la mise en page en ligne sur les vues plus grandes (que l'ordre de la source).

Par exemple, si nous voulions que les colonnes ci-dessus s'empilent dans l'ordre suivant (2,3,1,0), nous appliquerions les propriétés d'affichage suivantes aux cellules (de gauche à droite) : display: table-footer-group (0), display: table-row-group (1), display: table-caption (2), display: table-header-group (3). Et les cellules s'empileraient alors comme ceci :

0
1
2
3

Note : J'ai dû ajouter une règle width: 100% à l'élément display: table-caption car sinon il ne remplirait pas l'espace disponible.

Pourquoi diable voudriez-vous faire cela ? Eh bien, vous pourriez faire l'un de ces astuces de navigation en pied de page astucieuses pour les petites vues et vouloir que toute la navigation qui se trouve dans votre HEADER se déplace vers le bas de la page.

Ou peut-être que vous avez simplement une colonne dans une ligne qui devrait être vue en premier dans les petites vues mais qui est affichée au milieu d'une ligne dans les vues plus grandes.

Les possibilités ne sont PAS infinies ou presque aussi robustes que celles de flexbox, mais c'est quelque chose à ranger dans le coin de votre boîte à outils.

Notes

Les tableaux CSS étaient supportés dans Internet Explorer à partir de IE8. Sinon, le support est assez robuste.

Bien que Tout ce que vous savez sur CSS est faux puisse sembler être un titre de livre intimidant, c'est en fait une bonne lecture—et certains des commentaires sur Amazon sont amusants.

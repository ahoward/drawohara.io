L'un des gems rails que nous avons utilisé dans nos nouvelles applications est Better Errors, qui permet d'être déposé dans un REPL de type pry et une page d'erreur plus agréable lorsque rails lève une exception pendant le processus de requête du navigateur.


(Source de l'image)

Cela est utile pour nos développeurs car au lieu d'obtenir une page d'erreur statique, d'aller chercher dans le code où placer un binding.pry, et de réexécuter le code sujet aux erreurs, Better Errors charge une page où nous pouvons interagir avec le code immédiatement, nous permettant de trouver le problème plus rapidement, d'écrire une correction plus rapidement, et d'itérer plus rapidement.

L'une des fonctionnalités récentes que nous venons de découvrir est la possibilité de voir la dernière erreur rpc. Nous avons un schéma d'appel ajax d'une méthode rpc dans notre application rails lorsque nous devons simplement obtenir des informations ou poster un objet simple comme favoriser un utilisateur ou autre chose. Le code qui s'exécute sur notre serveur rails et qui gère cette requête ajax ne fera pas exploser tout le serveur et n'invoquera pas automatiquement Better Errors, donc la solution consiste à aller à une URL spéciale que better errors expose pour afficher la dernière erreur lancée dans notre code rails : http://localhost:3000/__better_errors.

Au fil des ans, Dojo4 a mis en place une excellente pile de développement et d'hébergement d'applications web.  Le cloud Amazon Web Services (AWS) constitue la base de cette pile.   La flexibilité et la commodité des volumes de disques Elastic Block Store (EBS) sont convaincantes et nous les utilisons pour les volumes système ainsi que pour les volumes de données pour déployer des applications et persister des bases de données sur nos ordinateurs cloud Elastic Compute Cloud (EC2).  Il y a quelque temps, à la fin d'une conversation sur l'architecture d'hébergement, Ara a déclaré de manière désinvolte 'Oh, et les sauvegardes périodiques des disques devraient simplement se faire automatiquement chaque fois que nous lançons une nouvelle instance EC2'.  J'ai donc commencé à travailler sur un petit script appelé DREBS (Disaster Recovery for Elastic Block Store).  DREBS peut être intégré à votre Amazon Machine Image (AMI) ou configuré dans vos mécanismes de provisionnement cloud Chef ou autres.   DREBS peut être configuré pour prendre des snapshots périodiques des volumes EBS attachés à l'instance EC2 sur laquelle il s'exécute.  Il prend en charge des stratégies de rétention de snapshots flexibles ainsi que des tâches pré et post-snapshots pour des choses comme le dump de bases de données sur disque pour des sauvegardes de bases de données cohérentes.  Consultez DREBS et faites-nous savoir ce que vous en pensez.

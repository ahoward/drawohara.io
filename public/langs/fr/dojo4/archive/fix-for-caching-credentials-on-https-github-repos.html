GitHub a récemment basculé vers un schéma https par défaut pour le clonage des dépôts. En conséquence, vous pouvez être soudainement invité à entrer un ‘Nom d'utilisateur’ et un ‘Mot de passe’ lorsque vous poussez, alors qu'auparavant vous pouviez le faire sans saisir d'identifiants. La solution consiste à faire en sorte que git mette en cache les identifiants https, ce qui est facile, puisque git utilise curl en arrière-plan.

Dans votre répertoire personnel, créez un fichier appelé ‘.netrc’, par exemple

/Users/ahoward/.netrc

Mettez-y les contenus suivants

machine github.com
login VOTRE_NOM_D_UTILISATEUR_GITHUB
password VOTRE_MOT_DE_PASSE_GITHUB

Corrigé !

Es bastante fácil configurar apache en un solo sistema. Se vuelve un poco más interesante si tienes múltiples vhosts en la configuración. Se vuelve aún más divertido cuando tienes una configuración compleja que incluye proxy, reescritura de URL y alguna sustitución de texto.

Pero aun así, nada que un administrador de sistemas experimentado no pueda manejar con un poco de cuidado.

Hasta que, bueno, tienes que hacerlo de nuevo.

Aquí es donde entran en juego herramientas como Chef y Puppet. Ambas herramientas permiten declarar y definir cómo deben ser los sistemas y cómo deben estar configurados. Ambas son potentes y ambas se están utilizando activamente para configurar desde estaciones de trabajo de desarrolladores hasta grandes implementaciones de cientos de miles de máquinas.

Al igual que el software, aún necesitas construir y probar la configuración que estás creando. Tus opciones para hacerlo van desde lo realmente costoso (comprar uno o más servidores), a lo moderadamente costoso (ejecutar algunas instancias en EC2) a lo realmente, realmente barato. La opción realmente, realmente barata sería construir y probar tus recetas en máquinas virtuales que se ejecutan en tu sistema. Aquí es donde entra en juego Vagrant.

Vagrant es un wrapper alrededor de la plataforma de virtualización de escritorio VirtualBox de Oracle. La configuración de Vagrant gira en torno a unos pocos conceptos simples. Tienes un archivo de configuración llamado Vagrantfile en la raíz del proyecto. Ese archivo contiene alguna configuración básica como qué caja base usar para el proyecto y qué puertos exponer al sistema host. Y tan importante, puede describir cómo aprovisionar la máquina virtual para que se adapte a las necesidades de tu proyecto. Si bien Vagrant admite tanto Puppet como Chef, elegí usar Chef-Solo para aprovisionar este proyecto.

Comenzar con Vagrant es sencillo. El primer paso es simplemente instalar la gema vagrant. Al igual que cualquier otro proyecto de Ruby, elegí instalar vagrant y algunas otras gemas a través de Bundler. Aquí está el Gemfile resultante:

source 'https://rubygems.org'
gem 'vagrant'
gem 'librarian'

Ejecutar bundle install instaló las gemas necesarias en el directorio del proyecto. Un rápido rbenv rehash hizo que el ejecutable de vagrant estuviera disponible para mi shell.

El siguiente paso fue configurar la configuración más básica de Vagrant. Ejecuté vagrant init y luego edité el Vagrantfile resultante para que se viera así:

Vagrant::Config.run do |config|
  # Cada entorno virtual de Vagrant requiere una caja para construir.
  config.vm.box = "ubuntu-precise-64"
  
  # La url de donde se obtendrá la caja 'config.vm.box' si no
  # ya existe en el sistema del usuario.
  config.vm.box_url = "http://files.vagrantup.com/precise64.box"
end

Esto logra 2 cosas. Primero, declara el nombre de la caja que vamos a construir y dónde encontrar esa caja. Un rápido vagrant up inicialmente descargará la caja base, la importará en el proyecto y arrancará la VM. Tarda un rato en descargar la caja, pero se almacena en caché en tu sistema para que no necesites volver a descargarla. También puedes reutilizar la misma caja base si eliges hacerlo en otros proyectos. Llamar a vagrant halt detiene la VM y llamar a vagrant destroy, bueno, destruye la VM en sí.

Todavía no hemos hecho mucho para aprovisionar nuestra VM. El siguiente paso es decirle a Vagrant cómo hacerlo.

Librarian es otra gema agradable en la línea de Bundler y CocoaPods que te permite declarar qué recetas de Chef estás usando. En nuestro caso, una copia redactada del Cheffile del proyecto se ve así:

#!/usr/bin/env ruby
#^syntax detection
site 'http://community.opscode.com/api/v1'
cookbook 'apt'
cookbook 'apache2', '>= 1.0.0'
cookbook 'rbenv', :git => 'git://github.com/fnichol/chef-rbenv.git', :ref => 'v0.7.2'
cookbook 'ruby_build'
cookbook 'openssh', :git => 'git://github.com/fnichol/chef-openssh.git'
cookbook 'git'
cookbook 'build-essential'
cookbook 'vim'
cookbook 'user'
cookbook 'sudo'

El mayor beneficio es que no necesitas vender los libros de cocina en tus proyectos. Ejecutar librarian-chef install descargará los libros de cocina y los instalará en el directorio cookbooks. Esto también generó un Cheffile.lock para documentar qué versión de los libros de cocina se instalaron.

Luego estaba en un punto donde podía decirle a Vagrant que usara Chef-Solo para aprovisionar la caja actualizando el Vagrantfile:

Vagrant::Config.run do |config|
  # Cada entorno virtual de Vagrant requiere una caja para construir.
  config.vm.box = "ubuntu-precise-64"
  
  # La url de donde se obtendrá la caja 'config.vm.box' si no
  # ya existe en el sistema del usuario.
  config.vm.box_url = "http://files.vagrantup.com/precise64.box"
  
  # Habilitar el aprovisionamiento con chef solo, especificando una ruta de libros de cocina, rutas de roles
  # y ruta de data_bags (todas relativas a este Vagrantfile), y agregando
  # algunas recetas y/o roles.
  config.vm.provision :chef_solo do |chef|
    chef.cookbooks_path = ["cookbooks", "site-cookbooks"]
    chef

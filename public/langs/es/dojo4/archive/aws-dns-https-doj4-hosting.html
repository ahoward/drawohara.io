El hosting, DNS y HTTPS a menudo requieren la interacción de más de unos pocos proveedores. A menudo se vuelve inmanejable si no se gestiona de manera efectiva.

DNS requiere tanto la gestión de registros como el registro de dominios. Para ambos, Dojo4 ha utilizado DyDNS durante muchos años.

Dojo4 generalmente utiliza computadoras en la nube Amazon AWS EC2 para hosting y, por lo tanto, tiende a utilizar balanceadores de carga AWS ELB frente a esos hosts web. Desafortunadamente, Amazon no proporciona direcciones IP consistentes para las instancias de ELB y, por lo tanto, la única forma de crear registros de dominio de segundo nivel que apunten a instancias de ELB es utilizar Amazon AWS Route53 para la gestión de registros, que tiene un registro de alias especial para este propósito. Entonces, tendemos a utilizar Route53 para la gestión de registros DNS.

Debido a la forma en que se negocian los handshakes de TLS por linux/apache, estás limitado al número de certificados SSL que se pueden servir al número de IP en el host. Con EC2, esto es básicamente un solo certificado si no estás utilizando una VPC. Dojo4 a menudo utiliza ELB para proporcionar SSL como un medio para evitar esto.

DNS y HTTPS están estrechamente relacionados ya que los dominios y subdominios asegurados deben ser enumerados antes de la provisión del certificado. A menudo, la estrategia más conveniente y flexible es obtener un certificado comodín. Ten en cuenta, sin embargo, que no todos los emisores de certificados comodín incluyen el dominio base en el certificado (por lo que solo puedes usar el certificado para proteger *.algundominio.com, y no algundominio.com). El certificado comodín de $100/año de dnsimple.com emitido por Comodo e incluyendo el dominio base es una buena opción incluso con el costo de hosting DNS de dnsimple de $96 ($8/mes * 12 meses).

dnsimple tiene algunas otras características convenientes, como poder delegar la gestión de conjuntos de registros a otras cuentas de usuario de dnsimple. A medida que hemos trasladado la gestión de conjuntos de registros de DynDNS a Route53 y tenemos una cuenta en dnsimple para la provisión de SSL, procedimos a trasladar el registro de dominios de DynDNS a dnsimple para consolidar servicios.

Algunos comandos útiles de línea de comandos:

Encontrar la Autoridad de Inicio para un registro DNS

dig SOA dojo4.com

Ver el certificado que está sirviendo un host

openssl s_client -showcerts -connect dojo4.com:443 < /dev/null

Acceder a un servidor web por IP ignorando la verificación SSL

curl -k https://184.72.61.241 -H “Host: projects.dojo4.com”

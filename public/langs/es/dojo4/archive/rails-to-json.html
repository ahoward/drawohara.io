rails, incluyendo rails3, tiene este maravilloso fragmento de código relacionado con proporcionar un método "to_json" en los objetos integrados:

# Hack para cargar primero la gema json para poder sobrescribir su to_json.
    begin
      require 'json'
    rescue LoadError
    end
    # La gema JSON añade algunos módulos a las clases principales de Ruby que contienen la definición de :to_json, sobrescribiendo
    # su comportamiento predeterminado. Dicho esto, necesitamos definir el método básico to_json en todos ellos,
    # de lo contrario, siempre usarán la implementación de la gema to_json, que es incompatible en
    # varios casos (por ejemplo, la implementación JSON para Hash no funciona) con herencia
    # y, por lo tanto, las clases como ActiveSupport::OrderedHash no se pueden serializar en json.
    [Object, Array, FalseClass, Float, Hash, Integer, NilClass, String, TrueClass].each do |klass|
      klass.class_eval <<-RUBY, __FILE__, __LINE__
# Volcado del objeto en JSON (Notación de Objeto de JavaScript). Ver www.json.org para más información.
def to_json(options = nil)
ActiveSupport::JSON.encode(self, options)
end
RUBY
    end

lo que esto hace efectivamente es anular el método "to_json" que proporciona la gema json con uno que active_support considera mejor. El comentario sobre json estando roto está desactualizado y esto fue una solución provisional desde el principio. En mi caso, realmente me gusta este tipo de cosas en mi controlador /api:

  def to_json(objecto)
    if Rails.env.production?
      JSON.generate(objecto)
    else
      JSON.pretty_generate(objecto)
    end
  end

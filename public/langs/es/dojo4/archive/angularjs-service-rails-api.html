Recientemente tuve que sumergirme en AngularJS en un sitio de Rails que estábamos construyendo para un cliente. Además del contenido de la página, el cliente tiene ciertas páginas que muestran titulares que enlazan a fuentes externas pero que se extraen en nuestro stack de Rails a través de una tarea rake que se ejecuta periódicamente. El sitio debe ser almacenado en caché en gran medida a través del uso de una CDN, como CloudFront, pero para mantener los titulares actualizados, usamos AngularJS (porque el cliente estaba familiarizado con él, por lo que era más fácil de mantener para ellos) para consultar el stack de rails directamente por los últimos titulares.

Esto requirió hacer las cosas de manera inter-dominio ya que el dominio del sitio apuntaría a CloudFront. No había escrito una API de Rails que devolviera JSONP o un servicio de AngularJS antes, así que tuve que hacer alguna exploración y esto es lo que funcionó para mí. (También puedes leer el gist.)

Nota: Si no estás familiarizado con JSONP, esto no tendrá mucho sentido. Lee esto.

Servicio AngularJS

AngularJS tiene el concepto de servicios, que son simplemente singletons que se pueden usar donde sea que los necesites. En nuestro caso, puedes pensar en nuestro servicio de titulares como el cliente de la API de titulares que envuelve nuestras llamadas JSONP en una sintaxis más simple y centralizada. AngularJS proporciona algunos servicios por defecto, pero puedes escribir los tuyos propios. De hecho, podríamos haber usado el servicio $resource integrado en lugar de escribir el nuestro, sin embargo, tuve algunos problemas para que JSONP funcionara y quería entender mejor los componentes de nivel inferior de AngularJS, así que escribí el mío:

angular.module('app')
  .factory('HeadlineService', ['$http',
    function($http) {
      'use strict';

      var BASE_URL = "<%= [App.settings.api_base_url, '/services/headlines'].join %>",
          CALLBACK_STRING = "?callback=JSON_CALLBACK";

      return {
        getHeadlines: function(){
          return $http.jsonp(BASE_URL + CALLBACK_STRING)
        },
        getHeadlineForId: function(id){
          return $http.jsonp(BASE_URL + "/" + id + ".json" + CALLBACK_STRING)
        }

    }
  ]);

El servicio anterior usa el servicio $http integrado, que simplemente proporciona un envoltorio AJAX, para hacer solicitudes a nuestro backend de rails para obtener titulares. Algunas cosas a tener en cuenta:

  • Devuelve un objeto con 2 funciones: getHeadlines y getHeadlineForId. Podemos usar estas funciones como atajos para hablar con la API de Rails en nuestros controladores de AngularJS.
  • Usa JSONP. Nota que estamos usando $http.jsonp y estamos pasando una cadena de consulta ?callback=JSON_CALLBACK. AngularJS reemplaza la cadena JSON_CALLBACK con el nombre de una función de devolución de llamada que crea para ti.
  • Usa un dominio diferente. Bueno, eso es lo esperado ya que estamos haciendo JSONP, pero lo estamos logrando agregando una extensión .erb al archivo y usando lo que establezcamos para App.settings.api_base_url en la aplicación Rails como el dominio con el que AngularJS debe hablar. Recuerda que esto es para eludir la capa de caché de CDN para que nuestros titulares sean siempre los más recientes.

Controlador AngularJS

Con nuestro HeadlinesService construido, ahora podemos usar las 2 funciones que devuelve en nuestra aplicación AngularJS:

angular.module('app')
  .controller('HeadlinesCtrl', ['$scope', 'HeadlineService',
    function($scope, HeadlineService) {
      'use strict';
 
      HeadlineService.getHeadlines().success(function(data, status, headers, config){
        $scope.headlines = data;
        
        HeadlineService.getHeadlineForId(data[0].id).success(function(data){
          $scope.single_headline = data;
        }).error(function(error, status, headers, config){
          alert('algo salió mal al obtener un titular');
        });
      }).error(function(error, status, headers, config){
        alert('algo salió mal al obtener todos los titulares');
      });
    }
  ]);

Por ejemplo, simplemente obtengo el objeto completo de titulares para el primer titular devuelto en la devolución de llamada de éxito de getHeadlines.

El objeto $scope está expuesto a las vistas en AngularJS, por lo que podemos mostrar los titulares en la página teniendo una vista como:

<section class="headline_section" ng-controller="HeadlinesCtrl">

  <h3>Aquí hay algunos titulares...</h3>

  <div class="headlines">

    <ul>
      <li ng-repeat="headline in headlines">
        <a href="{{ headline.url }}" target="_none">{{ headline.title }}</a>
      </li>
    <ul>

  </div>

  <div class="headline">
    <a href="{{ single_headline.url }}" target="_none">{{ single_headline.title }}</a>
  </div>

</section>

Backend Rails

Ahora que tenemos a AngularJS llamando a nuestro backend de Rails, ¡necesitamos escribir el backend de Rails! El siguiente controlador es bastante básico de ruby on rails, sin embargo, para devolver JSONP en lugar del viejo JSON regular, usamos la opción callback en el método render. Eso le dice a Rails que estamos usando JSONP y que envuelva los datos JSON que le dimos en esa devolución de llamada y establezca la cabecera Content-Type correctamente para que todo se ajuste al estándar JSONP y nuestro cliente AngularJS pueda procesar los datos.

class Services::HeadlinesController < ApplicationController
 
  def index
    page   = (params[:page] || 1).to_i
    per    = (params[:per]  || 5).to_i

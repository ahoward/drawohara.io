Recientemente, github cambió a un esquema https como el predeterminado para clonar repositorios. Como efecto secundario, es posible que se le solicite un ‘Nombre de usuario’ y ‘Contraseña’ cuando empuje, donde anteriormente podía hacerlo sin escribir credenciales. La solución es hacer que git almacene en caché las credenciales https, lo cual es fácil, ya que git usa curl debajo.

En su directorio principal, cree un archivo llamado ‘.netrc’, por ejemplo

/Users/ahoward/.netrc

En él, ponga estos contenidos

machine github.com
login SU_NOMBRE_DE_USUARIO_DE_GITHUB
password SU_CONTRASEÑA_DE_GITHUB

¡Arreglado!

ref: https://gist.github.com/2885020

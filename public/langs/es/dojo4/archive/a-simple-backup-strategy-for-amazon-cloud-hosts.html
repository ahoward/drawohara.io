A lo largo de los años, Dojo4 ha elaborado una excelente pila de desarrollo y alojamiento de aplicaciones web. La nube de Amazon Web Services (AWS) forma la base de esta pila. La flexibilidad y conveniencia de los volúmenes de disco Elastic Block Store (EBS) es convincente y los utilizamos para volúmenes de sistema, así como para volúmenes de datos para desplegar aplicaciones y persistir bases de datos en nuestros ordenadores en la nube Elastic Compute Cloud (EC2). Hace algún tiempo, al final de una conversación sobre la arquitectura de alojamiento, Ara dijo de manera casual 'Oh, y los respaldos periódicos de los discos deberían ocurrir automáticamente cada vez que lanzamos una nueva instancia de EC2'. Así que, comencé a trabajar en un pequeño script llamado DREBS (Recuperación de Desastres para Elastic Block Store). DREBS puede integrarse en su Amazon Machine Image (AMI) o configurarse en sus mecanismos de aprovisionamiento en la nube Chef u otros. DREBS puede configurarse para tomar instantáneas periódicas de los volúmenes de EBS adjuntos a la instancia de EC2 en la que se está ejecutando. Admite estrategias flexibles de retención de instantáneas, así como tareas previas y posteriores a las instantáneas para cosas como volcar bases de datos a disco para respaldos consistentes de bases de datos. Echa un vistazo a DREBS y hánoslo saber qué piensas.

Один з наших улюблених клієнтів, Gnip (щойно придбаний Twitter), покладається на нас для всього їх маркетингового дизайну та веб-сайтів, включаючи gnip.com, blog.gnip.com та engineering.gnip.com.

Їхні два блоги є інстансами WordPress (залишки до-dojo4 спадщини), які ніколи не здавалися вартими зусиль для міграції з WordPress на той час.

Gnip.com однак є повністю статичним веб-сайтом, побудованим за допомогою Middleman та розміщеним на S3 та CloudFront. Ми зробили цю рекомендацію після побудови та хостингу повноцінного додатку Rails для їхнього сайту та помітивши, що їхній сайт був би ідеальним кандидатом для простого статичного сайту. Чинунки, які підказали нам це, такі самі, як і для будь-якого іншого статичного сайту: немає облікових записів користувачів, швидкість та час роботи мають найвищий пріоритет, і так мало оновлень, що адміністративний інтерфейс є надмірним (хоча з нашим статичним сайтом dojo4.com ми побудували веб-хів для прослуховування публікацій блогу).

У нас була одна цікава запитання - оскільки ми навчаємо маркетингову команду, як користуватися цим веб-сайтом, а не розробники, яке відчуття вони матимуть, коли порівняють адміністративно-інтенсивний бік WordPress (для їхніх блогів) з оновленням коду на сайті Middleman (для gnip.com) та розгортанням через командний рядок?

Виявляється, вони люблять налаштування Middleman і бажають, щоб ми перевели їхні блоги на Middleman замість WordPress роками тому.

WordPress має одну функцію - інтерфейс для створення та публікації контенту, з яким люди знайомі. Ігноруючи емоційний хаос WordPress на мить, вартість обслуговування та зусиль для сайту WordPress є вражаючими для компанії середнього або великого розміру.

Плагіни здаються хорошим шляхом додавання функціональності до вашого сайту, поки вам не потрібно модифікувати або оновити один, тому що його не оновлювали два роки. Коли Gnip був придбаний Twitter, команда безпеки Twitter хотіла, щоб блоги забезпечували, що весь вміст був через HTTPS. Це просте запитання, але коли у вас є 25 плагінів WordPress, які включають власні ресурси, багато з яких навіть не підтримують HTTPS, це справжній кошмар. Тоді як з нашим статичним підходом Middleman, ми виконуємо швидкий пошук та заміну в коді (і з самого початку використовуємо лише локальні ресурси або ті, що з надійних джерел, які підтримують HTTPS, як Google), розгортаємо та закінчуємо.

Коли ви витрачаєте час, щоб допомогти людям навчитися простим навичкам, таким як HTML, CSS та Javascript (які є майже обов'язковими навичками, якщо ви залучені до технологій взагалі, незалежно від вашої посади), недоліки WordPress стають набагато більшими, ніж прийнятно, коли існує така проста та потужна опція, як статичний веб-вміст та хостинг.

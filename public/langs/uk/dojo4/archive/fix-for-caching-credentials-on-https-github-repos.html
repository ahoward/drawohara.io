нещодавно github перейшов на схему https за замовчуванням для клонування репозиторіїв. як побічний ефект вас можуть раптово запитати про ‘Ім'я користувача’ і ‘Пароль’ коли ви виконуєте push, тоді як раніше ви могли це робити без введення облікових даних. рішення полягає у тому, щоб змусити git кешувати https облікові дані, що є просто, оскільки git використовує curl під капотом

у вашому домашньому каталозі створіть файл під назвою ‘.netrc’, наприклад

/Users/ahoward/.netrc

в ньому вставте такі вмісти

machine github.com
login ВАШЕ_КОРИСТУВАЧСЬКЕ_ІМЯ_GITHUB
password ВАШ_ПАРОЛЬ_GITHUB

виправлено!

ref: https://gist.github.com/2885020

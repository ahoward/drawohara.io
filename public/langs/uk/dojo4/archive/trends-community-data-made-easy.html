Щорічний звіт Фонду Спільноти Боулдер Тренди надає “загальний набір вимірюваних даних для поліпшення якості життя в спільноті.” Нещодавно Code for America співпрацював з Фондом, щоб зробити ці спільні дані більш доступними. Партнерство під назвою Code for Boulder запросило до участі dojo4. Після спільного сеансу генерації ідей, який визначив потенційні методи для представлення цінної інформації зі звіту Тренди, dojo4 співпрацював з партнерством для створення повністю адаптивного, автономного мікросайту, який Фонд Спільноти міг би легко підтримувати. Основний сайт Фонду Спільноти перероблявся Vermillion Design, тож dojo4 витягнув дизайнерські шаблони з їхніх настільних макетів.

Ми почали свою роботу з створення статичного “заповнювального” сайту, щоб визначити, які частини контенту потребують регулярного оновлення, а які більш постійні. Зрештою це зробило інтеграцію адміністративної панелі сайту значно швидшою.

Спочатку ми не були впевнені, як буде оновлюватися сайт: 1) за допомогою нашого улюбленого генератора статичних сайтів Middleman або 2) за допомогою нашої власної CMS на базі Rails. Але оскільки однією з умов проекту було те, що сайт має бути легко оновлюваним волонтерами Code for Boulder, ми вибрали варіант, який пріоризував зручність підтримки постійним, короткочасним штатом з різними технічними здібностями. Ми вибрали новий інструмент CMS під назвою DatoCMS, “повністю налаштовувана адміністративна area для статичних веб-сайтів”. Дійсно, Стефано Верна, засновник Dato, зв'язався з нами та пояснив, як легко інтегрувати DatoCMS і, що ще важливіше, як легко ним керувати нетехнічним користувачам. Це був ідеальний час, і всі тепер можуть ефективніше взаємодіяти з веб-ресурсом, який повідомляє про ключові показники якості життя в спільноті Боулдера.

Написано разом з Стівом Бейлі, який виконав більшу частину роботи над цим проектом.

rails, включаючи rails3, містить цей чудовий шматок коду, пов'язаний із наданням методу “to_json” побудованим об'єктам:

# Хак для завантаження json-гемки спочатку, щоб ми могли перевизначити її to_json.
    begin
      require 'json'
    rescue LoadError
    end
    # JSON-гемка додає декілька модулів до Ruby-коробочних класів, які містять :to_json визначення, перевизначаючи
    # їх поведінку за замовчуванням. Тому ми повинні визначити базовий метод to_json у всіх них,
    # інакше вони завжди використовуватимуть гемкову реалізацію to_json, яка є несумісною назад у
    # кількох випадках (наприклад, JSON-реалізація для Hash не працює) з наслідуванням
    # і, як наслідок, класи, такі як ActiveSupport::OrderedHash, не можуть бути серіалізовані в json.
    [Object, Array, FalseClass, Float, Hash, Integer, NilClass, String, TrueClass].each do |klass|
      klass.class_eval <<-RUBY, __FILE__, __LINE__
# Здійснює дамп об'єкта в JSON (JavaScript Object Notation). Див. www.json.org для більшої інформації.
def to_json(options = nil)
ActiveSupport::JSON.encode(self, options)
end
RUBY
    end

Це ефективно замінює метод “to_json”, який надає json-гемка, на той, який active_support вважає кращим. Коментар про те, що json зламаний, неактуальний, і це був хак з самого початку. У моєму випадку мені дуже подобається такий підхід у моєму /api контролері:

  def to_json(object)
    if Rails.env.production?
      JSON.generate(object)
    else
      JSON.pretty_generate(object)
    end
  end

оскільки це робить налагодження значно простішим. ось як я це зробив у rails3 додатку -

перший крок полягав у тому, щоб зробити, щоб rails3 підтримував концепцію “preinitializer”, яку використовували старі рейлси:

    ### file: config/preinitializer.rb
    dirname = File.dirname(File.expand_path(__FILE__))
    glob = File.join(dirname, 'preinitializers', '**/**.rb')
    preinitializers = Dir.glob(glob)
    preinitializers.each{|preinitializer| Kernel.load(preinitializer)}

далі я додав цей preinitializer:

### file: config/preinitializers/un_fuck_to_json.rb
    begin
      require 'rubygems'
    rescue LoadError
    end
    begin
      require 'json'
    rescue LoadError
    end
    [Object, Array, FalseClass, Float, Hash, Integer, NilClass, String, TrueClass].each do |klass|
      klass.class_eval do
        alias_method('__to_json__', 'to_json') if method_defined?('to_json')
      end
    end

і нарешті цей ініціалізатор:

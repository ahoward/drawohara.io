=begin
це досить поширено в Ruby, коли модулі надають функціональність через міксіни.
наприклад:
=end
  module Mixin
    def foo
      42
    end
  end
  class C
    include Mixin
  end
  p C.new.foo #=> 42
=begin
хоча це зручно, вміст міксінів, особливо модулів, може легко витікати
до цільового об'єкта ненавмисно.
тут модуль 'Util' потрапляє між Mixin та C:
=end
  reset!
  module Mixin
    module Util
      def Util.foo
        42
      end
    end
  end
  class C
    include Mixin
    p Util.foo #=> 42
    module Util
      def Util.bar
        42.0
      end
    end
    p Util.bar #=> 42.0
  end
=begin
це створює проблему для авторів міксінів: як організувати код
*і* надати модуль, який безпечно можна міксувати з будь-якою ціллю.
існують два основні підходи:
1) вирізати міксін окремо від простору імен верхнього рівня
=end
  reset!
  module M
    module Mixin
      def foo
        42
      end
    end
    module Util
    end
  end
  class C
    include M::Mixin
  end
  p C.new.foo #=> 42
  p C.const_defined?(:Util) #=> false
=begin
або
2) використовувати const_missing, щоб дозволити прості константи-псевдоніми в приватному просторі імен
=end
  reset!
  module M
    module Namespace
      module Util
        def foo
          42
        end
      end
    end
    def Mixin.const_missing(const)
      begin
        Namespace.const_get(const)
      rescue Object
        raise
      end
    end
    def foo
      Util.foo
    end
  end
  class C
    include M

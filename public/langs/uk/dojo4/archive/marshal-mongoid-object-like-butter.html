Зберігання ваших моделей у кеш або їх серіалізація з інших причин не повинні бути болючими. На жаль, це так з більшістю ORM, оскільки вони мають жахливу стратегію маршалізації за замовчуванням. Однак це легко виправити: ключ полягає в розумінні, що кожен ORM вже знає, як взяти хеш інформації з бази даних і створити повноцінний екземпляр. Знаючи це, ми можемо легко зробити будь-яку модель серіалізованою як масло.


#! /usr/bin/env ruby
# деякі моделі мають дивні речі, які не виживуть під час маршалізації
#
  class Model
    include Mongoid::Document
    def initialize(*args, &block)
      super
    ensure
      @fail = Class.new, open(__FILE__)
    end
  end
# тому це не вдасться
#
  begin
    p Marshal.load(Marshal.dump(Model.create))
  rescue Object => o
    warn "#{ o.message } (#{ o.class })"
  end
# але моделі mongoid просто потребують хеш інформації від драйвера mongo
# щоб повністю оживити себе... тому це все, що нам потрібно зберегти під час
# маршалізації. це робить завантаження з маршалізованих даних *як* завантаження з
# бази даних.
#
# якщо запитати мене, це має бути поведінка за замовчуванням!
#
# хм - я в ядрі mongoid... @durran, що ти думаєш?
#
# до речі - це чудово працює з active_record також...
#
  class Model
    def _dump(*args, &block)
      Marshal.dump(raw_attributes, *args, &block)
    end
    def Model._load(string, *args, &block)
      raw_attributes = Marshal.load(string, *args, &block)
      instantiate(raw_attributes)
    end
  end
# тому тепер це просто працює (TM)
#
  p Marshal.load(Marshal.dump(Model.create))
BEGIN {
  require 'rubygems'
  require 'mongoid'
  Mongoid.configure{|config| config.connect_to('mongoid-marshal')}
}
__END__

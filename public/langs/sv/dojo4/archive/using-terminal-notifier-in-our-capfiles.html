Vi använder capistrano för våra distributioner och nyligen stötte jag på den underbara terminal-notifier gemmen som använder användarnotiser i OS X 10.8 och senare för att visa ett fint meddelande från kommandoraden. Varje gång jag distribuerar kollar jag alltid manuellt om distributionen är klar, och detta är ett bra sätt att undvika det. Ett problem som jag stötte på när jag lade till det i ett av våra projekt är att några i vårt team använder en version av Mac OS X före 10.8 (Jag vet!), så kommandot terminal-notifier skulle misslyckas och deras distribution skulle ha fel i sig. Så här är koden jag använder nu för att säkerställa att fel inte visas om du inte kan använda de fina användarnotiserna i nyare OS X:

desc 'använd terminal-notifier om i > osx 10.8 för att använda en användarvarning'
task :terminal do
 begin
  require 'terminal-notifier'
        url         = fetch(:url)
        application = fetch(:application)
        system "terminal-notifier -title #{ application } -message 'Distribuerat slutfört framgångsrikt till #{ url }.' -sound default"
 rescue LoadError
        warn "Du är troligen inte på OS X 10.8, så terminal-notifier används inte. Om du är, använd gem install terminal-notifier för att använda några fina användarnotiser."
 end
end

Självklart inkluderar vi också gem terminal-notifier i vår Gemfile. Se min SO fråga om att inkludera denna gem i din gemfile för en förklaring till varför jag använder den enkla gem-inkluderingssatsen.

När du arbetar med någon rails-applikation som inte är en vanlig hello-world-app behöver du ofta små ajax-hjälpare för att kommunicera med servern - för att kunna utföra klient-sidans logik. Jag talar inte om ett API i sig, utan ad-hoc-verktyg som detta:

markdown.change(function(){
  generate_html_on_the_server( markdown.val(), function(html){
  
    preview.html(html);
  
  });
});

eller

reservation.change(function(){
  check_availibity_on_the_server( reservation.val(), function(data){
  
    if(!data['available']){
     
     alert('den tiden är inte tillgänglig - välj en annan!');
     
    };
  })
});

etc.

Dessa typer av funktioner är inte en del av ett server-till-server offentligt API, de är bara samarbets-js/backend-ändpunkter som behövs för att få vyn att fungera.

De flesta rails-applikationer kommer att samla många av dessa och frågan uppstår:

“Var ska du placera dem?”

I de flesta team kommer du ha tre eller fyra utvecklare som namnger och organiserar dessa funktioner på olika sätt, vilket säkerställer att kodbasen blir en cowboy-spaghetti-röra på kort tid.

@dojo4 har vi abstraherat detta med en liten rpc-designmall: först har vi en liten kontroller-dsl som ingår i vår ApplicationController som tillåter deklarativa definitioner av 'rpc' js-hjälpare på en per-kontroller-basis.

class ApplicationController < ActionController::Base
  include(RPC)
end

Kolla in implementeringen här: https://gist.github.com/ahoward/7320900

På rent engelska definierar denna dsl bara en enda åtgärd på kontrollanten som multiplexar vilken metod som ska användas baserat på parametrar, och ett enkelt sätt att definiera dem. Den förväntar sig att alla rpc-åtgärder returnerar en hash och ger tillbaka json.

Denna implementering går ut på

  def rpc
    which = params['method']
    
    action = @rpc[which]
    
    result = action.call(params)
    
    render :json => result.to_json
  end

Dess användning bör vara uppenbar från koden

class GeoLocationController < ApplicationController
  rpc(:geo_location) do |params|
    geo_location = GeoLocation.geo_locate( params['address'] )
    geo_location.attributes
  end
  
  rpc(:lat_lng) do |params|
   geo_location = GeoLocation.geo_locate( params['address'] )
   { 'lat' => geo_location.lat, 'lng' => geo_location.lng }
  end
end

Användning från js kräver två saker: en väg och en liten js-klass. Först vägen:

  match ':controller#rpc', :action => 'rpc'
  

Nästa, js, även här -> https://gist.github.com/ahoward/7320900

Om du läser igenom det ser du att användningen av backend rpc-åtgärder från js är lika enkelt som


  var rpc = new RPC(url);

  rpc.call('geo_location', params, callback);

Normalt, i botten av en vy, instansierar vi bara ett rpc-objekt för sidan med en relativ url

Den årliga Community Foundation Boulder Countys Treender-rapport ger “en gemensam uppsättning mätbara data för att förbättra livskvaliteten i gemenskapen.” Och Code for America samarbetade nyligen med stiftelsen för att göra dessa gemenskapsdata mer tillgängliga. Samarbetet, kallat Code for Boulder, bjöd in dojo4 att delta. Efter en gemenskapsidégenereringssession för att upptäcka de potentiella metoderna som kunde användas för att exponera det värdefulla datainnehållet i Treender-rapporten, arbetade dojo4 med samarbetet för att implementera en fullt responsiv, fristående mikrosajt som Community Foundation enkelt kunde underhålla. Huvudwebbplatsen för Community Foundation omdesignades av Vermillion Design, så dojo4 destillerade designmönstren från deras skrivbordsmockups.

Vi började vårt arbete genom att bygga en statisk “placeholder”-sajt för att avgöra vilka innehållsdelar som behövde uppdateras regelbundet och vilka delar var mer permanenta. Till slut gjorde detta jobbet med att integrera sajtens admin-backend mycket snabbare.

I början var vi osäkra på hur sajten skulle uppdateras: 1) med vår favoritstatiska sajtgenerator Middleman eller 2) med vårt hemgjorda rails-baserade CMS. Men eftersom det var en förutsättning för projektet att sajten skulle kunna uppdateras enkelt av Code for Boulder-volontärer, valde vi en lösning som prioriterade underhållbarhet av en roterande, kortvarig personal med varierande tekniska förmågor. Vi valde ett nytt CMS-verktyg som heter DatoCMS, “ett fullt anpassningsbart administrativt område för statiska webbplatser”. Faktiskt nådde Stefano Verna, Dato’s grundare, ut till oss och förklarade hur enkelt det var att integrera DatoCMS, och viktigare, hur enkelt det var för icke-tekniska användare att hantera det. Det var perfekt timing och alla kan mer effektivt använda webbresursen som rapporterar de viktiga indikatorerna för livskvalitet i Boulder-gemenskapen.

Skrivet med Steve Bailey, som gjorde det mesta av arbetet med det här projektet.

Med Rails + Capistrano-distributioner är denna inställning

ssh_options[:forward_agent] = true

förhandsvald och det förnuftiga standardvalet. Den använder alla dina SSH-nycklars goda egenskaper för alla kommandon, vilket gör att de flesta fjärråtgärder 'Bara Fungerar ™'

Dock kan det ibland vara förvirrande, till exempel när du har två nycklar laddade i ssh-agent som mappar till olika GitHub- eller Bitbucket-konton. Vilket konto du får kommer att vara ett roligt litet spel att ta reda på ;-/

Följande mycket enkla strategi låter dig checka in per projekt SSH-nycklar på ett mycket naturligt sätt, och sedan få Capistrano att respektera dessa nycklar för alla 'cap XXX'-åtgärder.

Först vill du skapa en .ssh/-katalog i ditt projekt, och sedan placera alla relevanta nycklar där. Du kan ha många par, bara se till att de är namngivna enligt den normala *.id_rsa-konventionen.

a:~/git/dojo4/rails_app_m $ ls .ssh
id_rsa.dojo4            id_rsa.dojo4.pub

Sedan vill du ha något liknande detta i din Capfile, som upptäcker dessa lokala nycklar, om de finns, och använder dem via en temporär ssh-agent - annars använder den dina globala ssh-agent-laddade nycklar.

# om det här projektet har en .ssh/-katalog, använd dessa nycklar för distribution.
# annars använd de globala. Lokala nycklar används genom att starta
# en temporär ssh-agent och lägga till dessa lokala nycklar i den.
#
# troligen vill du .gitignore .ssh/ !
#
  dot_ssh = File.expand_path("#{ rails_root }/.ssh", __FILE__)
  globs = [
    File.join(dot_ssh, "*id_rsa"),
    File.join(dot_ssh, "id_rsa*"),
  ]
  ssh_keys = [
    ENV['SSH_KEYS'], globs.map{|glob| Dir.glob(glob)}
  ].join(' ').scan(/[^\s]+/).delete_if{|ssh_key| ssh_key =~ /\.pub$/}
  unless ssh_keys.empty?
    require 'session'
    sh = Session::Sh.new
    at_exit{ sh.close }
    sh.execute 'eval `ssh-agent -s -t 3600`'
    ssh_auth_sock = sh.execute('echo $SSH_AUTH_SOCK').first.to_s.strip
    ssh_agent_pid = sh.execute('echo $SSH_AGENT_PID').first.to_s.strip
    ENV['SSH_AUTH_SOCK'] = ssh_auth_sock
    ENV['SSH_AGENT_PID'] = ssh_agent_pid
    ssh_keys.each do |ssh_key|
      if system "SSH_AUTH_SOCK=#{ ssh_auth_sock } ssh-add #{ ssh_key.inspect } >/dev/null 2>&1"
        Say.say("Använder ssh-nyckel #{ ssh_key.inspect }", :color => :green)
      else
        Say.say("Kunde inte lägga till ssh-nyckel #{ ssh_key.inspect }", :color => :red)
      end
    end
  end
# det enda förnuftiga är att alltid vidarebefordra din lokala eller globala agent
#
  ssh_options[:forward_agent] = true

Du behöver session-gemmet i din Gemfile, naturligtvis

# fil: Gemfile
gem 'session'

Och glöm inte att .gitignore din nya .ssh-konfiguration

# fil: .gitignore
.ssh/*

Det finns egentligen inte så mycket mer till det: nu kan du konfigurera per projekt SSH-nycklar, och veta att alla cap deploy xxx-kommandon kommer att använda dessa projektlokala nycklar, och fortfarande kunna förorena din normala ssh-agent så mycket du vill.

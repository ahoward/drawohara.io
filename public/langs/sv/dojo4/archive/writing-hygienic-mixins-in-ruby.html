=begin
det är ganska vanligt i ruby att moduler tillhandahåller funktionalitet via mixins.
exempelvis:
=end
  module Mixin
    def foo
      42
    end
  end
  class C
    include Mixin
  end
  p C.new.foo #=> 42
=begin
även om det är praktiskt, kan innehållet i mixins, särskilt moduler, lätt läcka
in i målet oavsiktligt.
här läggs 'Util'-modulen mellan Mixin och C:
=end
  återställ!
  module Mixin
    module Util
      def Util.foo
        42
      end
    end
  end
  class C
    include Mixin
    p Util.foo #=> 42
    module Util
      def Util.bar
        42.0
      end
    end
    p Util.bar #=> 42.0
  end
=begin
detta skapar en utmaning för författarna av mixins: hur man ska hålla koden
organiserad *och* tillhandahålla en modul som är säker att mixa in i vilket mål som helst.
det finns två huvudsakliga metoder:
1) skär ut mixinen separat från det översta namnutrymmet
=end
  återställ!
  module M
    module Mixin
      def foo
        42
      end
    end
    module Util
    end
  end
  class C
    include M::Mixin
  end
  p C.new.foo #=> 42
  p C.const_defined?(:Util) #=> false
=begin
eller
2) använd const_missing för att tillåta enkla konstantalias i ett privat namnutrymme
=end
  återställ!
  module M
    module Namespace
      module Util
        def foo
          42
        end
      end
    end
    def Mixin.const_missing(const)
      begin
        Namespace.const_get(const)
      rescue Object
        raise
      end
    end
    def foo
      Util.foo
    end
  end
  class C
    include M::Mixin
  end
  p C.new.foo #=> 42
  p C.const_defined?(:Util) #=> false
=begin
så, där har du det: tänk noga på att använda vanliga namn
i dina mixins eftersom de absolut spyr ut alla sina interna delar i
mixee.
=end
BEGIN {
  def återställ!
    self.class.send :remove_const, :Mixin rescue false
    self.class.send :remove_const, :C rescue false

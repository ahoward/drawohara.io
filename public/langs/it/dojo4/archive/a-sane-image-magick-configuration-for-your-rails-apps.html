Prima una piccola configurazione/contesto: spesso distribuiamo una coppia di file RAILS_ROOT/env.yml e RAILS_ROOT/env.rb che l'applicazione rails assorbirà all'avvio. Non contengono dipendenze rails e possono essere utilizzati negli script di background per comprendere rapidamente la configurazione rails distribuita senza dover caricare l'intera applicazione rails.

Il primo file è un semplice file yaml di coppie di ambiente k=v.

# file: RAILS_ROOT/config/env.yml
# RAILS_ENV : production
# RAILS_STAGE : stage
# APP_RUBY : /usr/local/rbenv/versions/1.9.3-p0/bin/ruby
APP_RUBY_VERSION : 1.9.3
# APP_PORT : 4201
# APP_USER : dojo4
# APP_GROUP : dojo4

Il secondo è effettivamente il file che viene richiesto. Carica questo file e fa un tentativo dinamico di impostare ulteriori variabili di ambiente

# file: RAILS_ROOT/env.rb
#
# prima, integriamo eventuali impostazioni di ambiente trovate in
#
# RAILS_ROOT/env.yml
#
# facendo attenzione a non sovrapporre eventuali variabili ENV impostate manualmente. Il file env.yml viene
# normalmente creato durante una distribuzione cap.
#
  require 'erb'
  require 'yaml'
  require 'rbconfig'
  
  env_yml = File.expand_path('../env.yml', __FILE__)
  
  if test(?s, env_yml)
    buf = IO.read(env_yml)
    expanded = ERB.new(buf).result(binding)
    config = YAML.load(expanded)
    config.each{|key, val| ENV[key.to_s] ||= val.to_s} if config.is_a?(Hash)
  end

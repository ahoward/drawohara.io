Negli anni, Dojo4 ha creato una stack di sviluppo e hosting di applicazioni web davvero eccellente. Il cloud di Amazon Web Services (AWS) forma la base di questa stack. La flessibilità e la comodità dei volumi di dischi Elastic Block Store (EBS) è convincente e li utilizziamo per i volumi di sistema così come per i volumi di dati per distribuire app e per persistere database sui nostri computer cloud Elastic Compute Cloud (EC2). Qualche tempo fa, alla fine di una conversazione sull'architettura di hosting, Ara ha detto in modo casuale 'Oh, e i backup periodici dei dischi dovrebbero avvenire automaticamente ogni volta che lanciamo una nuova istanza EC2'. Quindi, ho iniziato a lavorare su uno script chiamato DREBS (Disaster Recovery for Elastic Block Store). DREBS può essere integrato nella tua Amazon Machine Image (AMI) o configurato nel tuo Chef o in altri meccanismi di provisioning cloud. DREBS può essere configurato per eseguire snapshot periodici dei volumi EBS collegati all'istanza EC2 su cui è in esecuzione. Supporta strategie di conservazione degli snapshot flessibili e pre e post task di snapshot per cose come il dump dei database su disco per backup coerenti del database. Dai un'occhiata a DREBS e facci sapere cosa ne pensi.

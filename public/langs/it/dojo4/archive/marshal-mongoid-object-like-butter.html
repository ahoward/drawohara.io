Mettere i tuoi modelli in cache o serializzarli per altri motivi non dovrebbe essere doloroso. Purtroppo lo è con la maggior parte degli ORM, poiché hanno una strategia di marshalling scarsa di default. Tuttavia, è facile da risolvere: la chiave sta nel capire che ogni ORM sa già come prendere un hash di informazioni dal database e istanziare un'istanza completa. Sapendo questo, possiamo facilmente far serializzare qualsiasi modello come burro.


#! /usr/bin/env ruby
# alcuni modelli hanno roba strana che non sopravvive a un giro di marshalling
#
  class Model
    include Mongoid::Document
    def initialize(*args, &block)
      super
    ensure
      @fail = Class.new, open(__FILE__)
    end
  end
# quindi fallirà
#
  begin
    p Marshal.load(Marshal.dump(Model.create))
  rescue Object => o
    warn "#{ o.message } (#{ o.class })"
  end
# ma i modelli mongoid hanno solo bisogno di un hash di informazioni dal driver mongo
# per vivificarsi completamente... ergo è tutto ciò che dobbiamo persistere quando
# marshallato. ciò rende il caricamento dai dati marshallati *proprio come* il caricamento dal
# db.
#
# se mi chiedi, questo dovrebbe essere il comportamento predefinito!
#
# hrm - sono nel core di mongoid... @durran, cosa ne pensi?
#
# comunque - questo funziona benissimo anche con active_record...
#
  class Model
    def _dump(*args, &block)
      Marshal.dump(raw_attributes, *args, &block)
    end
    def Model._load(string, *args, &block)
      raw_attributes = Marshal.load(string, *args, &block)
      instantiate(raw_attributes)
    end
  end
# quindi ora funziona semplicemente (TM)
#
  p Marshal.load(Marshal.dump(Model.create))
BEGIN {
  require 'rubygems'
  require 'mongoid'
  Mongoid.configure{|config| config.connect_to('mongoid-marshal')}
}
__END__

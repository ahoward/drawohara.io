Con qualsiasi progetto, sto eseguendo diversi programmi contemporaneamente durante il processo di sviluppo. Questo di solito comporta diverse schede nell'app Terminale: una per il mio server locale, una per il mio database, una per la mia applicazione client e una per git. (Al momento sto usando un editor GUI (ST2), ma potrei presto tornare a vim...) Questa configurazione a più schede si ripete per ogni progetto su cui sto lavorando e di solito passo da un progetto all'altro diverse volte al giorno.

Con questa configurazione a più schede, quando devo cambiare il progetto su cui sto lavorando, devo aprire una nuova finestra del Terminale con le sue schede per quel progetto e gestire un numero eccessivo di finestre aperte (non un'opzione per un minimalista come me), o chiudere tutte le schede che ho aperto, per poi riaprirle tutte quando torno a quel progetto.

Beh, come la maggior parte dei giorni, ho imparato una soluzione intelligente a questo problema dal saggio e onnipotente leader di dojo4, Ara. Usa il programma GNU Screen. All'interno di una singola scheda del Terminale, Screen ti consente di aprire più "finestre" (alias schede) all'interno della stessa scheda del Terminale. Ecco come usarlo:

# creare e riattaccare a una sessione
screen -S [nome] = crea uno screen con nome
screen -D -R [nome] = riattacca a uno screen con nome e scollega altre persone se necessario
screen -list = elenca gli screen

# (uscir e da tutte le finestre in una sessione screen terminerà quella sessione. Tuttavia una sessione screen è un processo in esecuzione, quindi puoi terminarla manualmente.)

# all'interno di una sessione screen:
ctrl-A + ? = aiuto
ctrl-A + " = elenca le finestre
ctrl-A + c = crea una nuova finestra
ctrl-A + n = finestra successiva
ctrl-A + p = finestra precedente
ctrl-A + A = rinomina una finestra

# alias che usiamo
alias sl='screen -list '
alias sdr='screen -d -r '
alias s='screen -D -R '
alias ss='screen -S'

sdr [nome] = riattacca a uno screen con nome e scollega altre persone se necessario
ss [nome] = crea uno screen con nome
sl = elenca gli screen

Un'altra funzione davvero interessante di screen è che quando ti disconnetti da una sessione screen, tutte le tue finestre rimangono in esecuzione e quando ti ricolleghi alla sessione screen, avrai tutte le tue finestre nuovamente a disposizione.

Quando ne abbiamo discusso in ufficio, Ryan Cook ha creato un piccolo aiutante Ruby per creare sessioni screen denominate come la tua directory corrente.

Quindi il mio flusso di lavoro è passato dall'avere più schede aperte nel Terminale a una singola scheda nel Terminale dove mi ricollego a una sessione screen per ogni progetto su cui sto lavorando. Questo mi permette di cambiare progetto estremamente velocemente e di smettere di perdere tempo a reimpostare e ripristinare tutte le schede del Terminale che uso per un progetto.

Ora devo solo convertire più cose all'uso del terminale (come il mio editor) per sfruttare davvero al massimo screen...

Теги HTML5 <audio> досить прості. Ось приклад:

<audio id="player"></audio>

Цей фрагмент JavaScript:

player = document.getElementById('player');
player.src = 'some-audio-file-url';
player.play()

запустить відтворення будь-якого джерела, на яке посилається src. (Порада: якщо ви використовуєте jQuery, вам потрібно отримати фактичний HTML-елемент за допомогою player = $('#player')[0] або player = $('#player').get(0))

Однак, якщо користувач працює на мобільному пристрої під керуванням iOS або Android, вищенаведене не відтворить нічого. Жодна з цих операційних систем не дозволяє веб-аудіо (або відео) починати трансляцію без взаємодії користувача. Логіка полягає в тому, що трансляція даних через мобільну мережу може коштувати грошей і не повинна відбуватися без того, щоб користувач фактично натиснув кнопку.

Все добре, але що, якщо ви хочете відтворити звук у відповідь на подію, наприклад, просигналізувати браузер один раз на хвилину:

player = document.getElementById('player');
player.src = 'beep-url';
beeper = setInterval(function() { player.play(); } ,1000 * 60);

Це працюватиме без проблем у настільних браузерах, але на мобільних пристроях виклик play() нічого не зробить.

Щастя в тому, що є три речі, які роблять обхід простим:

  1. Користувачеві потрібно натиснути play лише один раз для кожного аудіоелемента.
  2. «Натиснути play» також може означати виклик play() за допомогою події klik.
  3. Відтворення аудіоелемента без джерела є законною операцією без дії.

Коротше кажучи, все, що нам потрібно, це щоб користувач натиснув кнопку, яка відтворює порожній аудіоелемент, щоб отримати контроль над ним:

<audio id="player"></audio>
<button id="ok">OK to beep</button>
// Припускаємо, що використовується jQuery
$("#ok").click(function() {
 player = document.getElementById('player');
 $( this ).slideUp();
 player.play(); // Відтворити порожній елемент, щоб отримати контроль
 player.src = 'beep-url'; // Встановити реальне аудіоджерело
 beeper = setInterval(function() { player.play(); } ,1000 * 60);
});

Клік може бути будь-чим, більш прихованим варіантом може бути прив'язка до кнопки закриття інструментального модального вікна на початку гри.

Ця техніка, можливо, дещо розмиває межу між духом і буквою закону, але вона виконує свою роботу та дозволяє створити однаковий досвід як для настільних, так і для мобільних користувачів.

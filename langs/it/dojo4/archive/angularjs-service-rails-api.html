Recentemente ho dovuto immergermi in AngularJS su un sito Rails che stavamo costruendo per un cliente. Oltre ai contenuti della pagina, il cliente ha alcune pagine che mostrano titoli che collegano a fonti esterne ma che vengono inseriti nel nostro stack Rails tramite un task rake eseguito periodicamente. Il sito deve essere memorizzato nella cache in modo pesante tramite l'uso di una CDN, come CloudFront, ma per mantenere i titoli aggiornati, abbiamo utilizzato AngularJS (perché il cliente era familiare con esso quindi una manutenzione più semplice per loro) per interrogare lo stack rails direttamente per i titoli più recenti.

Questo ha richiesto di fare le cose in modo cross-domain poiché il dominio del sito sarebbe stato puntato su CloudFront. Non avevo mai scritto un'API Rails che restituisse JSONP o un servizio AngularJS prima, quindi ho dovuto fare un po' di esplorazione e questo è ciò che ha funzionato per me. (Puoi anche leggere il gist.)

Nota: se non sei familiare con JSONP, questo non avrà molto senso. Leggi questo.

AngularJS Service

AngularJS ha il concetto di servizi, che sono semplicemente singleton che possono essere utilizzati ovunque ne hai bisogno. Nel nostro caso, puoi pensare al nostro servizio di titoli come al client API dei titoli che avvolge le nostre chiamate JSONP in una sintassi più semplice e centralizzata. AngularJS fornisce alcuni servizi di default, ma puoi scrivere i tuoi. In realtà, avremmo potuto utilizzare il servizio $resource integrato invece di scrivere il nostro, tuttavia ho avuto qualche problema a far funzionare JSONP e volevo comprendere meglio i componenti di livello inferiore di AngularJS, quindi ho scritto il mio:

angular.module('app')
  .factory('HeadlineService', ['$http',
    function($http) {
      'use strict';

      var BASE_URL = "<%= [App.settings.api_base_url, '/services/headlines'].join %>",
          CALLBACK_STRING = "?callback=JSON_CALLBACK";

      return {
        getHeadlines: function(){
          return $http.jsonp(BASE_URL + CALLBACK_STRING)
        },
        getHeadlineForId: function(id){
          return $http.jsonp(BASE_URL + "/" + id + ".json" + CALLBACK_STRING)
        }

    }
  ]);

Il servizio sopra utilizza il servizio $http integrato, che fornisce semplicemente un wrapper AJAX, per effettuare richieste al nostro backend rails per ottenere i titoli. Alcune cose da notare:

  • Restituisce un oggetto con 2 funzioni: getHeadlines e getHeadlineForId. Possiamo quindi utilizzare queste funzioni come scorciatoie per parlare con l'API Rails nei nostri controller AngularJS.
  • Utilizza JSONP. Nota che stiamo utilizzando $http.jsonp e stiamo passando una stringa di query ?callback=JSON_CALLBACK. AngularJS sostituisce la stringa JSON_CALLBACK con il nome di una funzione di callback che crea per te.
  • Utilizza un dominio diverso. Beh, questo è previsto poiché stiamo utilizzando JSONP, ma lo otteniamo aggiungendo un'estensione .erb al file e utilizzando tutto ciò che impostiamo per App.settings.api_base_url nell'app Rails come dominio con cui AngularJS dovrebbe parlare. Ricorda che questo serve per aggirare il livello di caching della CDN in modo che i nostri titoli siano sempre i più recenti.

AngularJS Controller

Con il nostro HeadlinesService costruito, possiamo ora utilizzare le 2 funzioni che restituisce nella nostra app AngularJS:

angular.module('app')
  .controller('HeadlinesCtrl', ['$scope', 'HeadlineService',
    function($scope, HeadlineService) {
      'use strict';
 
      HeadlineService.getHeadlines().success(function(data, status, headers, config){
        $scope.headlines = data;
        
        HeadlineService.getHeadlineForId(data[0].id).success(function(data){
          $scope.single_headline = data;
        }).error(function(error, status, headers, config){
          alert('si è verificato un errore durante il recupero di un titolo');
        });
      }).error(function(error, status, headers, config){
        alert('si è verificato un errore durante il recupero di tutti i titoli');
      });
  }
  ]);

A scopo di esempio, ottengo semplicemente l'oggetto completo dei titoli per il primo titolo restituito nel callback di successo getHeadlines.

L'oggetto $scope è esposto alle viste in AngularJS, quindi possiamo visualizzare i titoli sulla pagina avendo una vista come:

<section class="headline_section" ng-controller="HeadlinesCtrl">

  <h3>Ecco alcuni titoli...</h3>

  <div class="headlines">

    <ul>
      <li ng-repeat="headline in headlines">
        <a href="{{ headline.url }}" target="_none">{{ headline.title }}</a

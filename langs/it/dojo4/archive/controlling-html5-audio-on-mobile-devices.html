I tag HTML5 <audio> sono abbastanza semplici. Data:

<audio id="player"></audio>

questo pezzo di Javascript:

player = document.getElementById('player');
player.src = 'some-audio-file-url';
player.play()

farà partire la riproduzione di qualsiasi cosa a cui punta la src. (Consiglio: Se stai usando jQuery, devi ottenere l'elemento HTML effettivo con player = $('#player')[0] o player = $('#player').get(0))

Tuttavia, se l'utente è su un dispositivo mobile iOS o Android, quanto sopra non riprodurrà effettivamente nulla. Nessuno di questi sistemi operativi consente l'avvio dello streaming audio (o video) web senza l'interazione dell'utente. La motivazione è che lo streaming di dati sulla rete cellulare può costare denaro e non dovrebbe avvenire senza che l'utente prema effettivamente un pulsante.

Tutto bene, ma cosa succede se si desidera riprodurre un suono in risposta a un evento, ad esempio far suonare il browser una volta al minuto:

player = document.getElementById('player');
player.src = 'beep-url';
beeper = setInterval(function() { player.play(); } ,1000 * 60);

Questo funzionerà correttamente nei browser desktop, ma su dispositivi mobili la chiamata a play() non farà nulla.

Fortunatamente, ci sono tre cose che rendono la soluzione semplice:

  1. L'utente deve premere play solo una volta per elemento audio.
  2. “Premere play” può anche significare attivare play() tramite un evento di clic.
  3. Riprodurre un elemento audio senza sorgente è un'operazione legale senza effetto.

In breve, tutto ciò di cui abbiamo bisogno è che l'utente clicchi su un pulsante che riproduce un elemento audio vuoto per ottenere il controllo:

<audio id="player"></audio>
<button id="ok">OK per suonare</button>
// Presuppone jQuery
$("#ok").click(function() {
 player = document.getElementById('player');
 $( this ).slideUp();
 player.play(); // Riproduci l'elemento vuoto per ottenere il controllo
 player.src = 'beep-url'; // Imposta la vera sorgente audio
 beeper = setInterval(function() { player.play(); } ,1000 * 60);
});

Il clic può essere qualsiasi cosa, una versione più sottile sarebbe collegarlo al pulsante chiudi di una modale di istruzioni all'inizio di un gioco.

Questa tecnica probabilmente sfuma un po' il confine tra lo spirito e la lettera della legge, ma svolge il lavoro e ti consente di creare un'esperienza coerente per gli utenti sia desktop che mobile.

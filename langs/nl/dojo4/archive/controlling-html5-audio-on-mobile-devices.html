HTML5 <audio> tags is vrij eenvoudig. Gegeven:

<audio id="player"></audio>

dit stukje Javascript:

player = document.getElementById('player');
player.src = 'some-audio-file-url';
player.play()

zal beginnen met het afspelen van wat de src aangeeft. (Pro-tip: Als je jQuery gebruikt, heb je het echte HTML-element nodig met player = $('#player')[0] of player = $('#player').get(0))

Echter, als de gebruiker zich op een iOS- of Android-mobiel apparaat bevindt, zal het bovenstaande niet echt iets afspelen. Geen van die besturingssystemen staat web-audio (of video) toe om te streamen zonder gebruikersinteractie. Het rationeel is dat het streamen van gegevens over het mobiele netwerk geld kan kosten en niet zonder dat de gebruiker echt op een knop drukt mag gebeuren.

Alles goed en goed, maar wat als je een geluid wilt afspelen als reactie op een gebeurtenis, zeg eens per minuut de browser laten piepen:

player = document.getElementById('player');
player.src = 'beep-url';
beeper = setInterval(function() { player.play(); } ,1000 * 60);

Dit werkt prima in desktopbrowsers, maar op mobiel wordt de oproep naar play() niets doen.

Gelukkig zijn er drie dingen die de omzeiling eenvoudig maken:

Kortom, alles wat we nodig hebben is dat de gebruiker op een knop drukt die een leeg audio-element afspeelt om er de controle over te krijgen:

<audio id="player"></audio>
<button id="ok">OK om te piepen</button>
// Vermoedt jQuery
$("#ok").click(function() {
 player = document.getElementById('player');
 $( this ).slideUp();
 player.play(); // Speel het lege element af om de controle te krijgen
 player.src = 'beep-url'; // Stel de echte audio-bron in
 beeper = setInterval(function() { player.play(); } ,1000 * 60);
});

De klik kan alles zijn, een subtielere versie zou zijn om het te koppelen aan de sluit knop van een instructiemodal aan het begin van een spel.

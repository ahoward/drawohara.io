/using-terminal-notifier-in-our-capfiles
gepubliceerd op: 2014-01-02

We gebruiken capistrano voor onze implementaties en recentelijk ben ik de geweldige terminal-notifier gem tegen gekomen die de gebruikersmeldingen in OS X 10.8 en hoger gebruikt om een leuke melding van de opdrachtregel te tonen. Elke keer dat ik implementeer, controleer ik handmatig of de implementatie is voltooid, en dit is een mooie oplossing om dat te vermijden. Een probleem waar ik tegen aanliep toen ik het aan een van onze projecten toevoegde, is dat sommige mensen in ons team een versie van Mac OS X v√≥√≥r 10.8 gebruiken (ik weet!), zodat de terminal-notifier opdracht zou falen en hun implementatie foutmeldingen zou bevatten. Hier is de code die ik nu gebruik om te zorgen dat er geen foutmeldingen verschijnen als je de leuke gebruikersmeldingen in nieuwere versies van OS X niet kunt gebruiken:

desc 'gebruik terminal notifier als in > osx 10.8 om een gebruikersalert te gebruiken'
task :terminal do
 begin
  require 'terminal-notifier'
        url         = fetch(:url)
        application = fetch(:application)
        system "terminal-notifier -title #{ application } -message 'Implementatie succesvol voltooid naar #{ url }.' -sound default"
 rescue LoadError
        warn "Je gebruikt waarschijnlijk niet os x 10.8, dus geen gebruik van terminal notifier. Als je dat wel doet, gebruik gem install terminal-notifier om enkele leuke gebruikersmeldingen te gebruiken."
 end
end

Uiteraard voegen we ook gem terminal-notifier toe aan onze Gemfile. Zie mijn SO vraag over het opnemen van deze gem in je gemfile voor een uitleg waarom ik de eenvoudige gem-include-instructie gebruik.

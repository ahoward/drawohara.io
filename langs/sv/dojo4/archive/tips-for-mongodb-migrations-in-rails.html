Jag vill dela en snabb och enkel metod för att testa Rails-migreringar när du använder MongoDB-databasen. Mongo och rubys flexibilitet gör detta ganska enkelt. I det här exemplet kommer vi att byta namn på en fält i vår exempelsamling "books" från "isbn" till "book_number". Det här är en ganska vanlig typ av migrering och när du väl har fått grepp om det här enkla fallet, följer mer komplexa migreringar samma mönster. Först, låt oss generera vår tidsstämplade migrationsskriptmall.

rails generate migration rename_isbn_to_book_number

Sedan kommer vi att redigera filen som genererades under db/migrate. Vi kommer att dela upp vår kod i flera avsnitt enligt följande.

  1. Några klasskonstanter som delas av alla metoder
  2. Metoden self.up som utför framåtmigreringen
  3. En hjälpmetod för att ställa in några exempeldata för att testa framåtmigreringen
  4. Metoden self.down som utför återställningen
  5. En hjälpmetod för att ställa in några exempeldata (om det behövs) för att testa återställningen

Först, definiera bara några konstanter som våra metoder kommer att använda. Vi lagrar Mongo-uppdateringsalternativhash MultiUpdate för bekvämlighet eftersom de flesta migrationsuppdateringsoperationer vill ha upsert falskt (skapa inga nya dokument), multi sant (uppdatera alla matchande dokument) och safe sant.

Sedan definierar vi även en konstant för samlingens namn. För den verkliga migreringen är samlingen "books", men för testning kommer vi att skapa en ny samling som heter "test_books_migration" när vi utvecklar vår kod.

class RenameIsbnToBookNumber < Mongoid::Migration
  MultiUpdate = {:upsert => false, :multi => true, :safe=>true}
  Collection = db.collection("books") #Slutgiltig produktionskod
  Collection = db.collection("test_books_migration") #Bara för testning i konsolen
  def up
  end

  def down
  end
end

OK, det är vår inledande mall. Nästa steg är att ta en säkerhetskopia av vår utvecklingsdatabas om det finns några data där som vi inte vill råka förstöra. Sedan börjar vi koda en liten hjälpmetod för att fylla på vår testsamling med falska dokument som liknar vad vi förväntar oss se i produktion, men bara fokuserar på de fält som är relevanta för migreringen. Lägg till den här metoden till din migrationsklass.

def mock_data_for_testing_up
  3.times {|number| Collection.insert({"isbn" => "#{number}"})}
end

def show_collection
  Collection.find({}).each {|_| puts _}
end

Detta kommer att skapa 3 dummydokument som vi kan använda för testning. Vi lägger den här koden i en metod så att det blir lätt att köra om när vi justerar och testar vår migrationskod. För komplexa migreringar kan många omgångar av justeringar behövas för att få alla gränsfall att fungera.

Vi kan nu starta en rails-konsol och köra den här koden genom att kopiera och klistra in de två klasskonstanterna och mock_data_for_testing_up-metoden i konsolen och sedan köra mock_data_for_testing_up

Så nu har vi en separat, välförstådd testsamling redo att testa vår enkla migrering. Låt oss koda upp vår up metod. För att göra vår migrering.

def up
  #Vi vill byta namn på fältet book.isbn till book.book_number
  Collection.find({"isbn" => {"$exists" => 1}}).each do |book|
    update_op = {
      "$unset" => {"isbn" => 1},
      "$set" => {"book_number" => book["isbn"]}
    }
  Collection.update({"_id" => book["_id"]}, update_op, MultiUpdate)
end

Vi kan klistra in det i konsolen och köra det för att testa vår migrering. Vi kan verifiera resultaten med show_colletion. Om vi vill testa andra poster för återställningen kan vi skapa en mock_data_for_testing_down metod.

Detta bör ge dig ett mycket snabbt sätt att experimentera och få din migrationskod att fungera. Mongo har några avancerade fråge- och ändringsfunktioner som kan göra fantastiska saker, och ett enkelt sätt att göra lite prov och fel är praktiskt. Om du råkar göra röra i dina testdata kan du använda Collection.drop för att få en rena tavla. Här är den slutgiltiga migrationskoden för referens. Glöm inte att ta bort testsamlingens konstant och ta bort testsamlingen från din databas när du är redo att börja köra din kod för riktigt med rake db:migrate.

class RenameIsbnToBookNumber < Mongoid::Migration
  MultiUpdate = {:upsert => false, :multi => true, :safe=>true}
  Collection = db.collection("books") #Slutgiltig produktionskod

  def up
    #Vi vill byta namn på fältet book.isbn till book.book_number
    Collection.find({"isbn" => {"$exists" => 1}}).each do |book|
      update_op = {
        "$unset" => {"isbn" => 1},
        "$set" => {"book_number" => book["isbn"]}
      }
      Collection.update({"_id" => book["_id"]}, update_op, MultiUpdate)
    end
  end

  def down
    #Vi vill byta namn på fältet book.book_number till book.isbn
    Collection.find({"book_number" => {"$exists" => 1}}).each do |book|
      update_op = {
        "$unset" => {"book_number" => 1},
        "$set" => {"isbn" => book["book_number"]}
      }
      Collection.update({"_id" => book["_id"]}, update_op, MultiUpdate)
    end
  end

  #Dessa metoder anropas inte av migreringen. Bara för manuell testning
  #genom att kopiera/klistra in i konsolen
  #För att testa (genom att kopiera/klistra in härifrån till konsolen)
  #1. Ställ in MultiUpdate-konstanten. Justera "Collection" för att vara en testsamling
  #2. Kopiera/klistra in de 2 metoderna nedan
  #2. Kör mock_data_for_testing_up
  #3. Kör kroppen på self.up
  #3b. Kör valfritt kroppen på self.up igen för att se till att den är idempotent
  def mock_data_for_testing_up
    3.times {|number| Collection.insert({"isbn" => "#{number}"})}
  end

  def show_collection
    Collection.find({}).each {|_| puts _}
  end
end

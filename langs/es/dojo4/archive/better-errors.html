/better-errors
publicado el: 2014-03-31

Una de las gemas de Rails que hemos estado usando en nuestras aplicaciones más recientes es Better Errors, que permite ser introducido en un REPL similar a pry y una página de error más agradable cuando Rails genera una excepción durante el proceso de solicitud del navegador.


(Fuente de la imagen)

Esto es útil para nuestros desarrolladores porque, en lugar de obtener una página de error estática, ir a buscar en el código dónde poner un binding.pry, y volver a ejecutar el código propenso a errores, Better Errors carga una página donde podemos interactuar con el código allí mismo, lo que nos permite encontrar el problema más rápido, escribir una solución más rápido e iterar más rápido.

Una de las características recientes que acabamos de descubrir es la capacidad de ver el último error rpc. Tenemos un patrón de llamada ajax a un método rpc en nuestra aplicación de Rails cuando solo necesitamos obtener alguna información o publicar un objeto simple como favoritar a un usuario o algo así. El código que se ejecuta en nuestro servidor de Rails que maneja esa solicitud ajax no hará explotar todo el servidor y automáticamente invocará Better Errors, por lo que la solución es ir a una URL especial que Better Errors expone para mostrar el último error lanzado en nuestro código de Rails: http://localhost:3000/__better_errors.

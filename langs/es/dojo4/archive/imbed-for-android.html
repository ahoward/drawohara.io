/imbed-for-android
publicado el: 2014-04-09

Hace un tiempo escribí sobre por qué phonegap apesta y cómo elegimos incorporar contenido web en nuestras aplicaciones nativas. El código abierto que resultó de la creación de varias aplicaciones nativas que incluían contenido web fue el proyecto Imbed. Lanzamos la versión para iOS primero.

El proyecto tiene como objetivo ser un punto de partida para lo que creemos que es la mejor manera de incorporar contenido web en una aplicación nativa: usando código nativo. En iOS, esto significa escribir Objective-C para usar una UIWebView y en Android, escribir Java para usar una WebView. Sí, esto requiere que sepas lo suficiente sobre iOS/Objective-C y Android/Java para escribir tus propias funciones nativas cuando quieras usar características del dispositivo, pero eso significa poder seguir las direcciones de Apple y Google exactamente y no depender de un tercero para soportar una nueva característica y soportarla sin errores propios.

Recientemente he estado investigando más a fondo sobre Android y he creado la versión para Android de Imbed. Además de proporcionar un buen punto de partida para incluir contenido web, el código html de ejemplo demuestra algunas limitaciones de javascript en la webview y proporciona soporte integrado para el importante botón de retroceso en los teléfonos Android cuando haces clic en un enlace que necesita abrirse en Chrome en lugar de en el contenido web dentro de tu aplicación.

Consulta la versión para Android y házmelo saber en Twitter tus pensamientos.

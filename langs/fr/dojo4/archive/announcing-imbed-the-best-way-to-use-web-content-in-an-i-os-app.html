Souvent, il est logique de faire du développement en HTML, JS et CSS, puis d'utiliser ce contenu dans une application native. Phonegap prétend être la meilleure solution pour ce cas, mais d'après mon expérience, c'est l'une des pires.

En réponse à Phonegap, j'ai open sourcé notre dépôt de démarrage de l'entreprise pour les applications iOS qui doivent utiliser du contenu web dans un UIWebView, en l'appelant Imbed.

Caractéristiques:

  • Petit et Open Source. Un UIViewController, un fichier HTML et le projet super petit, open source à 1 classe, WebViewJavascriptBridge.

  • Supportez immédiatement toute fonctionnalité iOS. Pas besoin d'attendre qu'un développeur tiers crée un plugin. Vous écrivez vous-même du code natif Objective-C. Donc, dès qu'Apple le publie, vous pouvez le supporter.

  • Suivez les guides d'Apple, pas ceux de quelqu'un d'autre. Apple fait un excellent travail de documentation sur l'utilisation des fonctionnalités d'iOS. La plupart des développeurs tiers ne le font pas. Lequel préférez-vous suivre?

  • Automatisez tout avec Rake. Concentrez-vous sur l'écriture de votre code, testez-le dans le navigateur, puis automatisez tout le reste dans un fichier Rake clair et open source. Même archivez, envoyez à Testflight et notifiez Campfire.

C'est votre application. Ne vous fiez pas à un outil tiers pour prendre en charge les fonctionnalités qu'iOS offre déjà. Prenez votre code en main.

Téléchargez Imbed maintenant.

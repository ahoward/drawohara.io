rails, y compris rails3, possède ce joli morceau de code lié à la fourniture d'une méthode « to_json » sur les types intégrés :

# Astuce pour charger la gemme json en premier afin de pouvoir écraser sa méthode to_json.
    begin
      require 'json'
    rescue LoadError
    end
    # La gemme JSON ajoute quelques modules aux classes de base de Ruby contenant la définition de :to_json, écrasant
    # leur comportement par défaut. Cela dit, nous devons définir la méthode to_json de base dans toutes,
    # sinon elles utiliseront toujours l'implémentation de la gemme to_json, qui est incompatible en arrière dans
    # plusieurs cas (par exemple, l'implémentation JSON pour Hash ne fonctionne pas) avec l'héritage
    # et par conséquent les classes telles que ActiveSupport::OrderedHash ne peuvent pas être sérialisées en json.
    [Object, Array, FalseClass, Float, Hash, Integer, NilClass, String, TrueClass].each do |klass|
      klass.class_eval <<-RUBY, __FILE__, __LINE__
# Déverse l'objet en JSON (JavaScript Object Notation). Voir www.json.org pour plus d'infos.
def to_json(options = nil)
ActiveSupport::JSON.encode(self, options)
end
RUBY
    end

ce que cela fait effectivement, c'est écraser la méthode « to_json » que fournit la gemme json avec une que active_support estime meilleure. Le commentaire sur le fait que json soit cassé est obsolète et cela a toujours été une astuce. Dans mon cas, j'aime vraiment ce genre de chose dans mon contrôleur /api :

  def to_json(object)
    if Rails.env.production?
      JSON.generate(object)
    else
      JSON.pretty_generate(object)
    end
  end

car cela facilite énormément le débogage. Voici comment je l'ai fait dans une application rails3 -

la première étape consistait à faire en sorte que rails3 supporte le concept de « préinitialisateur » que l'ancien rails utilisait :

    ### fichier : config/preinitializer.rb
    dirname = File.dirname(File.expand_path(__FILE__))
    glob = File.join(dirname, 'preinitializers', '**/**.rb')
    preinitializers = Dir.glob(glob)
    preinitializers.each{|preinitializer| Kernel.load(preinitializer)}

ensuite, j'ai ajouté ce préinitialisateur :

### fichier : config/preinitializers/un_fuck_to_json.rb
    begin
      require 'rubygems'
    rescue LoadError
    end
    begin
      require 'json'
    rescue LoadError
    end
    [Object, Array, FalseClass, Float, Hash, Integer, NilClass, String, TrueClass].each do |klass|
      klass.class_eval do
        alias_method('__to_json__', 'to_json') if method_defined?('to_json')
      end
    end

enfin, cet initialisateur :

### fichier : config/preinitializers/un_fuck_to_json.rb
    begin
      require 'rubygems'
    rescue LoadError
    end
    begin
      require 'json'
    rescue LoadError
    end
    [Object, Array, FalseClass, Float, Hash, Integer, NilClass, String, TrueClass].each do |klass|
      klass.class_eval do
        alias_method('__to_json__', 'to_json') if method_defined?('to_json')
      end
    end

en anglais, ce que cela fait, c'est

  • faire en sorte que rails supporte le code qui s'exécute avant tout le reste
  • pendant cette phase, absorber la gemme json et se souvenir de la façon dont elle fait les choses
  • nuire à la méthode de rails et restaurer la méthode json de to_json

Mettre vos modèles en cache ou les sérialiser pour d'autres raisons ne devrait pas être douloureux. Malheureusement, c'est le cas avec la plupart des ORM, car ils ont une stratégie de sérialisation médiocre par défaut. Cependant, cela est facile à corriger : la clé réside dans la compréhension que chaque ORM sait déjà comment prendre un hachage d'informations de la base de données et instancier une instance complète. En sachant cela, nous pouvons facilement faire en sorte que n'importe quel modèle se sérialise comme du beurre.


#! /usr/bin/env ruby
# certains modèles ont des trucs bizarres qui ne survivent pas à un cycle de sérialisation
#
  class Model
    include Mongoid::Document
    def initialize(*args, &block)
      super
    ensure
      @fail = Class.new, open(__FILE__)
    end
  end
# donc cela va échouer
#
  begin
    p Marshal.load(Marshal.dump(Model.create))
  rescue Object => o
    warn "#{ o.message } (#{ o.class })"
  end
# mais les modèles mongoid ont simplement besoin d'un hachage d'informations du pilote mongo
# pour se vivifier complètement... donc c'est tout ce que nous devons conserver lorsqu'ils sont sérialisés. Cela rend le chargement à partir de données sérialisées *juste comme* le chargement à partir de
# la base de données.
#
# si vous me demandez, cela devrait être le comportement par défaut !
#
# hrm - je suis sur le noyau de mongoid... @durran, qu'en penses-tu ?
#
# d'ailleurs - cela fonctionne très bien avec active_record aussi...
#
  class Model
    def _dump(*args, &block)
      Marshal.dump(raw_attributes, *args, &block)
    end
    def Model._load(string, *args, &block)
      raw_attributes = Marshal.load(string, *args, &block)
      instantiate(raw_attributes)
    end
  end
# alors maintenant cela fonctionne juste (TM)
#
  p Marshal.load(Marshal.dump(Model.create))

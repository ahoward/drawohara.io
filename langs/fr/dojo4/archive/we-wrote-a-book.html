Pendant les vacances, je réfléchissais à mon introduction à Ruby on Rails au dojo. En tant que développeur full stack qui se concentre principalement sur le front-end, il était accablant de se lancer dans un projet rails existant avec une petite tâche javascript. Quels fichiers modifier ? Comment mes scripts sont-ils inclus dans la page ? Comment une requête est-elle traitée dans rails ? Ahh!!

Alors j'ai écrit un livre à ce sujet. Mes incroyables coéquipiers du dojo4 m'ont aidé à le revoir pour sa solidité technique et je pense qu'il est à un excellent point pour être publié dans le monde. Espérons qu'il aide d'autres développeurs front-end qui se lancent dans un projet rails et réalisent qu'ils sont perdus. Que ce livre soit votre boussole !

Description :

Ruby on Rails est un framework d'application web full-stack extrêmement puissant, mais avec sa puissance vient la complexité. Pour les développeurs front-end qui doivent simplement modifier HTML, CSS et JS dans une application Ruby on Rails, cela peut être une tâche ardue. Ce livre explique ce que les développeurs front-end doivent savoir sur Ruby on Rails afin de travailler efficacement dans une application rails.

À la fin du livre, vous serez en mesure de :

* savoir où placer votre html, css et js dans la structure d'une application ruby on rails

* comment rails achemine les requêtes aux contrôleurs

* utiliser des vues, des mises en page et des partials dans les contrôleurs

* ce qu'est le pipeline d'assets et comment l'utiliser

* comment les contrôleurs et les vues dans rails travaillent ensemble pour rendre une réponse dans le navigateur

* comment les données sont ajoutées aux vues et comment lier les données dans votre html

/screen-set-up-your-project-s-terminal-environment-once-and-only-once
Publié le : 2013-11-01

Pour chaque projet, j'exécute plusieurs programmes en même temps pendant le processus de développement. Cela se traduit généralement par plusieurs onglets dans l'application Terminal - un pour mon serveur local, un pour ma base de données, un pour mon application cliente, et un pour git. (Actuellement, j'utilise un éditeur GUI (ST2), mais je pourrais revenir à vim bientôt…) Cette configuration à plusieurs onglets se reproduit pour chaque projet sur lequel je travaille, et je passe généralement d'un projet à l'autre plusieurs fois par jour.

Avec cette configuration à plusieurs onglets, lorsque je dois changer de projet, je dois soit ouvrir une nouvelle fenêtre Terminal avec ses propres onglets pour ce projet et gérer un nombre de fenêtres plus important que nécessaire (ce qui n'est pas une option pour un minimaliste comme moi), soit fermer tous les onglets que j'ai ouverts, pour les rouvrir tous lorsque je reviens à ce projet.

Eh bien, comme la plupart des jours, j'ai appris une solution astucieuse à ce problème de la part du sage et tout-puissant leader de dojo4, Ara. Il utilise le programme GNU Screen. Au sein d'un seul onglet Terminal, Screen vous permet d'ouvrir plusieurs "fenêtres" (alias onglets) dans le même onglet Terminal. Voici comment l'utiliser :

# création et réattachement à une session
screen -S [nom] = créer un écran nommé
screen -D -R [nom] = réattacher à un écran nommé et détacher les autres personnes si nécessaire
screen -list = lister les écrans

# (la fermeture de toutes les fenêtres dans une session d'écran quittera cette session. Cependant, une session d'écran est un processus en cours d'exécution, vous pouvez donc la tuer manuellement.)

# dans une session d'écran :
ctrl-A + ? = aide
ctrl-A + " = lister les fenêtres
ctrl-A + c = créer une nouvelle fenêtre
ctrl-A + n = fenêtre suivante
ctrl-A + p = fenêtre précédente
ctrl-A + A = renommer une fenêtre

# alias que nous utilisons
alias sl='screen -list '
alias sdr='screen -d -r '
alias s='screen -D -R '
alias ss='screen -S'

sdr [nom] = réattacher à un écran nommé et détacher les autres personnes si nécessaire
ss [nom] = créer un écran nommé
sl = lister les écrans

Une autre fonctionnalité vraiment cool de screen est que lorsque vous vous détachez d'une session d'écran, toutes vos fenêtres continuent de fonctionner, et lorsque vous vous réattachez à la session d'écran, vous avez toutes vos fenêtres à nouveau à votre disposition.

Lorsque nous en avons discuté dans notre bureau, Ryan Cook a créé un petit helper Ruby sympa pour créer des sessions d'écran nommées d'après votre répertoire actuel.

Mon flux de travail est donc passé de l'ouverture de plusieurs onglets dans Terminal à un seul onglet dans Terminal où je me reconnecte à une session d'écran pour chaque projet sur lequel je travaille. Cela me permet de changer de projet extrêmement rapidement et d'arrêter de perdre du temps à reconfigurer et à démonter tous les onglets Terminal que j'utilise pour un projet.

Maintenant, je dois simplement convertir plus de choses en utilisation de terminal (comme mon éditeur) afin de vraiment tirer le meilleur parti de screen…

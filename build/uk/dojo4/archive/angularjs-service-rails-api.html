Нещодавно мені довелося заглиблюватися в AngularJS на сайті Rails, який ми розробляли для клієнта. Крім вмісту сторінок, у клієнта є певні сторінки, які відображають заголовки, що ведуть до зовнішніх джерел, але потрапляють у наш стек Rails через періодично виконувану задачу rake. Сайт повинен бути сильно кешований за допомогою CDN, як CloudFront, але щоб заголовки залишалися свіжими, ми використовували AngularJS (тому що клієнт був знайомий з ним, тож це простіше в обслуговуванні для них), щоб запитувати стек rails безпосередньо щодо останніх заголовків.

Це вимагало робити це в кросдоменному стилі, оскільки домен сайту буде вказувати на CloudFront. Я ще не писав API Rails, який повертав би JSONP або сервіс AngularJS, тому мені довелося трохи дослідити, і ось що працювало для мене. (Ви також можете прочитати gist.)

Примітка: якщо ви не знайомі з JSONP, це не матиме для вас багато сенсу. Прочитайте це.

Сервіс AngularJS

AngularJS має концепцію сервісів, які просто синглтони, які можна використовувати де завгодно. У нашому випадку ви можете уявити наш сервіс заголовків як клієнт API заголовків, який обгортає наші виклики JSONP у простіший і централізований синтаксис. AngularJS надає деякі сервіси за замовчуванням, але ви можете писати власні. Насправді, ми могли б використовувати вбудований $resource сервіс замість написання власного, однак у мене виникли деякі проблеми з роботою JSONP, і я хотів краще зрозуміти нижорівневі компоненти AngularJS, тому я написав свій власний:

angular.module('app')
  .factory('HeadlineService', ['$http',
    function($http) {
      'use strict';

      var BASE_URL = "<%= [App.settings.api_base_url, '/services/headlines'].join %>",
          CALLBACK_STRING = "?callback=JSON_CALLBACK";

      return {
        getHeadlines: function(){
          return $http.jsonp(BASE_URL + CALLBACK_STRING)
        },
        getHeadlineForId: function(id){
          return $http.jsonp(BASE_URL + "/" + id + ".json" + CALLBACK_STRING)
        }
      }
    }
  ]);

Сервіс вище використовує вбудований $http сервіс, який просто надає AJAX-обгортку, для надсилання запитів до нашого бекенду rails для отримання заголовків. Декілька речей, на які варто звернути увагу:

  • Він повертає об'єкт з 2 функціями: getHeadlines та getHeadlineForId. Ми можемо використовувати ці функції як скорочення для звернення до API Rails у наших контролерах AngularJS.
  • Він використовує JSONP. Поміть, що ми використовуємо $http.jsonp, і ми передаємо рядок запиту ?callback=JSON_CALLBACK. AngularJS замінює рядок JSON_CALLBACK на ім'я функції зворотного виклику, яку він створює для вас.
  • Він використовує інший домен. Ну, це очікувано, оскільки ми використовуємо JSONP, але ми досягаємо цього, додаючи розширення .erb до файлу та використовуючи те, що ми встановили для App.settings.api_base_url в додатку Rails як домен, з яким має спілкуватися AngularJS. Пам'ятайте, що це для обходу кешувального шару CDN, щоб наші заголовки завжди були найактуальнішими.

Контролер AngularJS

З побудованим HeadlinesService ми тепер можемо використовувати 2 функції, які він повертає, у нашому додатку AngularJS:

angular.module('app')
  .controller('HeadlinesCtrl', ['$scope', 'HeadlineService',
    function($scope, HeadlineService) {
      'use strict';
 
      HeadlineService.getHeadlines().success(function(data, status, headers, config){
        $scope.headlines = data;
        
        HeadlineService.getHeadlineForId(data[0].id).success(function(data){
          $scope.single_headline = data;
        }).error(function(error, status, headers, config){
          alert('чтось пішло не так при отриманні заголовка');
        });
      }).error(function(error, status, headers, config){
        alert('щось пішло не так при отриманні всіх заголовків');
      });
    }
  ]);

Для прикладу я просто отримую повний об'єкт заголовків для першого заголовка, повернутого в успішному колбеку getHeadlines.

Об'єкт $scope доступний для переглядів у AngularJS, тому ми можемо відображати заголовки на сторінці, маючи перегляд такий:

<section class="headline_section" ng-controller="HeadlinesCtrl">

  <h3>Ось деякі заголовки...</h3>

  <div class="headlines">

    <ul>
      <li ng-repeat="headline in headlines">
        <a href="{{ headline.url }}" target="_none">{{ headline.title }}</a>
      </li>
    <ul>

  </div>

  <div class="headline">
    <a href="{{ single_headline.url }}" target="_none">{{ single_headline.title }}</a>
  </div>

</section>

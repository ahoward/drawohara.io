Сьогодні я дізнався цю істину про JavaScript від Джеймса.

“\r\n\t” == false

Що це? Мені довелося перевірити.

    % node
    > "\r\n\t" == false
true

І, звичайно, пояснення від Джеймса:

Ось чому: браузер виявляє, що два операнди мають різні типи, тому він перетворює false на 0 і повторює порівняння. Операнди все ще мають різні типи (рядок і число), тому браузер змушує “\r\n\t” перетворитися на число 0, бо якось ненульова кількість символів дорівнює 0. Воля—0 дорівнює 0! ЧУДОВО. Це пояснення було схоже на сюжет фільму «Початок», але закладена ідея була така: «правильність вашої програми була змушена до false».

Справді, я просто мусив поділитися.

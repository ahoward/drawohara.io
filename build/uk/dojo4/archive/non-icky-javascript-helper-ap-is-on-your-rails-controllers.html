Веб-додатки майже завжди потребують великої кількості маленьких допоміжних методів JavaScript: ви знаєте, автозаповнення форм, заповнення значень за замовчуванням, перевірки, тощо. Це не справжні API-виклики, лише невеликі шматочки функціональності, необхідні для роботи відображень.

Завжди є питання, до якого контролера підвісити ці методи. Чи потрібно створити глобальний допоміжний контролер для всього цього? Чи підвісити їх до контролера, що запитується? Як поділити бекенд JavaScript допоміжних методів між контролерами?

Крок перший

Додайте модуль RPC до вашого ApplicationController

  class ApplicationController < ::ActionController::Base
    include RPC
  end

Крок другий

У контролерах оголосіть допоміжні методи RPC

  class PostsController < ApplicationController
    rpc(:defaults) do |params|
      post = Post.new(params)
      post.set_defaults!
      post.as_json
    end
  end

Крок третій

Налаштуйте JS для виклику методів RPC відносно до поточного контролера

  // in file: app/views/layouts/application.html.erb
  //
  // notice the *relative* url rpc endpoint
  //
  // <script>
  
    jQuery(function(){
        var rpc = function(){
        // parse args
        //
          var args = Array.prototype.slice.call(arguments);
          var ajax;
          var method;
          if(args.length == 1 && typeof(args[0]) == 'object'){
            ajax = args.shift();
          } else {
            ajax = {};
            for(var i = 0; i < args.length; i++){
              var arg = args[i];
              switch (typeof(arg)) {
                case 'object':
                  ajax['data'] = arg;
                  break;
                case 'function':
                  ajax['success'] = arg;
                  break;
                case 'string':
                  method = arg;
                  break;
              };
            }
          };
        // normalize ajax options
        //
          ajax['url'] = ajax['url'] || rpc['url'];
          method = method || ajax['method'];
          delete ajax['method'];
          ajax['data'] = ajax['data'] || {};
          ajax['data']['method'] = method;
          ajax['async']          = true;
          ajax['type']           = 'GET';
          ajax['dataType']       = 'json';
          ajax['cache']          = false;
          ajax['contentType']    = 'application/json; charset = utf-8';
          var result = ajax;
        // ajax request - using callback or returning the result sync style
        //
          if(!ajax['success']){
            ajax['async'] = false;
            ajax['success'] = function(response){ result = response };
          };
          jQuery.ajax(ajax);
          return(result);
        };
        
        
        rpc['url'] = <%= raw url_for(:action => :rpc).to_json %>;
        
        
        window.rpc = rpc;
    });
  
  // </script>

Крок четвертий

Ввімкніть кінцеві точки RPC в config/routes.rb

  resources :posts do
    collection do
      get 'rpc'
    end
  end

Крок п'ятий

Тепер ви можете використовувати допоміжні засоби RPC у JS відносно ваших відображень

  var title = $('#title');
  var slug = $('#slug');
  title.change(function(){
    var value = title.val();
    
    var data = {'title' : value};
    
    var success = function(defaults){
      slug.val(defaults['slug']);
    };
        
    rpc('defaults', data, success);
  });
  

Ця організація тримає ваш код впорядкованим, уникаючи надмірності контролерів, і запобігає необхідності засмічувати чисті ресурсні контролери великою кількістю потворних дій допоміжних

Ну, як і в більшості днів, я дізнався інтелектуальне рішення цієї проблеми від мудрого та всемогутнього лідера dojo4, Ара. Він використовує програму GNU Screen. У рамках однієї вкладки терміналу Screen дозволяє відкривати кілька "вікон" (також відомих як вкладки) всередині тієї самої вкладки терміналу. Ось як цім користуватися:

# створення та повторне підключення до сеансу
screen -S [name] = створити названий екран
screen -D -R [name] = повторно підключитися до названого екран та відключити інших, якщо потрібно
screen -list = список екранів

# (вихід зі всіх вікон у сеансі екрану призведе до виходу з цього сеансу. Однак сеанс екрану - це запущений процес, тож ви можете вручну завершити його.)

# всередині сеансу екрану:
ctrl-A + ? = допомога
ctrl-A + " = список вікон
ctrl-A + c = створити нове вікно
ctrl-A + n = наступне вікно
ctrl-A + p = попереднє вікно
ctrl-A + A = перейменувати вікно

# псевдоніми, які ми використовуємо
alias sl='screen -list '
alias sdr='screen -d -r '
alias s='screen -D -R '
alias ss='screen -S'

sdr [name] = повторно підключитися до названого екран та відключити інших, якщо потрібно
ss [name] = створити названий екран
sl = список екранів

Ще одна дуже крута функція екрану полягає в тому, що коли ви відключаєтеся від сеансу екрану, всі ваші вікна залишаються запущеними, і коли ви знову підключаєтеся до сеансу екрану, у вас знову всі ваші вікна в наявності.

Коли ми обговорювали це в нашому офісі, Раян Кук створив гарний маленький допоміжник Ruby для створення сеансів екрану, які названі за вашою поточною директорією.

Таким чином, мій робочий процес змінився від наявності кількох вкладок відкритими в терміналі до однієї вкладки в терміналі, де я повторно підключаюся до сеансу екрану для кожного проекту, над яким працюю. Це дозволяє мені дуже швидко переключатися між проектами та припинити втрачати час на повторне налаштування та завершення всіх вкладок терміналу, які я використовую для проекту.

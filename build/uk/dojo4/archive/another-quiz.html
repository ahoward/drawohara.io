fubar doctype

зразкові відповіді

  • Відсутній doctype. Це ускладнює визначення версії HTML, що може призвести до цікавих відповідей…

  • Мішанина xhtml-стильних порожніх тегів із HTML-тегами.

  • Непослідовне використання лапок. Обидва варіанти працюють, виберіть якийсь один...

  • рядок 1: DELETE не підтримується в HTML-формах:
  • method=delete не підтримується безпосередньо в більшості/всіх? браузерах.
  • форми не підтримують delete у старих браузерах
  • Method delete

  • form method=’DELETE’ – це недопустимо

  • enctype=”multipart/form-data” а не enctype=”multipart/file-upload”

  • enc-type=”multipart/file-upload не має сенсу, якщо наша мета – DELETE

  • текстові введення в першому fieldset мають однакове ім'я
  • дублювання тегів “”. Або два теги з однаковим ім’ям знову
  • 2 текстових поля з ім'ям “tags” буде використано лише останнє.
  • Два введення з однаковим ім'ям ‘tags’. Другий перезапише значення першого.
  • Два введення з ім'ям ‘foo’ (не закриті)
  • текстові введення з однаковим ім'ям перезапишуть ре: введення з ім'ям tags
  • подвійні призначення імені input name=’tags’

  • подвійні призначення імені input name=’vehicle’

  • Немає for у мітці ‘Bar:’.

  • незв'язана мітка: (мітка для чого?)

  • readonly введення без значення безглузде

  • action=”./” здається поганою практикою

  • дія форми недійсна як дескриптор файлу unix – щоб відправити до поточного url видаліть дію повністю, оскільки це за замовчуванням

  • завантаження файлу не може мати атрибута значення (погана безпека)
  • Вкладення fieldset
  • введення файлу не приймають значення за замовчуванням – користувач повинен вибрати

  • автоматично вибирає ваш ~/.bashrc файл для завантаження

  • disabled повинен бути в лапках (також погана суміш одинарних і подвійних лапок)
  • Незакриті опції для випадаючого списку select та без тексту

  • відсутні лапки навколо атрибута disabled у неправильній декларації введення пароля

  • Немає типу в полі пароля (також немає в полі електронної пошти, але це менш важливо)
  • введення пароля повинно бути типу password

  • має бути <input type=’email’ name=’email’ …>

  • disabled=disabled має бути disabled

  • Немає тексту для жіночої опції, також радіокнопка для радіокнопок ‘Sex’, радіокнопки не налаштовані правильно для відправки значення / перемикання в групі. Все це погано в цій області.
  • fieldset “sex” радіокнопки не побудовані правильно. має бути Чоловік – теги використовуються зі списками вибору, а не радіокнопками. Всі радіокнопки в групі повинні мати однакове ім'я атрибута

  • радіокнопки не приймають опції, але замість цього повинні мати однакове ім'я з різними значеннями

  • кнопки повинні бути відкриті та закриті – відправка повинна бути введенням
  • чекбокси з однаковим ім'ям перезапишуть ре: чекбокси з ім'ям vehicle

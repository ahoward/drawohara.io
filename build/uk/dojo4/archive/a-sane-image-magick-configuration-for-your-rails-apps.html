я часто казав, що завантаження файлів — це самонав'язана DOS-атака. Звичайно, не треба далеко шукати приклади того, як здавалося б, невелика функція додатку спричиняє хаос на сервері додатків: https://www.google.com/search?q=heroku+image+magick+timed+out

у @dojo4 ми використовуємо можливість image magick встановлювати обмеження через змінні середовища як перший захист, щоб процеси magick були добре налаштовані.

спочатку трохи підготовки/інформації: ми часто розгортаємо пару файлів RAILS_ROOT/env.yml та RAILS_ROOT/env.rb, які додаток rails завантажує під час запуску. Вони не містять жодних залежностей rails і можуть використовуватися в фонових скриптах для швидкого розуміння конфігурації розгорнутого додатку rails без необхідності завантажувати весь додаток rails.

#
# спочатку ми включаємо будь-які налаштування середовища, знайдені в

# RAILS_ROOT/env.yml
#
# уважаючи вже встановлені змінні середовища ENV. Файл env.yml

#
  require 'erb'
  require 'yaml'
  require 'rbconfig'
  
  env_yml = File.expand_path('../env.yml', __FILE__)
  
  if test(?s, env_yml)
    buf = IO.read(env_yml)
    expanded = ERB.new(buf).result(binding)
    config = YAML.load(expanded)
    config.each{|key, val| ENV[key.to_s] ||= val.to_s} if config.is_a?(Hash)
  end

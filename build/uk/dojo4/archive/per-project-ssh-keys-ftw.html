З розгортанням Rails + Capistrano цей налаштування

ssh_options[:forward_agent] = true

є майже фактично і розумно за замовчуванням. Він використовує всю вашу SSH-ключову доброту для всіх команд, роблячи більшість віддалених дій “Просто працюють ™”.

Однак, є випадки, коли це може бути плутаним, наприклад, коли у вас є два ключі, завантажені в ssh-agent, які відповідають різним обліковим записам GitHub або Bitbucket. Який обліковий запис ви отримаєте, буде веселою маленькою грою, щоб з'ясувати ;-/

Наступна дуже проста стратегія дозволяє вам зберігати SSH-ключі на проект дуже природним способом, а потім щоб Capistrano поважав ці ключі для всіх операцій “cap XXX”.

Спочатку вам потрібно створити директорію .ssh/ у вашому проекті, потім завантажити всі відповідні ключі туди. Ви можете мати багато пар, просто переконайтеся, що вони названі за допомогою звичайної конвенції *.id_rsa.

a:~/git/dojo4/rails_app_m $ ls .ssh
id_rsa.dojo4            id_rsa.dojo4.pub

Далі вам потрібно щось схоже на це в вашому Capfile, яке виявляє ці локальні ключі, якщо вони присутні, і використовує їх за допомогою тимчасового ssh-agent — в іншому випадку повертаючись до ваших завантажених глобальних ключів ssh-agent.

# якщо цей проект має директорію .ssh/, використовуйте ці ключі для розгортання.
# інакше використовуйте глобальні. Локальні ключі використовуються за рахунок запуску
# тимчасового ssh-agent і додавання цих локальних ключів до нього.
#
# ймовірно, у вас є ключ(і), які дозволяють доступ до певного хоста та репозиторію
# (GitHub, Bitbucket тощо) у вашій локальній директорії .ssh.
#
# ймовірно, ви хочете .gitignore .ssh/ !
#
  dot_ssh = File.expand_path("#{ rails_root }/.ssh", __FILE__)
  globs = [
    File.join(dot_ssh, "*id_rsa"),
    File.join(dot_ssh, "id_rsa*"),
  ]
  ssh_keys = [
    ENV['SSH_KEYS'], globs.map{|glob| Dir.glob(glob)}
  ].join(' ').scan(/[^\s]+/).delete_if{|ssh_key| ssh_key =~ /\.pub$/}
  unless ssh_keys.empty?
    require 'session'
    sh = Session::Sh.new
    at_exit{ sh.close }
    sh.execute 'eval `ssh-agent -s -t 3600`'
    ssh_auth_sock = sh.execute('echo $SSH_AUTH_SOCK').first.to_s.strip
    ssh_agent_pid = sh.execute('echo $SSH_AGENT_PID').first.to_s.strip
    ENV['SSH_AUTH_SOCK'] = ssh_auth_sock
    ENV['SSH_AGENT_PID'] = ssh_agent_pid
    ssh_keys.each do |ssh_key|
      if system "SSH_AUTH_SOCK=#{ ssh_auth_sock } ssh-add #{ ssh_key.inspect } >/dev/null 2>&1"
        Say.say("Using ssh-key #{ ssh_key.inspect }", :color => :green)
      else
        Say.say("Cound not add ssh-key #{ ssh_key.inspect }", :color => :red)
      end
    end
  end
# єдине розумне рішення — це завжди передавати ваш локальний або глобальний агент
#
  ssh_options[:forward_agent] = true

Вам потрібен гем session у вашому Gemfile, звичайно

# файл: Gemfile
gem 'session'

І не забудьте .gitignore вашу нову налаштовану .ssh

# файл: .gitignore
.ssh/*

Ми використовуємо capistrano для наших розгортань і недавно я натрапив на чудовий terminal-notifier гем, який використовує сповіщення користувача в OS X 10.8 і вище, щоб показати гарне сповіщення з командного рядка. Кожного разу, коли я розгортаю, я завжди перевіряю вручну, чи завершилося розгортання, і це гарне рішення, щоб уникнути цього. Одна проблема, з якою я зіткнувся, коли додав його до одного з наших проектів, полягає в тому, що деякі люди в нашій команді використовують версію Mac OS X до 10.8 (я знаю!), тому команда terminal-notifier буде невдалою, і їхнє розгортання буде містити помилки. Ось код, який я тепер використовую, щоб переконатися, що помилки не з'являтимуться, якщо ви не можете використовувати круті сповіщення користувача в нових OS X:

desc 'використовувати термінал-нотифієр, якщо в > osx 10.8 для використання сповіщення користувача'
task :terminal do
 begin
  require 'terminal-notifier'
        url         = fetch(:url)
        application = fetch(:application)
        system "terminal-notifier -title #{ application } -message 'Розгортання успішно завершено до #{ url }.' -sound default"
 rescue LoadError
        warn "Ви, ймовірно, не на os x 10.8, тому не використовуєте термінал-нотифієр. Якщо ви, використовуйте gem install terminal-notifier, щоб використовувати деякі круті сповіщення користувача."
 end
end

Очевидно, ми також включаємо gem terminal-notifier у наш Gemfile. Дивіться моє питання на SO щодо включення цього гему у ваш Gemfile для пояснення, чому я використовую просту включення гему.

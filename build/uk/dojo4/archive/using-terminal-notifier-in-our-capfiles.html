```html використання terminal notifier у наших capfiles
@drawohara

Ми використовуємо capistrano для наших розгортань та нещодавно я дізнався про чудовий terminal-notifier гем, який використовує користувацькі сповіщення в OS X 10.8 і вище, щоб показати приємне сповіщення з командного рядка. Кожного разу, коли я розгортаю, я завжди перевіряю вручну, чи завершилось розгортання, і це хороше рішення, щоб уникнути цього. Одна проблема, з якою я зіткнувся, коли додавав його до одного з наших проектів, полягає в тому, що деякі люди в нашій команді використовують версію Mac OS X до 10.8 (я знаю!), тому команда terminal-notifier не спрацює, і їхнє розгортання міститиме помилки. Ось код, який я тепер використовую, щоб запобігти помилкам, якщо ви не можете використовувати чудові користувацькі сповіщення в нових OS X:

desc 'використовуйте terminal notifier якщо в > osx 10.8 для використання користувацького сповіщення'
task :terminal do
 begin
  require 'terminal-notifier'
        url         = fetch(:url)
        application = fetch(:application)
        system "terminal-notifier -title #{ application } -message 'Розгортання успішно завершено до #{ url }.' -sound default"
 rescue LoadError
        warn "Ви, ймовірно, не на os x 10.8, тому не використовуєте terminal notifier. Якщо ви на ній, використовуйте gem install terminal-notifier для використання чудових користувацьких сповіщень."
 end
end

Звичайно, ми також включаємо gem terminal-notifier у наш Gemfile. Дивіться моє питання на SO про включення цього гему в ваш gemfile для пояснення, чому я використовую просту інструкцію включення гему.
```

Я хотів поділитися швидким і легким методом для тестування міграцій Rails при використанні бази даних MongoDB. Гнучкість монго та рубі робить це досить простим. У цьому прикладі ми перейменуємо поле в нашій прикладній колекції "books" з "isbn" на "book_number". Це досить поширений тип міграції, і коли ви зрозумієте цей простий випадок, більш складні міграції слідують тому ж патерну. Спочатку створімо наш шаблон скрипту міграції з таймштампом.

rails generate migration rename_isbn_to_book_number

Потім ми відредагуємо файл, створений у db/migrate. Ми розділимо наш код на кілька розділів, як показано нижче.

  1. Кілька констант класу, які використовуються всіма методами
  2. Метод self.up, який виконує пряму міграцію
  3. Допоміжний метод для створення прикладних даних для тестування прямої міграції
  4. Метод self.down, який виконує відкат
  5. Допоміжний метод для створення прикладних даних (якщо потрібно) для тестування відкату

Спочатку просто визначимо деякі константи, які будуть використовуватись нашими методами. Ми зберігаємо хеш опцій оновлення монго MultiUpdate для зручності, оскільки більшість операцій оновлення міграції хочуть upsert false (не створювати нові документи), multi true (оновлювати всі відповідні документи) і safe true.

Потім ми також визначаємо константу для назви колекції. Для справжньої міграції колекція - "books", але для тестування ми створимо нову колекцію під назвою "test_books_migration", коли розробляємо наш код.

Потім ми почнемо писати невеликий допоміжний метод для заповнення нашої тестової колекції фейковими документами, схожими на те, що ми очікуємо побачити у виробництві, але зосереджуючись лише на полях, які мають відношення до міграції. Додайте цей метод до вашого класу міграції.

def mock_data_for_testing_up
  3.times {|number| Collection.insert({"isbn" => "#{number}"})}
end

def show_collection

Це створить 3 фейкових документи, які ми можемо використовувати для тестування. Ми поміщаємо цей код у метод, щоб його було легко повторно запускати під час налаштування та тестування нашого коду міграції. Для складних міграцій може знадобитися багато раундів налаштування, щоб врахувати всі граничні випадки.

Тепер у нас є окрема, добре зрозуміла тестова колекція, готова для тестування нашої простої міграції. Давайте напишемо наш метод up. Щоб виконати нашу міграцію.

def up
  #Ми хочемо перейменувати поле book.isbn на book.book_number
  Collection.find({"isbn" => {"$exists" => 1}}).each do |book|
    update_op = {
      "$unset" => {"isbn" => 1},
      "$set" => {"book_number" => book["isbn"]}
    }
  Collection.update({"_id" => book["_id"]}, update_op, MultiUpdate)
end

Ми можемо вставити це в консоль і запустити, щоб протестувати нашу міграцію. Ми можемо перевірити результати за допомогою show_colletion. Якщо ми хочемо протестувати інші записи для відкату, ми можемо створити метод mock_data_for_testing_down.

Це має дати вам дуже швидкий спосіб експериментувати та запускати ваш код міграції. Mongo має деякі потужні можливості запитів та модифікацій, які можуть робити дивовижні речі, і легкий спосіб робити деякі проби і помилки дуже зручний. Якщо ви заплутані в своїх тестових даних, ви можете використовувати Collection.drop, щоб отримати чистий аркуш. Ось фінальний код міграції для довідки. Не забудьте видалити константу тестової колекції та видалити тестову колекцію з вашої бази даних, коли ви будете готові почати запускати свій код для реального використання з rake db:migrate.

class RenameIsbnToBookNumber < Mongoid::Migration
  MultiUpdate = {:upsert => false, :multi => true, :safe=>true}
  Collection = db.collection("books") #Фінальний продакшн код

  def up
    #Ми хочемо перейменувати поле book.isbn на book.book_number
    Collection.find({"isbn" => {"$exists" => 1}}).each do |book|
      update_op = {
        "$unset" => {"isbn" => 1},
        "$set" => {"book_number" => book["isbn"]}
      }
      Collection.update({"_id" => book["_id"]}, update_op, MultiUpdate)
    end
  end

  def down
    #Ми хочемо перейменувати поле book.book_number на book.isbn
    Collection.find({"book_number" => {"$exists" => 1}}).each do |book|
      update_op = {
        "$unset" => {"book_number" => 1},
        "$set" => {"isbn" => book["book_number"]}
      }
      Collection.update({"_id" => book["_id"]}, update_op, MultiUpdate)
    end
  end

  #Ці методи не викликаються міграцією. Тільки для ручного тестування
  #для тестування (скопіюйте/вставте відсюди в консоль)
  #1. Встановіть константу MultiUpdate. Налаштуйте "Collection", щоб це була тестова колекція
  #2. Скопіюйте/вставте 2 методи нижче
  #2. Запустіть mock_data_for_testing_up
  #3. Запустіть тіло self.up
  #3b. Опційно запустіть тіло self.up знову, щоб переконатися, що воно є ідемпотентним
  def mock_data_for_testing_up
    3.times {|number| Collection.insert({"isbn" => "#{number}"})}
  end

  def show_collection
    Collection.find({}).each {|_| puts _}
  end
end

Хостинг, DNS та HTTPS часто вимагають взаємодії більше ніж кількох провайдерів. Часто стають некерованими, якщо не керувати ними ефективно.

DNS вимагає як управління записами, так і реєстрації доменів. Для обох Dojo4 використовує DyDNS протягом багатьох років.

Dojo4 зазвичай використовує облачні комп’ютери Amazon AWS EC2 для хостингу та таким чином тяжіє до використання балансувальників навантаження AWS ELB перед цими веб-хостами. На жаль, Amazon не надає вам постійних IP-адрес для інстансів ELB, і отже єдиний спосіб створити записи доменів другого рівня, що вказують на інстанси ELB, — це використовувати Amazon AWS Route53 для управління записами, який має спеціальний запис alias для цієї мети. Тому ми тяжіємо до використання Route53 для управління записами DNS.

Через спосіб, яким проводиться переговорний процес TLS-рукостискування лінуксом/апачем, ви обмежені кількістю сертифікатів SSL, які можуть обслуговуватися до кількості IP-адрес на хості. З EC2 це фактично один сертифікат, якщо ви не використовуєте VPC. Dojo4 часто використовує ELB для надання SSL як спосіб обійти це.

DNS та HTTPS тісно пов’язані, оскільки домени та піддомени, які потрібно захистити, повинні бути перераховані перед наданням сертифікату. Часто найзручнішою та гнучкою стратегією є отримання сертифікату з підстановочним знаком. Зверніть увагу, що не всі видавці сертифікатів з підстановочним знаком включають базовий домен у сертифікат (тобто ви можете використовувати сертифікат лише для захисту *.somedomain.com, а не somedomain.com). Річний сертифіkat з підстановочним знаком за $100 від dnsimple.com, виданий Comodo та включаючи базовий домен, є хорошою вартістю навіть з вартістю хостингу dnsimple dns у $96 ($8/місяць * 12 місяців).

dnsimple має деякі інші зручні функції, такі як можливість делегувати управління наборами записів іншим обліковим записам користувачів dnsimple. Оскільки ми перенесли управління наборами записів з DynDNS на Route53 та маємо обліковий запис на dnsimple для надання SSL, ми перенесли реєстрацію доменів з DynDNS на dnsimple, щоб об’єднати послуги.

Деякі корисні команди командного рядка:

Знайти Start Of Authority для запису DNS

dig SOA dojo4.com

Переглянути сертифікат, який обслуговує хост

openssl s_client -showcerts -connect dojo4.com:443 < /dev/null

Доступ до веб-сервера за IP, ігноруючи перевірку SSL

curl -k https://184.72.61.241 -H “Host: projects.dojo4.com”

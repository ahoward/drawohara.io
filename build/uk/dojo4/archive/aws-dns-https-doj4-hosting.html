Хостинг, DNS та HTTPS часто вимагають взаємодії більше ніж декількох провайдерів. Часто це стає незручним, якщо не керувати цим ефективно.

DNS вимагає як управління записами, так і реєстрації доменів. Для обох Dojo4 використовував DyDNS протягом багатьох років.

Dojo4 зазвичай використовує Amazon AWS EC2 хмарні комп'ютери для хостингу і, отже, схильний використовувати AWS ELB балансувальники навантаження перед цими веб-хостами. На жаль, Amazon не надає вам постійних IP-адрес для інстансів ELB, і тому єдиний спосіб створити записи другого рівня доменів, що вказують на інстанси ELB, - це використовувати Amazon AWS Route53 для управління записами, яка має спеціальний псевдонім запису для цієї мети. Тому ми схильні використовувати Route53 для управління записами DNS.

Через спосіб, яким проводиться TLS-узгодження linux/apache, кількість сертифікатів SSL, які можуть бути надані, обмежена кількістю IP-адрес на хості. З EC2 це фактично один сертифікат, якщо ви не використовуєте VPC. Dojo4 часто використовує ELB для надання SSL як засіб вирішення цієї проблеми.

DNS та HTTPS тісно пов'язані, оскільки домени та піддомени, які потрібно захистити, повинні бути перераховані до надання сертифіката. Часто найзручнішою та гнучкою стратегією є отримання сертифіката зі зірочкою. Зауважте, однак, що не всі видавці сертифікатів зі зірочкою включають базовий домен у сертифікат (тобто ви можете використовувати сертифікат лише для захисту *.somedomain.com, а не somedomain.com). $100/рік сертифікат зі зірочкою від dnsimple.com, виданий Comodo, включаючи базовий домен, є хорошою цінністю, навіть з $96 ($8/місяць * 12 місяців) вартістю хостингу dnsimple dns.

dnsimple має декілька інших зручних функцій, таких як можливість делегувати управління наборами записів іншим користувачам dnsimple. Оскільки ми перенесли управління наборами записів з DynDNS на Route53 та маємо обліковий запис у dnsimple для надання SSL, ми перенесли реєстрацію доменів з DynDNS на dnsimple, щоб об'єднати послуги.

Деякі корисні команди рядка:

Знайти Start Of Authority для запису DNS

dig SOA dojo4.com

Переглянути сертифікат, який надає хост

openssl s_client -showcerts -connect dojo4.com:443 < /dev/null

Доступ до веб-сервера за IP-адресою, ігноруючи перевірку SSL

curl -k https://184.72.61.241 -H “Host: projects.dojo4.com”

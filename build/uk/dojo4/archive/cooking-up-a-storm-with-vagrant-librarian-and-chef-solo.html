Налаштування Apache на одній системі - це досить просто. Якщо у конфігурації є кілька віртуальних хостів, це стає трохи складніше. А якщо конфігурація включає проксі, перенаправлення URL і деяку заміну тексту, це стає ще цікавіше. Але все це не складає труднощів для досвідченого системного адміністратора.

Аж доки вам не доведеться робити це знову.

Тут на допомогу приходять інструменти, такі як Chef і Puppet. Обидва інструменти дозволяють вам оголошувати та визначати, як повинні виглядати системи та як вони повинні бути налаштовані. Обидва потужні, і обидва активно використовуються для налаштування від робочих станцій розробників до великих розгортань сотень тисяч машин.

Як і програмне забезпечення, вам все одно потрібно будувати та тестувати конфігурацію, яку ви створюєте. Ваші можливості варіюються від дуже дорогого (купувати один або кілька серверів), до помірно дорогого (запускати кілька інстансів на EC2) аж до дуже-дуже дешевого. Найбільш дешевим варіантом буде будувати та тестувати ваші рецепти на віртуальній машині(ах), що працюють на вашій системі. Тут на допомогу приходить Vagrant.

Vagrant - це обгортка навколо VirtualBox - настільної платформи віртуалізації від Oracle. Конфігурація Vagrant обертається навколо кількох простих концепцій. У вас є конфігураційний файл Vagrantfile у корені проєкту. Цей файл містить базову конфігурацію, таку як базовий бокс для проєкту та які порти потрібно відкрити для гостьової системи. І так само важливо, він може описувати, як провізіювати віртуальну машину, щоб вона відповідала вимогам вашого проєкту. Хоча Vagrant підтримує і Puppet, і Chef, я вибрав Chef-Solo для провізіонування цього проєкту.

Почати роботу з Vagrant просто. Перший крок - це встановити гем vagrant. Як і будь-який інший Ruby-проєкт, я вирішив встановити vagrant та кілька інших гемів через Bundler. Ось результат Gemfile:

source 'https://rubygems.org'
gem 'vagrant'
gem 'librarian'

Виконання bundle install встановило потрібні геми у каталозі проєкту. Швидке rbenv rehash зробило виконуваний файл vagrant доступним у моїй оболонці.

Наступний крок - налаштувати найпростішу конфігурацію Vagrant. Я виконав vagrant init і потім відредагував створений Vagrantfile, щоб він виглядав так:

Vagrant::Config.run do |config|
  # Every Vagrant virtual environment requires a box to build off of.
  config.vm.box = "ubuntu-precise-64"
  
  # The url from where the 'config.vm.box' box will be fetched if it
  # doesn't already exist on the user's system.
  config.vm.box_url = "http://files.vagrantup.com/precise64.box"
end

Це досягає двох речей. По-перше, це оголошує назву боксу, на якому ми будемо будувати, та де знайти цей бокс. Швидке vagrant up спочатку завантажить базовий бокс, імпортує його в проєкт та запустить VM. Завантаження боксу займає деякий час, але він кешується на вашій системі, тому вам не потрібно завантажувати його знову. Ви також можете повторно використовувати той самий базовий бокс, якщо вирішите зробити це в інших проєктах. Виконання vagrant halt зупиняє VM, а виконання vagrant destroy знищує саму VM.

Все ще не зробили багато для провізіювання нашої VM. Наступний крок - сказати Vagrant, як це зробити.

Librarian - це ще один чудовий гем у стилі Bundler та CocoaPods, який дозволяє вам оголошувати, які рецепти Chef ви використовуєте. У нашому випадку скорочена копія Cheffile проєкту виглядає так:

#!/usr/bin/env ruby
#^syntax detection
site 'http://community.opscode.com/api/v1'
cookbook 'apt'
cookbook 'apache2', '>= 1.0.0'
cookbook 'rbenv', :git => 'git://github.com/fnichol/chef-rbenv.git', :ref => 'v0.7.2'
cookbook 'ruby_build'
cookbook 'openssh', :git => 'git://github.com/fnichol/chef-openssh.git'
cookbook 'git'
cookbook 'build-essential'
cookbook 'vim'
cookbook 'user'
cookbook 'sudo'

Найбільша перевага полягає в тому, що вам не потрібно вендорити кукбуки у ваші проєкти. Виконання librarian-chef install завантажить кукбуки та встановить їх у каталозі cookbooks. Це також створює Cheffile.lock, щоб задокументувати, які версії кукбуків були встановлені.

Тоді я досягнув точки, де я міг сказати Vagrant використовувати Chef-Solo для провізіонування боксу, оновивши Vagrantfile:

Vagrant::Config.run do |config|
  # Every Vagrant virtual environment requires a box to build off of.
  config.vm.box = "ubuntu-precise-64"
  
  # The url from where the 'config.vm.box' box will be fetched if it
  # doesn't already exist on the user's system.
  config.vm.box_url = "http://files.vagrantup.com/precise64.box"
  
  # Enable provisioning with chef solo, specifying a cookbooks path, roles
  # path, and data_bags path (all relative to this Vagrantfile), and adding
  # some recipes and/or roles.
  config.vm.provision :chef_solo do |chef|
    chef.cookbooks_path = ["cookbooks", "site-cookbooks"]
    chef.add_recipe "apt"
    chef.add_recipe "build-essential"
    chef.add_recipe "ruby_build"
    chef.add_recipe "rbenv::system"
    chef.add_recipe "rbenv::vagrant"
    chef.add_recipe "git"
    chef.add_recipe "user"
    chef.add_recipe "vim"
    chef.add_recipe "sudo"
    chef.add_recipe "openssh"
    chef.add_recipe "apache2"
    
    chef.json = {
      authorization: {
        sudo: {
          passwordless: true,
          groups: ["admin", "sudo"]
        }
      },
      openssh: {
        permit_root_login: "no",
        password_authentication: "yes"
      },

        rubies: [ "1.9.3-p327" ],
        upgrade: true,

        gems: {
          "1.9.3-p327" => [
            { name: "bundler" },
            { name: "main" },
            { name: "map" },

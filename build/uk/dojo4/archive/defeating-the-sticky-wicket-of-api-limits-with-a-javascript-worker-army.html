@drawohara

TL;DR

коли у вас дуже багато роботи на сервері з огляду на обмеження api

передайте частину роботи клієнтським браузерам для виконання - відправте результати назад до вашої бази даних

проблема

минулого тижня я мав справу з (g-provider) обмеженнями api та надто великою кількістю геолокаційних запитів, що надходили з сервер(ів). я трохи попрацював над цим, виконуючи завдання в фоновому режимі, з періодичними повторними спробами - ви знаєте: стандартні речі...

api, про яке йде мова, обмежується за ip, а документація рекомендує скористатися цим, виконуючи більшість api-запитів на стороні клієнта - на js.

тепер, у цьому конкретному випадку мені абсолютно потрібно було, щоб результати api-запитів знаходилися на сервері, тому я знайшов цей компромісний (у більш ніж одному сенсі) рішення

рішення

припустімо, що деякі клієнти готові витратити свій cpu для заповнення даних, якими їм цікаво працювати. іншими словами, змусіть деяких ваших клієнтів виконувати завдання з js-черги завдань, щоб підготувати те, чим їм цікаво користуватися.

у моєму випадку деякі вибрані інтерфейси (ті, що цікавляться результатом цих завдань) запускають маленький iframe

далі йдуть деякі зразкові відповіді, сгруповані за категоріями, та коментарі до кожної з них

fubar doctype

  • Відсутній doctype. Це ускладнює визначення версії HTML, що може призвести до цікавих відповідей…
  • Містяться теги у стилі xhtml разом з HTML тегами.

напевно більшість пропустили те, що багато помилок залежать від оголошення doctype.
який стиль тегів можна використовувати повністю залежить від doctype.

method=delete

  • рядок 1: DELETE не підтримується в HTML формах:
  • method=delete не підтримується безпосередньо в більшості/у всіх? браузерах.
  • форми не підтримують delete у старих браузерах
  • Метод delete
  • form method='DELETE' – це не дозволено

хоча ми думаємо про http дієслова з клієнтськими api, включаючи javascript.
для більшості браузерів ви отримуєте GET та POST. це важливо, якщо ви
розглядали api на основі xhtml.

enctype=bunk

  • enctype="multipart/form-data" а не enctype="multipart/file-upload"
  • enc-type="multipart/file-upload не має сенсу, якщо наш намір – це DELETE

помилки enctype завжди псують день. помилиться і ви втратите день
відправляючи назву файлу як рядок, замість фактичного закодованого вмісту файлу.
також важливо зазначити, що enctype лише визначає, як кодується форма, і повністю ортогональний до http дієслова, яке використовується.

input=foo && input=foo

  • текстові вхідні теги у першому fieldset мають однакове ім'я
  • дублювання тегів "". Або два теги з однаковим ім'ям знову
  • 2 текстових полів з ім'ям "tags" використовується лише останнє.
  • Два входи з однаковим ім'ям 'tags'. Другий перезапише значення першого.
  • Два входи з ім'ям 'foo' (не самозакриті)
  • текстові входи з однаковим ім'ям перезапишуть щодо входів з ім'ям tags
  • подвійні призначення input name='tags'
  • подвійні призначення input name='vehicle'

багато непорозумінь щодо цього. це повністю легітимно надсилати один і той самий
параметр > 1 разів у формі. специфікація підтримує це навіть давні cgi
підтримують ідею надсилання багатьох значень під одним і тим самим ключем. це лише
конвенція, що багато контролерів скидають ці значення в хеш, де "останній виграє"

      end
      
      def [](key)

      def []=(key, value)

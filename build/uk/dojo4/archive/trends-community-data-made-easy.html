Щорічний звіт Фонду спільноти округу Боулдер Тренди надає “набір спільних вимірюваних даних для поліпшення якості життя в громаді.” Недавно Code for America співпрацював з Фондом, щоб зробити ці дані громади більш доступними. Партнерство, назване Code for Boulder, запросило dojo4 приєднатися. Після громадської сесії ідей для виявлення потенційних методів, які можна використовувати для розкриття цінного вмісту даних у звіті Тренди, dojo4 співпрацював з партнерством для створення повністю адаптивного, автономного мікросайту, який Фонд спільноти міг би легко підтримувати. Основний сайт Фонду спільноти перероблявся Vermillion Design, тому dojo4 виокремив дизайнерські шаблони з їхніх настільних макетів.

Ми почали свою роботу зі створення статичного “заповнювача” сайту, щоб визначити, які частини контенту потрібно було б регулярно оновлювати, а які були більш постійними. Зрештою, це зробило роботу з інтеграцією адміністративної частини сайту набагато швидшою.

Спочатку ми не були впевнені, як сайт буде оновлюватися: 1) за допомогою нашого улюбленого генератора статичних сайтів Middleman або 2) за допомогою нашої власної CMS на основі Rails. Але оскільки однією з умов проєкту було те, що сайт мав бути легко оновлюваним волонтерами Code for Boulder, ми вибрали варіант, який пріоризував підтримуваність непостійним, короткостроковим персоналом з різними технічними здібностями. Ми вибрали новий інструмент CMS під назвою DatoCMS, “повністю налаштовувану адміністративну область для статичних веб-сайтів”. Дійсно, Стефано Верна, засновник Dato, зв'язався з нами та пояснив, як легко інтегрувати DatoCMS, і що ще важливіше, як легко ним керувати нетехнічним користувачам. Це було ідеальним часом, і тепер усі можуть більш ефективно використовувати веб-ресурс, який повідомляє про ключові показники якості життя в громаді Боулдера.

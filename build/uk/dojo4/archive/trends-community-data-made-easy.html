@drawohara

Щорічний Community Foundation Boulder County звіт Trends надає “загальний набір вимірюваних даних для поліпшення якості життя в громаді.” Останнім часом Code for America співпрацював з Фондом, щоб зробити ці громадські дані більш доступними. Партнерство, назване Code for Boulder, запросило dojo4 взяти участь. Після громадської сесії з генерації ідей, яка допомогла виявити потенційні методи, щоб відкрити цінний вміст даних у звіті Trends, dojo4 працював з партнерством над створенням повністю адаптивного, автономного мікросайту, який Фонд громади міг би легко підтримувати. Основний сайт Фонду громади перероблявся Vermillion Design, тому dojo4 переніс дизайнерські шаблони з їхніх настільних макетів.

Ми почали свою роботу із створення статичного “замісного” сайту, щоб визначити, які частини вмісту потребують регулярного оновлення, а які більш постійні. Зрештою, це прискорило інтеграцію адміністративного бекенду сайту.

Спочатку ми не знали, як сайт буде оновлюватися: 1) за допомогою нашого улюбленого генератора статичних сайтів Middleman або 2) за допомогою нашої власної системи управління контентом на основі Rails. Але оскільки однією з умов проєкту було, щоб сайт легко оновлювався волонтерами Code for Boulder, ми вибрали варіант, який пріоризував простоту підтримки ротаційним, короткостроковим персоналом з різними технічними здібностями. Ми вибрали новий інструмент CMS під назвою DatoCMS, “повністю налаштовувану адміністративну область для статичних веб-сайтів”. Насправді, Stefano Verna, засновник Dato, зв’язався з нами та пояснив, наскільки просто інтегрувати DatoCMS і, що ще важливіше, наскільки легко ним керувати нетехнічним користувачам. Це був ідеальний час, і всі можуть більш ефективно взаємодіяти з веб-ресурсом, який повідомляє про ключові показники якості життя в громаді Боулдера.

Написано з Steve Bailey, який виконав більшу частину роботи над цим проектом.

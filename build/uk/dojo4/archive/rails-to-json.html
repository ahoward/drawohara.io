Rails, включаючи rails3, має цей чудовий шматок коду, пов'язаний із наданням методу “to_json” на вбудованих об'єктах:

# Хак для завантаження gem json першим, щоб ми могли перевизначити його to_json.
    begin
      require 'json'
    rescue LoadError
    end
    # Gem JSON додає декілька модулів до базових класів Ruby, які містять визначення :to_json, перевизначаючи
    # їх стандартну поведінку. Тому нам потрібно визначити базовий метод to_json у всіх них,
    # інакше вони завжди будуть використовувати реалізацію gem to_json, яка є несумісною з
    # декількома випадками (наприклад, реалізація JSON для Hash не працює) та спадковістю,
    # наслідком чого класи, такі як ActiveSupport::OrderedHash, не можуть бути серіалізовані в json.
    [Object, Array, FalseClass, Float, Hash, Integer, NilClass, String, TrueClass].each do |klass|
      klass.class_eval <<-RUBY, __FILE__, __LINE__
# Серіалізує об'єкт у JSON (JavaScript Object Notation). Див. www.json.org для отримання додаткової інформації.
def to_json(options = nil)
ActiveSupport::JSON.encode(self, options)
end

Що це ефективно робить - це замінює метод “to_json”, який надається gem json, на той, який active_support вважає кращим. Коментарі про те, що json зламаний, застарілі, і це було хаком з самого початку. У моєму випадку мені дуже подобається такий підхід у моєму /api контролері:

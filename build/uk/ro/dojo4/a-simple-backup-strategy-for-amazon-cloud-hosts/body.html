Протягом років Dojo4 створила дійсно чудовий стек розробки та хостингу веб-застосунків. Хмара Amazon Web Services (AWS) складає основу цього стеку. Гнучкість та зручність томі EBS (Elastic Block Store) є переконливими, і ми використовуємо їх для системних томів, а також для томів даних для розгортання додатків та збереження баз даних на наших хмарних комп'ютерах Elastic Compute Cloud (EC2). Деякий час тому, наприкінці розмови про архітектуру хостингу, Ара недбало зауважив: "О, і періодичні резервні копії дисків мають відбуватися автоматично кожного разу, коли ми запускаємо новий екземпляр EC2". Тож я почав працювати над невеликим скриптом під назвою DREBS (Відновлення після аварій для Elastic Block Store). DREBS можна інтегрувати в ваш Amazon Machine Image (AMI) або налаштувати в вашому Chef чи інших механізмах хмарного провізіонінгу. DREBS можна налаштувати для створення періодичних знімків томів EBS, підключених до екземпляра EC2, на якому він працює. Він підтримує гнучкі стратегії збереження знімків, а також завдання до та після створення знімків для таких операцій, як збереження баз даних на диск для узгоджених резервних копій баз даних. Ознайомтесь з DREBS і дайте нам знати, що ви думаєте.

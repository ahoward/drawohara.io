Я хотів би поділитися швидким і легким методом для тестування міграцій Rails при використанні бази даних MongoDB. Гнучкість mongo і ruby роблять це досить простим. В цьому прикладі ми будемо перейменовувати поле в нашій прикладній колекції "books" з "isbn" на "book_number". Це досить поширений тип міграції, і як тільки ви зрозумієте цей простий випадок, більш складні міграції слідуватимуть тому ж патерну. Спочатку, створімо наш шаблон скрипту міграції з відміткою часу.

rails generate migration rename_isbn_to_book_number

Потім відредагуємо згенерований файл під db/migrate. Ми розділимо наш код на декілька секцій, як показано нижче.

  1. Кілька констант класу, спільних для всіх методів
  2. Метод self.up, який виконує пряму міграцію
  3. Допоміжний метод для створення прикладних даних для тестування прямої міграції
  4. Метод self.down, який виконує відкат
  5. Допоміжний метод для створення прикладних даних (якщо потрібно) для тестування відкату

Спочатку визначимо кілька констант, які використовуватимуть наші методи. Ми зберігаємо хеш опцій оновлення mongo MultiUpdate для зручності, оскільки більшість операцій оновлення міграції хочуть upsert false (не створювати нові документи), multi true (оновлювати всі відповідні документи) і safe true.

Отже, тепер у нас є окрема, добре зрозуміла тестова колекція, готова для тестування нашої простої міграції. Давайте напишемо наш метод up, щоб виконати міграцію.

def up
  #Ми хочемо перейменувати поле book.isbn на book.book_number
  Collection.find({"isbn" => {"$exists" => 1}}).each do |book|
    update_op = {
      "$unset" => {"isbn" => 1},
      "$set" => {"book_number" => book["isbn"]}
    }
  Collection.update({"_id" => book["_id"]}, update_op, MultiUpdate)
end

Це має дати вам дуже швидкий спосіб експериментувати та довести ваш міграційний код до робочого стану. Mongo має деякі потужні можливості запитів та модифікацій, які можуть робити дивовижні речі, і легкий спосіб провести кілька спроб і помилок буде корисним. Якщо ви заплутаєтесь у тестових даних, ви можете використати Collection.drop, щоб отримати чистий стан. Ось фінальний міграційний код для довідки. Не забудьте видалити тестову константу колекції та видалити тестову колекцію з вашої бази даних, коли будете готові почати запускати свій код для реального використання з rake db:migrate.

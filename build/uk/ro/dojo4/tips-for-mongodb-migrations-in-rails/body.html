Я хотів би поділитися швидким і простим методом для тестування міграцій Rails при використанні бази даних MongoDB. Гнучкість mongo та ruby робить це досить простим. У цьому прикладі ми перейменуємо поле в нашій прикладній колекції "books" з "isbn" на "book_number". Це досить поширений тип міграції, і коли ви звикнете до цього простого випадку, складніші міграції дотримуються того ж шаблону. Спочатку створімо наш міграційний скрипт з відміткою часу.

rails generate migration rename_isbn_to_book_number

Потім відредагуємо файл, який згенерувався в db/migrate. Розділимо наш код на декілька розділів таким чином.

  1. Деякі константи класу, які використовуються всіма методами
  2. Метод self.up, який виконує пряму міграцію
  3. Допоміжний метод для створення тестових даних для перевірки прямої міграції
  4. Метод self.down, який виконує відкат
  5. Допоміжний метод для створення тестових даних (якщо потрібно) для перевірки відкату

Спочатку визначимо деякі константи, які використовуватимуть наші методи. Ми зберігаємо хеш опцій оновлення mongo MultiUpdate для зручності, оскільки більшість операцій оновлення міграції хочуть upsert false (не створювати нові документи), multi true (оновити всі відповідні документи) та safe true.

Також визначимо константу для назви колекції. Для реальної міграції колекція називається "books", але для тестування ми створимо нову колекцію під назвою "test_books_migration", коли ми розроблятимемо свій код.

Тепер у нас є окрема, добре зрозуміла тестова колекція для перевірки нашої простої міграції. Напишемо наш метод up, щоб виконати міграцію.

def up
  #Ми хочемо перейменувати поле isbn на book_number
  Collection.find({"isbn" => {"$exists" => 1}}).each do |book|
    update_op = {
      "$unset" => {"isbn" => 1},
      "$set" => {"book_number" => book["isbn"]}
    }
  Collection.update({"_id" => book["_id"]}, update_op, MultiUpdate)
end

Ми можемо вставити це в консоль і запустити, щоб перевірити нашу міграцію. Ми можемо перевірити результати за допомогою show_colletion. Якщо ми хочемо перевірити інші записи для відкату, ми можемо створити метод mock_data_for_testing_down.

Це має дати вам дуже швидкий спосіб експериментувати та запускати ваш міграційний код. Mongo має деякі потужні можливості запитів і модифікацій, які можуть дивувати, і зручний спосіб виконати деякі проби і помилки дуже корисний. Якщо ви заплутаєте свої тестові дані, ви можете використати Collection.drop, щоб отримати чистий аркуш. Ось фінальний міграційний код для довідки. Не забудьте видалити константу тестової колекції та видалити тестову колекцію з вашої бази даних, коли ви будете готові почати запускати свій код у реальному режимі за допомогою rake db:migrate.

  Collection = db.collection("books") #Фінальний продакшн-код

        "$set" => {"book_number" => book["isbn"]}

    end
  end

  def down

  #2. Запустіть mock_data_for_testing_up
  #3. Запустіть тіло self.up

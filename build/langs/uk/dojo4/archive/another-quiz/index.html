приклади відповідей

  • Немає документа. Це ускладнює визначення типу HTML, що може призвести до цікавих відповідей…

  • Змішує xhtml стиль пустих тегів з HTML тегами.

  • Непослідовне використання лапок. Обидва варіанти працюють, обирайте один…

  • рядок 1: DELETE не підтримується в HTML формах:
  • method=delete не підтримується безпосередньо в більшості/всіх? браузерах.
  • форми не підтримують delete в старих браузерах
  • Method delete

  • form method=’DELETE’ – це не дозволено

  • enctype=”multipart/form-data” а не enctype=”multipart/file-upload”

  • enc-type=”multipart/file-upload не має сенсу, якщо наша мета – DELETE

  • текстові вхідні теги в першому fieldset мають назву
  • дубльовані “” теги. Або два теги з однаковою назвою знову
  • 2 текстові поля з назвою “tags” використовуватиметься лише останнє.
  • Два входи з однаковою назвою ‘tags’. Другий перепише значення першого.
  • Два входи з назвою ‘foo’ (не самозакриті)
  • текстові входи з однаковими назвами перевизначатьмуть: входи з назвою tags
  • дубльовані призначення input name=’tags’

  • дубльовані призначення input name=’vehicle’

  • Немає для на мітці ‘Bar:’.

  • неприв’язана мітка: (мітка для чого?)

  • readonly вхід без значення безглуздий

  • action=”./” здається мені поганою практикою

  • форма дії недійсна як описувач файлу unix – щоб відправити до поточної url видалити дію повністю, оскільки це за замовчуванням

  • завантаження файлу не може мати атрибуту значення (погана безпека)
  • вкладення fieldset
  • входи файлів не приймають значень за замовчуванням – користувач повинен вибрати

  • він автоматично вибирає ваш ~/.bashrc файл для завантаження

  • disabled повинні мати лапки навколо (також погана суміш одинарних і подвійних лапок протягом усього)
  • Незакриті опції для випадаючого списку dropdown і не включений текст

  • відсутні лапки навколо атрибуту disabled у неправильній декларації входу password

  • Немає типу на полі password (також немає на полі email, але там це менш важливо)
  • вхід password повинен бути типу password

  • повинно бути <input type=’email’ name=’email’ …>

  • disabled=disabled повинно бути disabled

  • Немає тексту для жіночої опції, ні кнопки радіо для радіокнопок ‘Sex’, радіокнопки не налаштовані правильно для відправки значення / перемикання в групі. Все це погано в цій області.
  • fieldset “sex” радіокнопки побудовані неправильно. повинно бути Male – теги використовуються з випадаючими списками, а не радіокнопками. Всі радіокнопки в групі повинні мати однаковий атрибут name

  • радіо входи не приймають опції, але повинні мати однакову назву з різними значеннями

  • кнопки повинні бути відкриті та закриті – відправка повинна бути входом замість
  • checkbox з однаковими назвами перевизначатьмуть: checkbox входи з назвою vehicle

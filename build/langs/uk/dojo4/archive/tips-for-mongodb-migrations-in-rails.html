Я хотів би поділитися швидким і легким методом для тестування міграцій Rails при використанні бази даних MongoDB. Гнучкість mongo та ruby робить це досить простим. У цьому прикладі ми перейменуємо поле в нашій прикладній колекції "books" з "isbn" на "book_number". Це досить поширений тип міграції, і як тільки ви освоїте цей простий випадок, більш складні міграції дотримуються того ж шаблону. Спочатку створімо наш шаблон скрипта міграції з таймштампом.

rails generate migration rename_isbn_to_book_number

Потім ми відредагуємо файл, який згенерувався під db/migrate. Ми розділимо наш код на декілька розділів, як показано нижче.

  1. Кілька констант класу, які використовуються всіма методами
  2. Метод self.up, який виконує пряму міграцію
  3. Допоміжний метод для створення зразкових даних для тестування прямої міграції
  4. Метод self.down, який виконує відкат
  5. Допоміжний метод для створення зразкових даних (якщо потрібно) для тестування відкату

Спочатку визначимо деякі константи, які будуть використовуватися нашими методами. Ми зберігаємо хеш опцій оновлення mongo MultiUpdate для зручності, оскільки більшість операцій оновлення міграції не хочуть створювати нові документи (upsert false), оновлювати всі відповідні документи (multi true) і безпечні (safe true).

Потім ми також визначаємо константу для назви колекції. Для реальної міграції колекція - "books", але для тестування ми створимо нову колекцію під назвою "test_books_migration", коли розроблятимемо наш код.

Потім почнемо писати невеликий допоміжний метод для заповнення нашої тестової колекції фейковими документами, які нагадують те, що ми очікуємо побачити у виробництві, але зосереджуючись лише на полях, що стосуються міграції. Додайте цей метод до вашого класу міграції.

def mock_data_for_testing_up

end

def show_collection

Це створить 3 фейкових документа, які ми можна використовувати для тестування. Ми розміщуємо цей код у методі, щоб його було легко перезапустити, коли ми коригуємо і тестуємо наш код міграції. Для складних міграцій може знадобитися багато раундів коригувань, щоб врахувати всі граничні випадки.

Тепер у нас є окрема, добре зрозуміла тестова колекція, готова для тестування нашої простої міграції. Давайте напишемо наш метод up. Для виконання міграції.

def up
  #Ми хочемо перейменувати поле book.isbn на book.book_number
  Collection.find({"isbn" => {"$exists" => 1}}).each do |book|
    update_op = {
      "$unset" => {"isbn" => 1},
      "$set" => {"book_number" => book["isbn"]}
    }
  Collection.update({"_id" => book["_id"]}, update_op, MultiUpdate)
end

Ми можемо вставити це в консоль і запустити його, щоб протестувати нашу міграцію. Ми можемо перевірити результати за допомогою show_colletion. Якщо ми хочемо протестувати інші записи для відкату, ми можемо створити метод mock_data_for_testing_down.

Це має дати вам дуже швидкий спосіб експериментувати і запускати ваш код міграції. Mongo має деякі передові можливості запитів і модифікацій, які можуть робити дивовижні речі, і простий спосіб робити деякі проби і помилки є корисним. Якщо ви заплутаєте свої тестові дані, ви можете використовувати Collection.drop, щоб отримати чистий аркуш. Ось фінальний код міграції для довідки. Не забудьте видалити константу тестової колекції та видалити тестову колекцію з вашої бази даних, коли ви будете готові почати запускати свій код для реального використання з rake db:migrate.

class RenameIsbnToBookNumber < Mongoid::Migration
  MultiUpdate = {:upsert => false, :multi => true, :safe=>true}
  Collection = db.collection("books") #Фінальний продакшн-код

  def up
    #Ми хочемо перейменувати поле book.isbn на book.book_number
    Collection.find({"isbn" => {"$exists" => 1}}).each do |book|
      update_op = {
        "$unset" => {"isbn" => 1},
        "$set" => {"book_number" => book["isbn"]}
      }
      Collection.update({"_id" => book["_id"]}, update_op, MultiUpdate)
    end
  end

  def down
    #Ми хочемо перейменувати поле book.book_number на book.isbn
    Collection.find({"book_number" => {"$exists" => 1}}).each do |book|
      update_op = {
        "$unset" => {"book_number" => 1},
        "$set" => {"isbn" => book["book_number"]}
      }
      Collection.update({"_id" => book["_id"]}, update_op, MultiUpdate)
    end
  end

  #Ці методи не викликаються міграцією. Тільки для ручного тестування
  #для тестування (скопіюйте/вставте звідси в консоль)
  #1. Встановіть константу MultiUpdate. Налаштуйте "Collection", щоб це була тестова колекція
  #2. Скопіюйте/вставте 2 методи нижче
  #2. Запустіть mock_data_for_testing_up
  #3. Запустіть тіло self.up
  #3b. Опціонально запустіть тіло self.up знову, щоб переконатися, що воно є ідемпотентним
  def mock_data_for_testing_up
    3.times {|number| Collection.insert({"isbn" => "#{number}"})}
  end

  def show_collection
    Collection.find({}).each {|_| puts _}
  end
end

З будь-яким проєктом я запускаю кілька програм одночасно під час розробки. Це зазвичай призводить до кількох вкладок у додатку Terminal - одна для локального сервера, одна для бази даних, одна для клієнтського додатку та одна для git. (Наразі я використовую GUI редактор (ST2), але можливо, я скоро повернуся до vim...) Ця конфігурація з кількома вкладками повторюється для кожного проєкту, над яким я працюю, і я зазвичай перемикаюся між кількома проєктами кілька разів на день.

З цією багатовікловою конфігурацією, коли мені доводиться перемикатися на інший проєкт, я маю або відкривати нове вікно Terminal із власними вкладками для цього проєкту та мати справу з більшою кількістю вікон, ніж потрібно (не варіант для мінімаліста, як я), або закривати всі вкладки, які я відкрив, лише для того, щоб знову відкрити їх усі, коли я повертаюся до цього проєкту.

Ну, як і більшість днів, я дізнався розумне рішення цієї проблеми від мудрого та всемогутнього лідера dojo4, Ара. Він використовує програму GNU Screen. У межах однієї вкладки Terminal, Screen дозволяє відкривати кілька "вікон" (тобто вкладок) у тій самій вкладці Terminal. Ось як це використовувати:

# створення та повторне підключення до сесії
screen -S [name] = створити названу екранну сесію
screen -D -R [name] = повторно підключитися до названої екранної сесії та відключити інших, якщо необхідно
screen -list = перелік екранів

# (вихід із усіх вікон у сесії екрану завершить цю сесію. Однак сесія екрану є запущеним процесом, тому ви можете вручну завершити її.)

# в межах сесії екрану:
ctrl-A + ? = допомога
ctrl-A + " = перелік вікон
ctrl-A + c = створити нове вікно
ctrl-A + n = наступне вікно
ctrl-A + p = попереднє вікно
ctrl-A + A = перейменувати вікно

# псевдоніми, які ми використовуємо
alias sl='screen -list '
alias sdr='screen -d -r '
alias s='screen -D -R '
alias ss='screen -S'

sdr [name] = повторно підключитися до названої екранної сесії та відключити інших, якщо необхідно
ss [name] = створити названу екранну сесію
sl = перелік екранів

Ще одна дуже крута функція screen полягає в тому, що коли ви відключаєтеся від сесії екрану, усі ваші вікна залишаються запущеними, і коли ви повторно підключаєтеся до сесії екрану, у вас знову є усі ваші вікна в розпорядженні.

Коли ми обговорювали це в нашому офісі, Раян Кук створив гарненький Ruby допоміжник для створення екранних сесій, названих за вашою поточною директорією.

Так що мій робочий процес змінився від використання кількох вкладок у Terminal до використання однієї вкладки в Terminal, де я повторно підключаюся до сесії екрану для кожного проєкту, над яким працюю. Це дозволяє мені дуже швидко перемикатися між проєктами та припинити марнувати час на повторне налаштування та закриття всіх вкладок Terminal, які я використовую для проєкту.

Тепер мені просто потрібно перевести більше речей на використання терміналу (наприклад, мій редактор), щоб дійсно максимально використовувати screen...

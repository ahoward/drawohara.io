Нещодавно мені довелося заглиблюватися в AngularJS на сайті Rails, який ми будували для клієнта. Крім вмісту сторінки, у клієнта є певні сторінки, які відображають заголовки, що ведуть до зовнішніх джерел, але втягуються в наш стек Rails через періодично запускану задачу Rake. Сайт повинен сильно кешуватися за допомогою CDN, наприклад, CloudFront, але щоб заголовки залишалися свіжими, ми використовували AngularJS (тому що клієнт був знайомий з ним, тому проще підтримувати для них) для запитів до стеку Rails безпосередньо для отримання останніх заголовків.

Це вимагало робити речі міждоменним чином, оскільки домен сайту буде вказувати на CloudFront. Я ще не писав API Rails, який повертав би JSONP, або сервіс AngularJS, тому мені довелося досліджувати, і ось що працювало для мене. (Ви також можете прочитати гіст.)

Примітка: якщо ви не знайомі з JSONP, це мало сенсу. Прочитайте це.

Сервіс AngularJS

AngularJS має концепцію сервісів, які є просто синглтонами, які можна використовувати де завгодно. У нашому випадку ви можете уявити наш сервіс заголовків як клієнт API заголовків, який інкапсулює наші виклики JSONP у простішому та централізованому синтаксисі. AngularJS надає деякі сервіси за замовчуванням, але ви можете написати власні. Насправді, ми могли б використовувати вбудований $resource сервіс замість написання власного, однак у мене були проблеми з використанням JSONP, і я хотів краще зрозуміти низькорівневі компоненти AngularJS, тому я написав власний:

angular.module('app')
  .factory('HeadlineService', ['$http',
    function($http) {
      'use strict';

      var BASE_URL = "<%= [App.settings.api_base_url, '/services/headlines'].join %>",
          CALLBACK_STRING = "?callback=JSON_CALLBACK";

      return {
        getHeadlines: function(){
          return $http.jsonp(BASE_URL + CALLBACK_STRING)
        },
        getHeadlineForId: function(id){
          return $http.jsonp(BASE_URL + "/" + id + ".json" + CALLBACK_STRING)
        }
      }

    }
  ]);

Вищевказаний сервіс використовує вбудований $http сервіс, який просто надає AJAX-обертку для виконання запитів до нашого бекенду Rails для отримання заголовків. Декілька речей, на які слід звернути увагу:

  • Він повертає об’єкт з 2 функціями: getHeadlines та getHeadlineForId. Ми можемо використовувати ці функції як скорочення для спілкування з API Rails у наших контролерах AngularJS.
  • Він використовує JSONP. Поміть, що ми використовуємо $http.jsonp, і ми передаємо рядок запиту ?callback=JSON_CALLBACK. AngularJS замінює рядок JSON_CALLBACK іменем функції зворотного виклику, яку він створює для вас.
  • Він використовує інший домен. Ну, це очікувано, оскільки ми використовуємо JSONP, але ми досягаємо цього, додавши розширення .erb до файлу та використовуючи те, що ми встановили для App.settings.api_base_url у додатку Rails як домен, з яким повинен спілкуватися AngularJS. Пам'ятайте, що це необхідно для обходу кешування CDN, щоб наші заголовки завжди були найбільш актуальними.

Контролер AngularJS

З побудованим HeadlinesService ми тепер можемо використовувати дві функції, які він повертає, у нашому додатку AngularJS:

angular.module('app')
  .controller('HeadlinesCtrl', ['$scope', 'HeadlineService',
    function($scope, HeadlineService) {
      'use strict';
 
      HeadlineService.getHeadlines().success(function(data, status, headers, config){
        $scope.headlines = data;
        
        HeadlineService.getHeadlineForId(data[0].id).success(function(data){
          $scope.single_headline = data;
        }).error(function(error, status, headers, config){
          alert('щось пішло не так при отриманні заголовку');
        });
      }).error(function(error, status, headers, config){
        alert('щось пішло не так при отриманні всіх заголовків');
      });
    }

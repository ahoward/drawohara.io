Хостинг, DNS та HTTPS часто вимагають взаємодії більше ніж кількох провайдерів. Зазвичай це стає незручним, якщо не керувати ефективно.

DNS вимагає як управління записами, так і реєстрації доменів. Для обох цих завдань Dojo4 використовує DyDNS протягом багатьох років.

Dojo4 зазвичай використовує Amazon AWS EC2 хмарні комп’ютери для хостингу та, відповідно, схильні використовувати AWS ELB балансувальники навантаження перед цими веб-хостами. На жаль, Amazon не надає постійних IP-адрес для інстансів ELB, тому єдиним способом створити записи доменів другого рівня, що вказують на інстанси ELB, є використання Amazon AWS Route53 для управління записами, який має спеціальний запис alias для цієї мети. Тому ми схильні використовувати Route53 для управління записами DNS.

Через спосіб, яким здійснюються TLS-handshakes у linux/apache, ви обмежені кількістю SSL-сертифікатів, які можуть бути надані кількістю IP-адрес на хості. З EC2 це фактично один сертифікат, якщо ви не використовуєте VPC. Dojo4 часто використовує ELB для надання SSL як засіб для обходу цього.

DNS та HTTPS тісно пов’язані, оскільки домени та субдомени, які потрібно захистити, повинні бути перераховані до надання сертифіката. Часто найзручнішою та гнучкою стратегією є отримання сертифіката з підстановою знака. Однак не всі видавці сертифікатів з підстановкою знака включають базовий домен у сертифікат (тобто ви можете використовувати сертифікат лише для захисту *.somedomain.com, а не somedomain.com). Сертифікат з підстановкою знака від dnsimple.com на $100/рік, виданий Comodo та включаючи базовий домен, є хорошою вартістю навіть з вартістю хостингу dnsimple dns на $96 ($8/місяць * 12 місяців).

dnsimple має деякі інші зручні функції, такі як можливість делегувати управління наборами записів іншим обліковим записам користувачів dnsimple. Оскільки ми перенесли управління наборами записів з DynDNS до Route53 та маємо обліковий запис на dnsimple для надання SSL, ми перенесли реєстрацію доменів з DynDNS до dnsimple для консолідації послуг.

Корисні команди командного рядка:

Знайти Start Of Authority для запису DNS

dig SOA dojo4.com

Переглянути сертифікат, який надає хост

openssl s_client -showcerts -connect dojo4.com:443 < /dev/null

Доступ до веб-сервера за IP, ігноруючи перевірку SSL

curl -k https://184.72.61.241 -H “Host: projects.dojo4.com”

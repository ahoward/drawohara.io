я часто говорив, що завантаження файлів - це самозавдана DOS-атака. Безперечно, немає потреби далеко ходити, щоб знайти приклади, коли, здавалося б, невелика функція програми спричиняє хаос на серверах програм: https://www.google.com/search?q=heroku+image+magick+timed+out

У @dojo4 ми використовуємо можливість Image Magick встановлювати обмеження через середовище як першу лінію захисту, щоб процеси magick вели себе правильно.

спочатку трохи підготовки/контексту: ми часто розгортаємо пару файлів RAILS_ROOT/env.yml та RAILS_ROOT/env.rb, які програма Rails завантажує при запуску. Вони не містять жодних залежностей Rails і можуть використовуватися у фонових скриптах для швидкого розуміння розгорнутої конфігурації Rails без необхідності завантажувати всю програму Rails.

перший файл - це простий файл yaml з парами ключ=значення.

# файл: RAILS_ROOT/config/env.yml
# RAILS_ENV : production
# RAILS_STAGE : stage
# APP_RUBY : /usr/local/rbenv/versions/1.9.3-p0/bin/ruby
APP_RUBY_VERSION : 1.9.3
# APP_PORT : 4201
# APP_USER : dojo4
# APP_GROUP : dojo4

другий файл насправді є тим, який потрібно завантажити. Він завантажує цей файл і має динамічний вплив на встановлення додаткових змінних середовища

# файл: RAILS_ROOT/env.rb
#
# спочатку ми включаємо будь-які налаштування середовища, знайдені в
#
# RAILS_ROOT/env.yml
#
# уважно, щоб не перекривати жодні вручну встановлені змінні ENV. файл env.yml
# зазвичай створюється під час розгортання cap.
#
  require 'erb'
  require 'yaml'
  require 'rbconfig'
  
  env_yml = File.expand_path('../env.yml', __FILE__)
  
  if test(?s, env_yml)
    buf = IO.read(env_yml)
    expanded = ERB.new(buf).result(binding)
    config = YAML.load(expanded)
    config.each{|key, val| ENV[key.to_s] ||= val.to_s} if config.is_a?(Hash)
  end

що нас цікавить тут, так це розділ, де налаштовується середовище imagemagick.

ось він у всій своїй повноті - коментарі повинні надати контекст і пояснення:

# переконайтесь, що RAILS_* встановлені
#
  ENV['RAILS_ENV'] ||= 'development'
  ENV['RAILS_ROOT'] ||= File.dirname(File.dirname(__FILE__))
# встановіть середовище Imagick Magick
#
# посилання: http://www.imagemagick.org/script/resources.php
#
# використовуйте команду системи "convert -list resource" в консолі для перегляду

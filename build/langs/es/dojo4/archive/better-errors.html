Una de las gemas de rails que hemos estado usando en nuestras aplicaciones más recientes es Better Errors, que permite ser dejado en un REPL similar a pry y una página de error más agradable cuando rails lanza una excepción durante el proceso de solicitud del navegador.


(Fuente de la imagen)

Esto es útil para nuestros desarrolladores porque, en lugar de obtener una página de error estática, ir a buscar en el código dónde poner un binding.pry, y volver a ejecutar el código propenso a errores, Better Errors carga una página donde podemos interactuar con el código allí mismo, lo que nos permite encontrar el problema más rápido, escribir una solución más rápido e iterar más rápido.

Una de las características recientes que acabamos de descubrir es la capacidad de ver el último error rpc. Tenemos una pauta de llamada ajax a un método rpc en nuestra aplicación de rails cuando solo necesitamos obtener alguna información o publicar un objeto simple como favorito a un usuario o algo así. El código que se ejecuta en nuestro servidor de rails que maneja esa solicitud ajax no hará que todo el servidor explote e invoque automáticamente Mejores Errores, por lo que la solución alternativa es ir a una URL especial que Mejores Errores expone para mostrar el último error lanzado en nuestro código de rails: http://localhost:3000/__better_errors.

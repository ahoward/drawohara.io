Con cualquier proyecto, estoy ejecutando varios programas a la vez durante el proceso de desarrollo. Esto generalmente resulta en varias pestañas en la aplicación Terminal: una para mi servidor local, una para mi base de datos, una para mi aplicación cliente y una para git. (Actualmente uso un editor GUI (ST2), pero puede que vuelva a usar vim pronto...) Esta configuración de múltiples pestañas se repite para cada proyecto en el que estoy trabajando, y generalmente cambio entre múltiples proyectos varias veces al día.

Con esta configuración de múltiples pestañas, cuando tengo que cambiar el proyecto en el que estoy trabajando, tengo que abrir una nueva ventana de Terminal con sus propias pestañas para ese proyecto y lidiar con tener más ventanas abiertas de las necesarias (no es una opción para un minimalista como yo) o cerrar todas las pestañas que tengo abiertas, solo para volver a abrirlas todas cuando vuelva a cambiar a ese proyecto.

Bueno, como la mayoría de los días, aprendí una solución inteligente a este problema del sabio y todopoderoso líder de dojo4, Ara. Él usa el programa GNU Screen. Dentro de una sola pestaña de Terminal, Screen te permite abrir múltiples "ventanas" (es decir, pestañas) dentro de la misma pestaña de Terminal. Aquí tienes cómo usarlo:

# creando y volviendo a adjuntar a una sesión
screen -S [nombre] = crea una pantalla con nombre
screen -D -R [nombre] = vuelve a adjuntar a una pantalla con nombre y desconecta a otras personas si es necesario
screen -list = lista de pantallas

# (salir de todas las ventanas en una sesión de pantalla saldrá de esa sesión. Sin embargo, una sesión de pantalla es un proceso en ejecución, por lo que puedes matarlo manualmente.)

# dentro de una sesión de pantalla:
ctrl-A + ? = ayuda
ctrl-A + " = listar ventanas
ctrl-A + c = crear nueva ventana
ctrl-A + n = siguiente ventana
ctrl-A + p = ventana anterior
ctrl-A + A = renombrar una ventana

# alias que usamos
alias sl='screen -list '
alias sdr='screen -d -r '
alias s='screen -D -R '
alias ss='screen -S'

sdr [nombre] = vuelve a adjuntar a una pantalla con nombre y desconecta a otras personas si es necesario
ss [nombre] = crea una pantalla con nombre
sl = lista de pantallas

Otra característica realmente genial de screen es que cuando te desconectas de una sesión de pantalla, todas tus ventanas siguen funcionando, y cuando vuelves a conectarte a la sesión de pantalla, tienes todas tus ventanas de nuevo a tu disposición.

Cuando discutimos esto en nuestra oficina, Ryan Cook creó un pequeño ayudante de Ruby para crear sesiones de pantalla que se llaman como tu directorio actual.

Entonces, mi flujo de trabajo ha cambiado de tener múltiples pestañas abiertas en Terminal a una sola pestaña en Terminal donde me vuelvo a conectar a una sesión de pantalla para cada proyecto en el que estoy trabajando. Eso me permite cambiar de proyecto extremadamente rápido y dejar de perder tiempo volviendo a configurar y desmontar todas las pestañas de Terminal que estoy usando para un proyecto.

Ahora solo tengo que convertir más cosas al uso de la terminal (como mi editor) para aprovechar realmente screen...

  • Comillas inconsistentes. Ambas funcionan, elige una ya…

  • línea 1: DELETE no es una acción compatible con los formularios HTML:
  • method=delete no es compatible directamente en la mayoría/todos? los navegadores.
  • los formularios no admiten delete en navegadores antiguos
  • Método delete

  • form method=’DELETE’ – eso no está permitido

  • enctype=”multipart/form-data” no enctype=”multipart/file-upload”

  • enc-type=”multipart/file-upload no tiene sentido si nuestra intención es ELIMINAR

  • las etiquetas de entrada de texto en el primer fieldset tienen el nombre
  • etiquetas “” duplicadas. O dos etiquetas con el mismo nombre nuevamente
  • 2 campos de texto llamados “tags” solo se usará el último.
  • Dos entradas con el mismo nombre 'tags'. La segunda sobrescribiría el valor de la 1ª.
  • Dos entradas con el nombre 'foo' (no autocerrado)
  • las entradas de texto con el mismo nombre sobrescribirían re: entradas con el nombre tags
  • asignaciones duplicadas a input name=’tags’

  • asignaciones duplicadas a input name=’vehicle’

  • No hay for en la etiqueta ‘Bar:’.

  • etiqueta no vinculada: (etiqueta para qué?)

  • entrada readonly sin valor es inútil

  • action=”./” me parece una mala práctica

  • la acción del formulario no es válida como descriptor de ruta de archivo unix – para enviar a la URL actual, elimina la acción por completo ya que esa es la configuración predeterminada

  • la carga de archivos no puede tener un atributo de valor (mala seguridad)
  • Anidamiento de fieldset
  • las entradas de archivo no aceptan valores predeterminados – el usuario debe seleccionar

  • disabled debe tener comillas alrededor (también una mala mezcla de comillas simples y dobles en todo)

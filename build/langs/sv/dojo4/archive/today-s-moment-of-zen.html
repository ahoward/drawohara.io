Den här utgåvan, To Wash It All Away, besvikar inte.

Det jag lärde mig idag om JavaScript från James.

“\r\n\t” == false

Vad fan? Jag måste testa det.

    % node
    > "\r\n\t" == false
true

Och naturligtvis, förklaringen från James:

Här är varför: webbläsaren upptäcker att de två operanderna har olika typer, så den konverterar false till 0 och försöker jämförelsen igen. Operanderna har fortfarande olika typer (sträng och nummer), så webbläsaren tvingar “\r\n\t” till numret 0, för någon anledning är ett icke-noll antal tecken lika med 0. Voila—0 är lika med 0! FANTASISKT. Den förklaringen var som handlingen i Inception, men den inplanterade idén var “rättheten i ditt program har tvingats till false.”

Verkligen, jag måste bara dela med mig.

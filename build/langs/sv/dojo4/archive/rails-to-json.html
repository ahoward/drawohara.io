rails, inklusive rails3, har denna härliga bit av kod som är relaterad till att tillhandahålla en “till_json”-metod på inbyggda funktioner:

# Hack för att ladda json-gemmet först så att vi kan skriva över dess till_json.
    begin
      require 'json'
    rescue LoadError
    end
    # JSON-gemmet lägger till några moduler till Ruby-kärnklasser som innehåller :till_json-definitionen, skriver om
    # deras standardbeteende. Det sades, vi behöver definiera den grundläggande till_json-metoden i alla av dem,
    # annars kommer de alltid att använda till_json-gemmets implementation, vilket är bakåtkompatibelt i
    # flera fall (till exempel fungerar inte JSON-implementeringen för Hash med arv)
    # och som ett resultat kan klasser som ActiveSupport::OrderedHash inte serialiseras till json.
    [Object, Array, FalseClass, Float, Hash, Integer, NilClass, String, TrueClass].each do |klass|
      klass.class_eval <<-RUBY, __FILE__, __LINE__
# Dumper objekt i JSON (JavaScript Object Notation). Se www.json.org för mer info.
def till_json(options = nil)
ActiveSupport::JSON.encode(self, options)
end
RUBY
    end

vad detta effektivt gör är att överskriva “till_json”-metoden som json-gemmet tillhandahåller med en som aktiv_support anser är bättre. Kommentaren om att json är trasig är oaktuell och detta var ett hack från början. I mitt fall gillar jag verkligen den här typen av saker i min /api-kontrollant:

  def till_json(objekt)
    if Rails.env.production?
      JSON.generate(objekt)
    else
      JSON.pretty_generate(objekt)
    end
  end

eftersom det gör felsökningen mycket enklare. Så här gjorde jag det i en rails3-app -

första steget var att få rails3 att stödja konceptet med “förinitialisering” som gamla rails-versioner brukade ha:

    ### fil: config/förinitialisering.rb
    dirname = File.dirname(File.expand_path(__FILE__))
    glob = File.join(dirname, 'förinitialiseringar', '**/**.rb')
    förinitialiseringar = Dir.glob(glob)
    förinitialiseringar.each{|förinitialisering| Kernel.load(förinitialisering)}

därefter lade jag till den här förinitialiseringen:

### fil: config/förinitialiseringar/un_fuck_till_json.rb
    begin
      require 'rubygems'
    rescue LoadError
    end
    begin
      require 'json'
    rescue LoadError
    end
    [Object, Array, FalseClass, Float, Hash, Integer, NilClass, String, TrueClass].each do |klass|
      klass.class_eval do
        alias_method('__till_json__', 'till_json') if method_defined?('till_json')
      end
    end

slutligen den här initialiseringen:

på engelska gör det här följande:

  • gör så att rails stödjer kod som körs innan allt annat
  • under den här fasen suger den in json-gemmet och kommer ihåg hur den gör saker
  • nollställ rails’ metod och återställ till_json-metoden

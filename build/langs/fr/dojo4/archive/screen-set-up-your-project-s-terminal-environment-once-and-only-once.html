Avec n'importe quel projet, j'exécute plusieurs programmes simultanément pendant le processus de développement. Cela entraîne généralement plusieurs onglets dans l'application Terminal - un pour mon serveur local, un pour ma base de données, un pour mon application client, et un pour git. (Pour l'instant, j'utilise un éditeur GUI (ST2), mais je pourrais bientôt revenir à vim...) Cette configuration à plusieurs onglets se reproduit pour chaque projet sur lequel je travaille, et je passe généralement d'un projet à l'autre plusieurs fois par jour.

Avec cette configuration à plusieurs onglets, lorsque je dois changer de projet, je dois soit ouvrir une nouvelle fenêtre Terminal avec ses propres onglets pour ce projet et gérer un nombre de fenêtres supérieur à celui nécessaire (pas une option pour un minimaliste comme moi), soit fermer tous les onglets que j'ai ouverts, pour les réouvrir tous lorsque je reviens à ce projet.

Eh bien, comme la plupart des jours, j'ai appris une solution astucieuse à ce problème de la part du leader sage et tout-puissant de dojo4, Ara. Il utilise le programme GNU Screen. Au sein d'un seul onglet Terminal, Screen vous permet d'ouvrir plusieurs "fenêtres" (alias onglets) dans le même onglet Terminal. Voici comment l'utiliser :

# créer et se reconnecter à une session
screen -S [nom] = créer un écran nommé
screen -D -R [nom] = se reconnecter à un écran nommé et déconnecter les autres personnes si nécessaire
screen -list = lister les écrans

# (quitter toutes les fenêtres dans une session screen quittera cette session. Cependant, une session screen est un processus en cours d'exécution, vous pouvez donc la tuer manuellement.)

# dans une session screen :
ctrl-A + ? = aide
ctrl-A + " = lister les fenêtres
ctrl-A + c = créer une nouvelle fenêtre
ctrl-A + n = fenêtre suivante
ctrl-A + p = fenêtre précédente
ctrl-A + A = renommer une fenêtre

# alias que nous utilisons
alias sl='screen -list '
alias sdr='screen -d -r '
alias s='screen -D -R '
alias ss='screen -S'

sdr [nom] = se reconnecter à un écran nommé et déconnecter les autres personnes si nécessaire
ss [nom] = créer un écran nommé
sl = lister les écrans

Une autre fonctionnalité vraiment cool de screen est que lorsque vous vous déconnectez d'une session screen, toutes vos fenêtres restent en cours d'exécution, et lorsque vous vous reconnectez à la session screen, vous avez toutes vos fenêtres à votre disposition.

Lorsque nous en avons discuté dans notre bureau, Ryan Cook a créé un petit assistant Ruby sympa pour créer des sessions screen nommées d'après votre répertoire actuel.

Mon flux de travail est donc passé de l'ouverture de plusieurs onglets dans Terminal à un seul onglet dans Terminal où je me reconnecte à une session screen pour chaque projet sur lequel je travaille. Cela me permet de changer de projet extrêmement rapidement et d'arrêter de perdre du temps à réinitialiser et à réorganiser tous les onglets Terminal que j'utilise pour un projet.

Maintenant, je dois convertir plus de choses en utilisation du terminal (comme mon éditeur) afin de vraiment tirer le meilleur parti de screen...

/fix-for-caching-credentials-on-https-github-repos
publié le: 2012-06-06

github a récemment basculé vers un schéma https par défaut pour le clonage des dépôts. en conséquence, vous pouvez soudainement être invité à saisir un ‘Nom d'utilisateur’ et un ‘Mot de passe’ lorsque vous poussez, alors qu'auparavant, vous pouviez le faire sans saisir de identifiants. la solution consiste à faire en sorte que git mette en cache les identifiants https, ce qui est facile, puisque git utilise curl en coulisses

dans votre répertoire personnel, créez un fichier appelé ‘.netrc’, par exemple

/Users/ahoward/.netrc

mettez-y ces contenus

machine github.com
login VOTRE_NOM_D_UTILISATEUR_GITHUB
password VOTRE_MOT_DE_PASSE_GITHUB

corrigé !

réf : https://gist.github.com/2885020

Les clients veulent souvent savoir si et pourquoi une agence, comme la nôtre, facture le temps que nous passons à corriger des bugs. L'idée fausse que le développement peut se faire complètement sans bugs est compréhensible et courante. Ara, notre fidèle CTO, vient de répondre à cette question/préoccupation avec brio, et nous espérons que cela a été utile pour le client (et peut-être pour vous dans vos communications sur des fronts similaires) :

QUESTION :
Nous facturez-vous les bugs qui sont le résultat du développement effectué par votre équipe ?

RÉPONSE :
Une bonne, et courante question.

Nous ne facturons jamais la négligence, cependant les bugs sont une autre histoire…

Tout logiciel contient des bugs, tout le temps : google chrome, par exemple, a actuellement 59767 bugs ouverts ->
https://code.google.com/p/chromium/issues/list.

Le logiciel web est particulièrement vulnérable puisqu'il cible de nombreuses plateformes : tous les sites, de Google à Facebook, auront également de nombreux bugs à tout moment dans certains navigateurs. Souvent, les navigateurs eux-mêmes "corrigent" ces bugs. Parfois, le code qui fonctionnait mardi ne fonctionne plus mercredi en raison d'une mise à jour de version - c'est une petite course amusante à laquelle nous jouons dans le secteur du logiciel ;-)

Donc, bien sûr, les bugs font partie du développement de logiciels et aucun développeur ne peut écrire un logiciel sans bugs tout le temps, surtout lorsque des dépendances telles que les API, les opérations, les délais variables et les changements de conception/ux sont également des facteurs.

Nous essayons vraiment, vraiment de facturer aux clients uniquement pour les logiciels fonctionnels, et vous verrez certainement quelques éléments de facture facturés à 0 $ puisque nous faisions des réunions/révisions internes et/ou parce qu'ils étaient des bugs flagrants.

En fin de compte, c'est un jugement qui me ferait plaisir de discuter en personne.

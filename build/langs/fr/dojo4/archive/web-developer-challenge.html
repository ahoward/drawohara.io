Aujourd'hui, je fais des tests de compatibilité inter-navigateurs pour un nouveau site avant de le lancer. Comme d'habitude, cela implique quelques acrobaties de style pour que le site soit présentable dans Internet Explorer. Mais aujourd'hui, je suis tombé sur un problème dans Firefox que je ne pouvais pas expliquer.

Le site que je construis fait la promotion d'un événement cycliste. Il comporte quelques blocs comme celui-ci :

<section class="sponsor">...</section>

Ils contiennent des informations sur les sponsors de l'événement. Étonnamment, bien que ces blocs apparaissent dans le navigateur préféré de tous, âgé de dix ans, ils n'apparaissent dans aucune version de Firefox sur mon disque dur.

Sérieusement ? Oui.

Ces blocs sont invisibles dans Firefox. Le navigateur les modifie pour leur donner un attribut display:none;.

Mais le meilleur, c'est que si je change la classe de la section en autre chose que "sponsor", les blocs apparaissent parfaitement. En fait, une fois que vous connaissez la cause, c'est la seule solution : ne donnez pas aux éléments de votre page une classe CSS de "sponsor".

Pensez-vous savoir ce qui a causé cela ? Vérifiez cela.

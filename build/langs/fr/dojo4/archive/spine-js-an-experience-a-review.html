Donc, nous aimons essayer des frameworks front-end aléatoires ici chez dojo4 sur des projets, et le dernier que nous avons essayé est SpineJS. Voici mes pensées après l'avoir utilisé sur quelques projets :

Avantages :

1. Code court et open source. Chaque fois que j'avais une question sur la manière dont le framework interprétait mon code, il était incroyablement simple d'ouvrir les fichiers source et de lire le code moi-même.

2. Outil de serveur web solide avec Hem Tout ce dont vous avez besoin pour lancer une application front-end est fourni avec hem et SpineJS. Les tests sont automatiquement configurés et faciles à modifier. Hem dispose également d'un joli drapeau -d, qui ne minifiera pas votre application pour la rendre plus facile à déboguer en développement.

3. Couche de persistance de données intégrée SpineJS est livré avec des adaptateurs Ajax et LocalStorage pour vos modèles qui sont très faciles à utiliser et à sous-classer. Il est également facile d'écrire les vôtres puisque vous avez le code source des adaptateurs Ajax et LocalStorage et qu'ils sont assez courts.

4. Groupe Google et communauté github actifs Chaque fois que j'avais des questions en cours de route, il y avait toujours quelqu'un dans la communauté capable de donner une idée. Ils ont un Groupe Google et une organisation github active.

4. Coffeescript est génial. Ce point concerne moins SpineJS, mais SpineJS était ma première utilisation de Coffeescript et c'est génial. Je devrais écrire un article de blog entièrement sur Coffeescript mais il est génial pour sa facilité de lecture sans toutes ces parenthèses et points-virgules encombrants, la portée des fonctions avec la flèche grasse, et le point d'interrogation pour m'éviter d'avoir à faire des tonnes de conditionnels existentiels. Et oh oui – les classes.

Inconvénients :

La documentation n'est pas totalement complète. Je suppose que ce point n'est pas tout à fait juste car leur documentation est plus une collection d'exemples expliqués plutôt qu'une véritable documentation d'API, mais le fait est qu'il y a des fonctions dans le framework qui ne sont pas abordées dans la documentation. Cependant, avec le point #1 ci-dessus, il est super facile de lire le code source vous-même.

Globalement :

Globalement, je recommanderais ce framework et il est très probable que je l'utilise à nouveau. Entre la simplicité et la facilité d'utilisation de Coffeescript et SpineJS, ils semblent faits l'un pour l'autre au paradis du web.

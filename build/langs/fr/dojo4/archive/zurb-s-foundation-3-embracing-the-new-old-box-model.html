La semaine dernière, les gens intelligents de ZURB ont lancé la version 3.0 de leur framework CSS Foundation, un concurrent de Bootstrap de Twitter.

Ce qui m'a frappé immédiatement à propos de la dernière version de ZURB, c'est qu'elle réinitialise globalement le modèle de boîte du W3C, une stratégie que Paul Irish a proposée dans un récent article de blog. Cela signifie que Foundation 3 n'offre aucun support pour IE7.

Bootstrap revient également au modèle de boîte traditionnel pour ses mises en page row-fluid, en ajustant les largeurs et les marges à l'aide d'un star hack pour permettre le support de IE7. Mais ses mises en page fixes utilisent toujours le modèle W3C.

Personnellement, je suis un grand fan de la stratégie de réinitialisation du modèle de boîte, et je sais que notre équipe backend serait ravie—lorsqu'ils déposent du contenu dans les vues, ils utilisent des tableaux juste pour éviter tout l'arcane entourant le modèle W3C. Le modèle de boîte traditionnel a simplement du sens.

De plus, je suis excité par la perspective que les développeurs—spécifiquement, ceux qui contribuent aux frameworks CSS—aient une main dans la façonnage du paysage des navigateurs supportés. Aussi agréable que cela ait été d'avoir des grandes entreprises technologiques s'intéresser à ce paysage, il est encore mieux pour nous de prendre possession de celui-ci.

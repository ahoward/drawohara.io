/fix-for-caching-credentials-on-https-github-repos
gepubliceerd op: 2012-06-06

github heeft recentelijk overgeschakeld naar een https-schema als standaard voor het klonen van repos. Als bijwerking kan het voorkomen dat je plotseling om een 'Gebruikersnaam' en 'Wachtwoord' wordt gevraagd wanneer je pusht, terwijl je dat eerder zonder het invoeren van referenties kondet. De oplossing is om git te laten cachen van https-referenties, wat eenvoudig is, aangezien git curl onder de motorkap gebruikt.

maak in je thuisdirectory een bestand genaamd '.netrc', bijvoorbeeld

/Users/ahoward/.netrc

zet hier de volgende inhoud in

machine github.com
login Jouw_GITHUB_GEBRUIKERSNAAM
password Jouw_GITHUB_WACHTWOORD

opgelost!

ref: https://gist.github.com/2885020

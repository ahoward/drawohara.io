Bij elk project dat ik doe, gebruik ik meerdere programma's tegelijkertijd tijdens het ontwikkelen. Dit resulteert meestal in meerdere tabbladen in de Terminal-app - een voor mijn lokale server, een voor mijn database, een voor mijn client-applicatie en een voor git. (Op dit moment gebruik ik een GUI-editor (ST2), maar ik zal misschien snel terugkeren naar vim...) Deze setup met meerdere tabbladen herhaalt zich voor elk project waar ik aan werk, en ik wissel meestal meerdere malen per dag tussen verschillende projecten.

Met deze setup met meerdere tabbladen moet ik, als ik van project verander, ofwel een nieuw Terminal-venster openen met zijn eigen tabbladen voor dat project en omgaan met meer vensters dan nodig (geen optie voor een minimalist als ik) of alle tabbladen die ik open heb sluiten, om ze vervolgens allemaal weer te openen wanneer ik terugkeer naar dat project.

Nu, zoals de meeste dagen, heb ik een slimme oplossing voor dit probleem geleerd van de wijze en almachtige leider van dojo4, Ara. Hij gebruikt het GNU Screen-programma. Binnen één Terminal-tabblad laat Screen je meerdere "vensters" (aka tabbladen) openen binnen hetzelfde Terminal-tabblad. Dit is hoe je het kunt gebruiken:

# een sessie maken en opnieuw verbinden
screen -S [naam] = maak een benoemd scherm
screen -D -R [naam] = verbind opnieuw met een benoemd scherm en ontkoppel andere personen indien nodig
screen -list = lijst van schermen

# (alle vensters in een schermsessie sluiten zal die sessie beëindigen. Echter, een schermsessie is een actief proces, dus je kunt het handmatig stoppen.)

# binnen een schermsessie:
ctrl-A + ? = help
ctrl-A + " = lijst van vensters
ctrl-A + c = nieuw venster maken
ctrl-A + n = volgend venster
ctrl-A + p = vorige venster
ctrl-A + A = hernoem een venster

# aliassen die we gebruiken
alias sl='screen -list '
alias sdr='screen -d -r '
alias s='screen -D -R '
alias ss='screen -S'

sdr [naam] = verbind opnieuw met een benoemd scherm en ontkoppel andere personen indien nodig
ss [naam] = maak een benoemd scherm
sl = lijst van schermen

Een echt geweldige functie van screen is dat wanneer je je afkoppelt van een schermsessie, alle je vensters blijven draaien, en wanneer je je opnieuw verbindt met de schermsessie, heb je al je vensters weer ter beschikking.

Toen we dit in ons kantoor bespraken, maakte Ryan Cook een mooi klein Ruby-hulpprogramma om schermsessies te maken die zijn genoemd naar je huidige map.

Mijn werkstroom is veranderd van het hebben van meerdere tabbladen in Terminal open naar één tabblad in Terminal waar ik me opnieuw verbind met een schermsessie voor elk project waar ik aan werk. Dat laat mij snel van project wisselen en stopt het verliezen van tijd met het opnieuw instellen en opnieuw afbreken van alle Terminal-tabbladen die ik gebruik voor een project.

Nu moet ik alleen nog meer dingen overzetten naar terminalgebruik (zoals mijn editor) om echt het meeste uit screen te halen...

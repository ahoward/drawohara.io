Gepubliceerd op: 2013-10-24

Vaak maakt het zin om ontwikkeling te doen in HTML, JS en CSS en die inhoud vervolgens te gebruiken in een native app. Phonegap beweert de beste oplossing te zijn voor dit geval, maar uit mijn ervaring is het een van de slechtste.

Als reactie op Phonegap heb ik de start-repo van ons bedrijf voor iOS-apps die webinhoud in een UIWebView moeten gebruiken, open source gemaakt en Imbed genoemd.

Kenmerken:

  • Klein & Open Source. Een UIViewController, een HTML-bestand en het super kleine, 1 klasse, open source WebViewJavascriptBridge project.

  • Ondersteun elke iOS-functie onmiddellijk. Geen wachten tot een derde partij ontwikkelaar een plugin maakt. Je schrijft zelf natieve Objective-C code. Dus zodra Apple het uitbrengt, kun je het ondersteunen.

  • Volg de gidsen van Apple, niet die van iemand anders. Apple doet een geweldig werk bij het documenteren van het gebruik van de functies van iOS. De meeste derde partij ontwikkelaars doen dat niet. Welke zou je liever volgen?

  • Automatiseer alles met Rake. Concentreer je op het schrijven van je code, test deze in de browser en automatiseer vervolgens alles in een duidelijke, open source Rakefile. Heck, zelfs archiveren, verzenden naar Testflight en melden bij Campfire.

Het is jouw app. Vertrouw niet op een derde partij tool om functies te ondersteunen die iOS al biedt. Pak je code bij de horens.

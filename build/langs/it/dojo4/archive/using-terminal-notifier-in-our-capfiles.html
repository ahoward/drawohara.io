Utilizziamo capistrano per i nostri deployment e recentemente ho scoperto il fantastico terminal-notifier gem che utilizza le notifiche utente in OS X 10.8 e superiori per mostrare una bella notifica dalla riga di comando. Ogni volta che effettuo un deployment, controllo manualmente se il deployment è terminato, e questa è una buona soluzione per evitarlo. Un problema che ho riscontrato, tuttavia, quando l'ho aggiunto a uno dei nostri progetti, è che alcune persone del nostro team utilizzano una versione di Mac OS X prima della 10.8 (lo so!), quindi il comando terminal-notifier fallirebbe e il loro deployment avrebbe errori. Ecco quindi il codice che uso ora per assicurarmi che non compaiano errori se non è possibile utilizzare le notifiche utente più recenti di os x:

desc 'usa terminal notifier se in > osx 10.8 per usare un avviso utente'
task :terminal do
 begin
  require 'terminal-notifier'
        url         = fetch(:url)
        application = fetch(:application)
        system "terminal-notifier -title #{ application } -message 'Deploy completato con successo a #{ url }.' -sound default"
 rescue LoadError
        warn "Probabilmente non sei su os x 10.8, quindi non stai usando terminal notifier. Se lo sei, usa gem install terminal-notifier per utilizzare alcune notifiche utente fantasiose."
 end
end

Ovviamente includiamo anche gem terminal-notifier nel nostro Gemfile. Vedi la mia domanda su SO sull'inclusione di questa gem nel tuo gemfile per una spiegazione del motivo per cui sto utilizzando la semplice istruzione di inclusione della gem.

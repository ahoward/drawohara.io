  • Nessun doctype. Rende difficile capire quale versione di HTML sia, il che dovrebbe portare a risposte interessanti…

  • Mescola tag vuoti stile xhtml con tag HTML.

  • Virgolette inconsistenti. Entrambi funzionano, scegline uno già…

  • riga 1: DELETE non è un'azione supportata nei moduli HTML:
  • method=delete non è direttamente supportato nella maggior parte/tutti? i browser
  • i moduli non supportano delete nei browser più vecchi
  • Metodo delete

  • form method=’DELETE’ – non è permesso

  • enctype=”multipart/form-data” non enctype=”multipart/file-upload”

  • enc-type=”multipart/file-upload non ha senso se la nostra intenzione è di DELETE

  • i tag di input di testo nel primo fieldset hanno lo stesso nome
  • tag “” duplicati. O due tag con lo stesso nome nuovamente
  • 2 campi di testo chiamati “tags” solo l'ultimo verrà utilizzato.
  • Due input con lo stesso nome ‘tags’. Il secondo sovrascriverebbe il valore del primo.
  • Due input con nome ‘foo’ (non auto chiusi)
  • i campi di input di testo con lo stesso nome sovrascriverebbero re: input con nome tags
  • assegnazioni duplicate al nome dell'input=’tags’

  • assegnazioni duplicate al nome dell'input=’vehicle’

  • Nessun for sull'etichetta ‘Bar:’.

  • etichetta non associata: (etichetta per cosa?)

  • input readonly senza valore è inutile

  • action=”./” sembra una cattiva pratica per me

  • l'azione del modulo non è valida come descrittore di percorso di file unix – per inviare all'URL corrente, rimuovi l'azione completamente poiché quella è l'impostazione predefinita

  • il caricamento del file non può avere un attributo di valore (cattiva sicurezza)
  • Nidificazione di fieldset
  • gli input di file non prendono valori predefiniti – l'utente deve selezionare

  • seleziona automaticamente il tuo file ~/.bashrc per il caricamento

  • disabled dovrebbe avere virgolette attorno e (anche una cattiva combinazione di virgolette singole e doppie in tutto)
  • Opzioni non chiuse per il menu a discesa di selezione e nessun testo incluso

  • virgolette mancanti attorno all'attributo disabled nella dichiarazione errata dell'input della password

  • Nessun tipo nel campo password (né nel campo email, ma meno importante lì)
  • il campo password dovrebbe essere di tipo password

  • dovrebbe essere <input type=’email’ name=’email’ …>

  • disabled=disabled dovrebbe essere disabled

  • Nessun testo per l'opzione femminile, né pulsante di opzione per i pulsanti di opzione ‘Sex’, i pulsanti di opzione non sono configurati correttamente per l'invio del valore / commutazione nel gruppo. È tutto sbagliato in quell'area.

  • gli input radio non prendono opzioni, ma dovrebbero invece avere lo stesso nome con diversi valori

  • i pulsanti devono essere aperti e chiusi – il submit dovrebbe essere un input invece
  • le caselle di controllo con lo stesso nome sovrascriverebbero re: input con casella di controllo con nome vehicle

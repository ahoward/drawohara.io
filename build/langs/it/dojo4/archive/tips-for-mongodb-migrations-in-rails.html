Volevo condividere un metodo rapido e semplice per testare le migrazioni di Rails quando si utilizza il database MongoDB. La flessibilità di mongo e ruby rende questa operazione piuttosto semplice. In questo esempio, rinomineremo un campo nella nostra collezione di esempio "libri" da "isbn" a "numero_libro". Questo è un tipo di migrazione abbastanza comune e, una volta che avrai familiarità con questo semplice caso, le migrazioni più complesse seguiranno lo stesso schema. Prima, generiamo il nostro script di migrazione boilerplate con timestamp.

rails generate migration rename_isbn_to_book_number

Poi modificheremo il file generato sotto db/migrate. Suddivideremo il nostro codice in diverse sezioni come segue.

  1. Alcune costanti di classe condivise da tutti i metodi
  2. Il metodo self.up che esegue la migrazione in avanti
  3. Un metodo ausiliario per impostare alcuni dati di esempio per testare la migrazione in avanti
  4. Il metodo self.down che esegue il rollback
  5. Un metodo ausiliario per impostare alcuni dati di esempio (se necessario) per testare il rollback

Prima, definiamo semplicemente alcune costanti che i nostri metodi utilizzeranno. Memorizziamo l'hash delle opzioni di aggiornamento di mongo MultiUpdate per comodità, poiché la maggior parte delle operazioni di aggiornamento della migrazione desidera upsert false (non creare nuovi documenti), multi true (aggiorna tutti i documenti corrispondenti) e safe true.

Definiamo anche una costante per il nome della collezione. Per la migrazione reale, la collezione è "libri", ma per il test, creeremo una nuova collezione chiamata "test_libri_migrazione" mentre sviluppiamo il nostro codice.

class RenameIsbnToBookNumber < Mongoid::Migration
  MultiUpdate = {:upsert => false, :multi => true, :safe=>true}
  Collection = db.collection("libri") #Codice di produzione finale
  Collection = db.collection("test_libri_migrazione") #Solo per il test sulla console
  def up
  end

  def down
  end
end

OK, questo è il nostro boilerplate iniziale. Il passaggio successivo è fare un backup del nostro database di sviluppo se ci sono dati lì che non vogliamo accidentalmente danneggiare. Poi iniziamo a codificare un piccolo metodo ausiliario per popolare la nostra collezione di test con documenti fittizi che assomigliano a quelli che ci aspettiamo di vedere in produzione, concentrandoci solo sui campi rilevanti per la migrazione. Aggiungi questo metodo alla tua classe di migrazione.

def mock_data_for_testing_up
  3.times {|number| Collection.insert({"isbn" => "#{number}"})}
end

def show_collection
  Collection.find({}).each {|_| puts _}
end

Questo creerà 3 documenti fittizi che possiamo utilizzare per il test. Stiamo inserendo questo codice in un metodo in modo che sia facile ripetere mentre regoliamo e testiamo il nostro codice di migrazione. Per migrazioni complesse, potrebbero essere necessari molti round di regolazione per coprire tutti i casi limite.

Quindi ora abbiamo una collezione di test separata e ben compresa pronta per testare la nostra semplice migrazione. Codifichiamo il nostro metodo up. Per eseguire la nostra migrazione.

def up
  #Vogliamo rinominare il campo libro.isbn in libro.numero_libro
  Collection.find({"isbn" => {"$exists" => 1}}).each do |book|
    update_op = {
      "$unset" => {"isbn" => 1},
      "$set" => {"numero_libro" => book["isbn"]}
    }
  Collection.update({"_id" => book["_id"]}, update_op, MultiUpdate)
end

Possiamo incollare questo nella console ed eseguirlo per testare la nostra migrazione. Possiamo verificare i risultati con show_colletion. Se vogliamo testare altri record per il rollback, possiamo creare un metodo mock_data_for_testing_down.

Questo dovrebbe darti un modo molto rapido per sperimentare e far funzionare il tuo codice di migrazione. Mongo ha alcune capacità di query e modifica avanzate che possono fare cose sorprendenti, e un modo facile per fare qualche prova ed errore è utile. Se fai un pasticcio con i tuoi dati di test, puoi utilizzare Collection.drop per ottenere una tabula rasa. Ecco il codice di migrazione finale per riferimento. Non dimenticare di rimuovere la costante della collezione di test e di eliminare la collezione di test dal tuo database quando sei pronto per iniziare a eseguire il tuo codice per davvero con rake db:migrate.

class RenameIsbnToBookNumber < Mongoid::Migration
  MultiUpdate = {:upsert => false, :multi => true, :safe=>true}
  Collection = db.collection("libri") #Codice di produzione finale

  def up
    #Vogliamo rinominare il campo libro.isbn in libro.numero_libro
    Collection.find({"isbn" => {"$exists" => 1}}).each do |book|
      update_op = {
        "$unset" => {"isbn" => 1},
        "$set" => {"numero_libro" => book["isbn"]}
      }
      Collection.update({"_id" => book["_id"]}, update_op, MultiUpdate)
    end
  end

  def down
    #Vogliamo rinominare il campo libro.numero_libro in libro.isbn
    Collection.find({"numero_libro" => {"$exists" => 1}}).each do |book|
      update_op = {
        "$unset" => {"numero_libro" => 1},
        "$set" => {"isbn" => book["numero_libro"]}
      }
      Collection.update({"_id" => book["_id"]}, update_op, MultiUpdate)
    end
  end

  #Questi metodi non vengono chiamati dalla migrazione. Solo per il test manuale
  #copiando/incollando nella console
  #Per testare (copiando/incollando da qui alla console)
  #1. Imposta la costante MultiUpdate. Regola "Collection" affinché sia una collezione di test
  #2. Copia/incolla i 2 metodi seguenti
  #2. Esegui mock_data_for_testing_up
  #3. Esegui il corpo di self.up
  #3b. Facoltativamente, esegui nuovamente il corpo di self.up per assicurarti che sia idempotente
  def mock_data_for_testing_up
    3.times {|number| Collection.insert({"isbn" => "#{number}"})}
  end

  def show_collection
    Collection.find({}).each {|_| puts _}
  end
end
